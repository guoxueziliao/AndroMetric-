# Changelog

所有显著变更记录于此文件。

## [0.2.13] - 2026-05-30

### Added
- 新增健康项目与补剂周期系统 v1：用户可创建补剂、晒太阳、拉伸、康复、作息干预等健康项目。
- 支持每日、隔日、每周若干天、连续 N 天和按需五种周期。
- 补剂项目支持剂量、单位和服用时间标签（早晨/中午/傍晚/睡前/饭前/饭后/随时）。
- 项目状态支持进行中、暂停、恢复、结束和归档。
- Dashboard 新增今日健康项目卡片，一键完成/跳过。
- 我的页面新增健康项目管理入口。
- Dexie v9 新增 health_projects、health_project_plans、health_project_logs 三张表。
- LATEST_VERSION=50，snapshot/import/export/reset 全覆盖新表。
- 旧 supplements 字段继续作为 legacy daily fact 兼容读取。

## [0.2.12] - 2026-05-30

### Added
- 新增个人阶段回顾 v1：Stats 新增"回顾"tab，按月度/季度聚合个人常态、解释卡片、观察计划和经验卡。
- 支持本月、上月和本季度三个窗口选择。
- 只读派生，不新增 schema，不持久化，不输出评分或诊断。
- 章节可折叠，缺数据时隐藏而不展示空壳。
- 展示记录缺口和限制说明。

## [0.2.11] - 2026-05-30

### Added
- 新增个人经验沉淀 v1：观察计划结束后可手动保存经验卡。
- 经验卡包含事实摘要（系统预填）和用户心得（手动输入），存储复用 GoalCheckin.note，不新增 schema。
- Stats 常态 tab 新增经验卡历史区域，支持按上下文类型筛选。
- 观察计划回看新增"保存这次观察经验"入口。
- 经验卡默认私密本地保存，不导出、不分享。

## [0.2.10] - 2026-05-30

### Added
- 新增个人观察计划 v1：从解释卡片手动创建 7 / 14 天短期观察计划。
- 观察计划复用 TrainingGoal（新增 'observation' 分类），不新增 schema。
- Stats 常态 tab 新增观察计划区域，展示进行中和已完成的观察。
- 解释卡片新增"记录一段时间再看"入口，弹出确认弹层选择窗口。
- 完成后展示事实总结：记录天数、缺失天数和限制说明，不做任何结论。
- Dashboard 新增观察计划轻提示。

## [0.2.9] - 2026-05-30

### Added
- 个人常态新增变化解释层：当指标偏离个人常态时，展示同一窗口内的上下文观察卡片。
- 解释卡片覆盖 6 类上下文：睡眠、压力、性活动负荷、运动、饮酒、屏幕使用。
- 每张卡片含窗口、样本量、置信度和限制说明，不做因果结论。
- Stats 常态 tab 内嵌解释层，无独立工作台。
- Dashboard 新增"变化提示"轻提示，可跳转到统计查看。

## [0.2.8] - 2026-05-30

### Added
- 新增个人常态系统 v1：从用户 90 天历史中派生个人基线，用 14 / 30 天窗口比较最近状态。
- Stats 新增"常态"tab，展示指标卡片（三状态判定：正常、偏离、数据不足）、记录缺口和全局限制说明。
- 指标分两层：第一层（硬度、睡眠、压力、性负荷）为核心，第二层（色情使用、满意度、运动、目标历史、关系上下文）为扩展。
- 置信度基于覆盖率降级：<50% 仅事实、50-69% 最多 low、>=70% 最多 medium，总体 summary 需 >=50% 指标达 medium。
- 新增 `personalNormalEngine.ts` 纯函数引擎（中位数、P25-P75、MAD、isScalar 零过滤）。

## [0.2.6] - 2026-05-30

### Added
- 导出新增"全部 / 按日期"区间切换，JSON 始终全量备份，CSV 按日期过滤。
- JSON 导出改用 `createSnapshot()` 确保覆盖全部 10 个数据存储。

- 导出格式网格改为 2 列。

## [0.1.2] - 2026-05-26

### Added
- 建立版本号单一来源，应用显示、导出元信息与构建版本统一来自 `package.json`。
- 新增关于弹窗，集中展示应用版本、数据版本、IndexedDB 版本、隐私说明与更新历史。
- CSV 导出包新增 `meta.json`，记录 `appVersion`、`dataVersion` 与导出时间。
- PWA 更新改为用户可见提示，用户点击刷新后才切换到新版本。
- 备份快照写入后会立即读回自检，发现异常会回滚失败快照并提示错误。
- 新增 idle 自动安全快照，应用空闲且距离上次自动快照超过 24 小时时自动备份。
- 导入合并新增字段级冲突预览，可选择全部保留当前值或全部使用导入值。
- 新增导出选项弹窗，支持按日期区间、数据维度和 JSON / CSV / Markdown 格式选择性导出。
- 自动安全快照保留数可配置为 3 / 7 / 15 / 30，手动快照继续不限数量。

### Changed
- 导入预览明确区分 `dataVersion` 与 `appVersion`：只有 `dataVersion` 决定兼容性，`appVersion` 仅用于展示。
- 用户设置中不再保存应用版本号，避免旧本地设置卡住旧版本显示。
- 导入和导出统计现在覆盖日志、伴侣、标签、周期事件、怀孕事件与内部快照。

# 硬度日记 (Hardness Diary) v0.0.6

**硬度日记** 是一款专为男性设计的隐私优先、数据驱动的健康追踪应用。它不仅记录数据，更致力于通过科学的量化分析（如晨勃质量、睡眠周期、生活习惯关联），帮助您找回身体的最佳状态。

## v0.0.6 新特性

*   **🧬 全局时间轴 (Global Timeline)**: 自动将睡眠、运动、咖啡因摄入与性活动串联，直观展现一天的精力流向。
*   **💯 数据质量评分 (Quality Score)**: 引入实时评分算法，通过反馈记录完整度，鼓励建立高质量的健康基准线。
*   **🛡️ 标签治理体系 (Tag System)**: 包含自动“标签体检”与反熵规范，防止长期记录导致的语义污染（如合并同义词、拆分复合标签）。
*   **🖐️ 自慰记录 2.0**: 重构生理维度，新增**射精强度（量/力）**评级与**贤者时间**身心状态评估。
*   **☕ 咖啡因追踪**: 独立追踪咖啡、茶与功能饮料摄入，探究兴奋剂对睡眠质量及次日晨勃的长期影响。

## 主要功能

*   **核心追踪**: 晨勃硬度、维持时间、睡眠质量与梦境记录。
*   **性爱日记**: 支持多伴侣、多阶段互动、体位/道具/着装细节的全维度记录。
*   **生活方式**: 饮酒（纯酒精换算）、咖啡因、运动、压力与心情。
*   **隐私安全**: **本地优先 (Local-First)**，数据仅存设备，支持 JSON 导出备份与快照回滚。
*   **数据洞察**: 
    *   **趋势分析**: 7日移动平均线揭示身体活力变化。
    *   **XP 雷达**: 基于六维标签体系的性偏好深度画像。
    *   **因子相关性**: 自动分析熬夜、酒精、压力对硬度的具体抑制程度。

## 开发指南

本项目使用 React + TypeScript + Tailwind CSS 构建，数据存储基于 IndexedDB (Dexie)。

⚠️ **重要提示**: 在进行任何代码修改前，请务必阅读 **`DEVELOPER_MANUAL.md`**。该文档详细记载了项目的核心架构、日期判定逻辑（Physiological Day）、以及 V37 版本的 ContentItem 迁移规则。

### 快速开始

```bash
npm install
npm run dev
```

### 项目结构

```
.
├── components/      # UI 组件 (Modals, Charts, Forms)
├── contexts/        # 全局状态管理
├── docs/            # 系统治理规范 (宪章/标签规则)
├── hooks/           # 自定义 Hooks (useLogs, useLocalStorage)
├── services/        # 业务逻辑层 (Storage, Logger, Plugins)
├── utils/           # 工具函数 (Migration, Stats, Validators)
├── types.ts         # TypeScript 类型定义
└── db.ts            # Dexie 数据库配置
```

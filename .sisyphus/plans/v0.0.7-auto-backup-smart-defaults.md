# 硬度日记 v0.0.7 - 自动备份与智能默认值

## TL;DR

**目标**: 实现自动备份系统和智能默认值推荐

**核心功能**:
1. **自动备份**: 每次记录保存后自动导出到本地文件，智能分级保留策略
2. **智能默认值**: 基于历史数据学习用户习惯，智能推荐字段值

**技术重点**:
- File System Access API 实现本地文件备份
- 智能算法：工作日/周末模式 + 30天历史学习
- TDD 开发方式

**工作量**: 1周+ | **并行度**: 高（6-8个并行任务）

---

## Context

### 项目状态
- **当前版本**: 0.0.6
- **数据版本**: 38
- **数据库版本**: 5 (Dexie.js v4)

### 需求来源
**用户痛点**:
1. 数据安全担忧 - 希望自动备份到本地文件
2. 快速记录繁琐 - 默认值不合适，需要智能推荐

**已确认方案**:
- 自动备份: 每次保存后触发，File System Access API，智能分级保留
- 智能默认值: 工作日/周末模式 + 30天历史学习

---

## Work Objectives

### Core Objective
实现自动备份到本地文件系统，并基于历史数据提供智能默认值推荐，提升用户数据安全性和记录效率。

### Concrete Deliverables
- `services/BackupService.ts` - 自动备份服务
- `utils/smartDefaults.ts` - 智能默认值引擎
- `hooks/useSmartDefaults.ts` - 智能默认值 Hook
- `components/BackupSettings.tsx` - 备份设置面板
- 新增测试: `*.test.ts` (TDD)
- 数据版本升级: 38 → 39

### Definition of Done
- [ ] 每次保存记录后自动触发文件备份
- [ ] 备份文件使用分级保留策略（12周+12月）
- [ ] 晨间打卡自动推荐硬度等级
- [ ] 睡眠记录自动推荐质量、时长
- [ ] 所有测试通过（TDD）
- [ ] 代码通过类型检查（tsc --noEmit）

### Must Have
- File System Access API 集成
- 自动备份触发机制
- 智能默认值算法（工作日/周末 + 30天）
- 用户可配置备份开关
- 备份失败优雅降级

### Must NOT Have (Guardrails)
- 不要实现真正的云端同步（超出Local-first原则）
- 不要过度复杂的AI算法（简单统计即可）
- 不要破坏现有的版本历史功能
- 不要要求用户每次手动选择备份位置


# UI / 使用逻辑规范

## 设计心智模型

用户每次只在处理 **一个** 素材

## 创建流程（强制）

1.  选择内容类型
2.  → 选择平台
3.  → 填写素材详情
4.  → 保存

**保存后：**
*   回到素材列表
*   显示为一条独立素材卡片

## 多素材使用方式

每次点击「+ 添加素材」
⬇️
创建一个新的 ContentItem

**不支持在同一界面同时编辑多个素材**

## 素材列表行为

**支持：**
*   编辑单个素材
*   删除单个素材

**不支持：**
*   批量修改平台
*   批量合并素材

## 编辑保护规则

**编辑某个 ContentItem：**
*   仅影响该条素材
*   不联动其他素材

**删除素材：**
*   不影响自慰记录本体
*   不影响其他素材

## 迁移素材的 UI 提示

若 `notes` 含迁移说明：
*   在素材卡片显示 ⚠️ 标记
*   点击可查看说明
*   **不强制修改**

## 错误与缺失提示（轻量）

*   缺少 `type`：提示「未选择内容形式」
*   缺少 `platform`（非回忆/幻想）：提示「未选择来源」

**不阻断保存（v0.0.6 阶段）**

## 明确不做的事

❌ 自动推荐类型
❌ 根据平台猜内容
❌ 根据 XP 自动补全素材信息
❌ 自动拆 / 合并用户素材

## 最终声明

**ContentItem 是素材系统的最小可信单位**
一旦确立，不再回退到“多选大杂烩结构”

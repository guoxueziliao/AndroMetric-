## 当前状态

- 状态：completed。
- 前置：0.2.3 复盘引擎；0.2.5 目标历史；0.2.6 数据安全；0.2.7 关系上下文。
- 主方向：从短期复盘推进到个人常态系统 v1，让用户知道最近状态和自己的长期记录相比是否值得留意。
- 已定边界：只和用户自己的历史比较，不做人群标准比较，不做医学诊断、不做异常判定、不做强因果。
- 数据基线：优先复用现有 logs、adult behavior events、review facts、StatsEngine 和目标历史；默认不新增 schema。

## 核心决策

- 0.2.8 的核心基调：把“短期发生了什么”推进到“和我自己的长期记录相比，这是否值得留意”。
- 产品定位优先叫“个人常态系统 v1”，长期趋势是其中的呈现方式。
- 默认基线使用 90 天；最近窗口使用 14 / 30 天。
- 第一层指标优先做硬度 / 晨勃、恢复 / 疲劳、睡眠 / 压力、性负荷。
- 第二层指标包括色情使用、满意度、运动和目标历史。
- 指标口径优先复用 `StatsEngine` 和 0.2.3 review facts；实现前必须逐项校准真实代码。
- 输出只分三类：仍在个人常态内、偏离个人常态但样本有限、数据不足只展示事实。
- 0.2.8 v1 用户可见 confidence 最高到 medium；不输出 high confidence 强结论。
- 个人基线只能来自本地历史数据，不能引入人群平均值、医学阈值或外部标准。
- “偏离”只表示偏离用户自己的记录范围，不表示异常、疾病或风险诊断。
- 所有趋势都必须带样本量、时间窗口、缺失数据提示和 confidence。
- 默认只做派生计算，不持久化 baseline cache；若需要 cache，必须先回到规划讨论。
- 0.2.8 派生结果只服务 UI，不进入 JSON backup、CSV 导出或 IndexedDB store。
- 0.2.8 不替代 0.2.3 复盘，而是把 0.2.3 的窗口事实延长到 60 / 90 / 180 天级别。

## 明确不做

- 医学异常检测。
- ED、成瘾、心理健康或关系问题诊断。
- 人群标准、年龄段标准或“正常男人应当如何”的比较。
- 预测未来硬度、性表现或恢复。
- 外部 AI、云端模型或联网分析。
- 伴侣评分 / 排名。
- 面向他人的报告或分享图。

## 下一步

后续实现建议按以下顺序：

1. 刀 1：现有统计、复盘数据和指标口径校准，先产真实代码校准结果。
2. 刀 2：个人基线派生模型。
3. 刀 3：个人常态工作台信息架构与界面。
4. 刀 4：偏离提示与记录缺口。
5. 刀 5：Safety / Privacy 审计。

如果真实代码中的 0.2.3 复盘事实或 0.2.5 目标历史尚未稳定落地，0.2.8 应缩小为只读趋势视图，不扩展 baseline 语义。

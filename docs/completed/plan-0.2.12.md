> 本文是 0.2.12 的短入口。详细讨论拆到专题文档，避免单个计划文件过长。

## 当前状态

- 状态：completed。
- 前置：0.2.8 个人常态；0.2.9 变化解释；0.2.10 观察计划；0.2.11 经验卡。
- 主方向：按月度 / 季度把个人常态变化、解释卡片、观察计划和经验卡整理成阶段回顾。
- 产品归属：0.2.12 是单独实现版本，但用户体验上作为 0.2.8 的只读月度回看聚合层。
- 核心基调：阶段回顾是个人本地复盘，不是健康报告、诊断报告或分享材料。
- 默认不新增 schema；优先从既有派生结果和历史记录生成只读回顾。

## 核心决策

- 0.2.12 回答：“这一阶段，我的记录和经验有什么值得回看？”
- 对外弱化为“本月回看”，不强调报告。
- 默认窗口为自然月；季度回顾作为可选扩展。
- 最低依赖是 0.2.8 个人常态；0.2.9 - 0.2.11 是增强章节。
- 回顾只聚合已存在的数据和派生结果，不新增用户输入。
- 章节包括个人常态、变化解释、观察计划、经验卡和记录缺口。
- 不输出健康评分、阶段评级、成功 / 失败判断或医学结论。
- 不做分享图、对外报告或医疗就诊报告。
- 章节缺失时隐藏，不展示“未完成模块”的空壳。

## 明确不做

- 年度总结。
- 社交分享。
- 医学报告。
- AI 自动总结。
- 健康评分或排名。
- 成就徽章。
- 未来预测。

## 开发刀序

1. 刀 1：代码校准与来源确认。
2. 刀 2：阶段回顾契约与章节构建。
3. 刀 3：回顾 UI 与导航。
4. 刀 4：记录缺口与限制说明。
5. 刀 5：Safety / Privacy 审计。

如果只有 0.2.8 落地，0.2.12 应缩小为基础月度个人常态回看；0.2.9 - 0.2.11 未落地时隐藏对应增强章节。

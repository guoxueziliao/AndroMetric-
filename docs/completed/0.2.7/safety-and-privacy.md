# 0.2.7 Safety / Privacy Rails

> 本文负责 0.2.7 关系上下文和伴侣资料的安全、隐私、文案和导出边界。

## 核心风险

关系上下文系统最容易滑向：

- 伴侣评分。
- 伴侣排名。
- 性表现归因。
- 关系诊断。
- 隐私备注外泄。
- 用系统标签评价伴侣身份、年龄、关系状态或性能力。
- 把用户记录的关系冲突或边界问题变成“对方的问题”。
- 用月经、排卵或窗口期催促、安排或合理化性生活。
- 把周期估算包装成排卵、怀孕或女性健康诊断。
- 把备孕 / 避孕目标变成压力、任务或打卡。

0.2.7 必须把这些风险前置为验收条件。

## 禁止文案

不得出现：

- “这个伴侣让你表现更好 / 更差。”
- “最佳伴侣。”
- “伴侣评分。”
- “匹配度。”
- “谁更适合你。”
- “分享给伴侣。”
- “用记录证明对方的问题。”
- “窗口期适合安排性生活。”
- “根据记录判断伴侣已经排卵 / 怀孕。”
- “为了备孕本周应增加频率。”

## 允许文案

可以出现：

- “本次记录包含关系上下文。”
- “本次记录包含沟通 / 边界信息。”
- “关系沟通目标有历史 check-in。”
- “伴侣反馈样本不足，只展示事实。”
- “这条历史记录尚未关联稳定伴侣 ID。”
- “私密备注默认不进入可读导出。”
- “周期估算仅作上下文，请以伴侣实际感受为准。”
- “性生活计划必须以沟通和同意为前提。”

## 隐私边界

默认本地展示。

默认不进入 CSV / 报告类导出：

- 伴侣 notes。
- 敏感点。
- 刺激偏好。
- 不喜欢 / 禁忌行为。
- 沟通记录全文。
- 关系冲突或私密反馈。
- 本次关系上下文原文。
- 周期不适、疼痛、出血、验孕、恢复或生殖健康备注全文。

JSON backup 仍应完整保留用户数据，因为它是迁移和恢复格式。

## 指标边界

可以统计：

- 是否有关联伴侣上下文。
- 是否有沟通类目标。
- 是否有 check-in。
- 是否存在 orphan partner id。
- 是否存在 legacy name 记录。
- 是否存在周期 / 生殖健康上下文。

不能统计：

- 伴侣排名。
- 伴侣得分。
- 不同伴侣下的性表现排行榜。
- 伴侣导致硬度 / 满意度变化的强因果。
- 多伴侣数量挑战。
- 排卵 / 怀孕确定性。
- “最佳性生活日期”。

## 审计要求

0.2.7 收口前必须审计：

- 伴侣详情页。
- 性行为记录中的关系上下文采集。
- 成人行为复盘关系上下文。
- 月经 / 生殖健康面板和周期关怀提示。
- 训练历史关系沟通目标。
- 导出提示和 CSV 字段。
- import preview / snapshot integrity warning 文案。
- 空状态、toast、confirm。

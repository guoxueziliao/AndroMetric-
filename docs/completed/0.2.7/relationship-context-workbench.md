# 0.2.7 关系上下文工作台

> 本文规划 0.2.7 的关系上下文呈现方式。伴侣详情只是入口之一，不是完整关系管理系统。

## 信息分组

工作台面向三类信息：

- 本次上下文：单次性行为或复盘中的沟通、边界、反馈和关系氛围。
- 长期上下文：来自伴侣资料的长期偏好、边界、沟通方式和私密备注。
- 周期关怀上下文：来自月经 / 生殖健康记录的经期、不适、备孕 / 避孕目标和恢复背景。
- 复盘上下文：被复盘 / 训练历史安全读取的中性摘要。

伴侣详情可以整理为几组长期上下文：

- 基础信息：显示名、关系类型、头像色、可选备注。
- 关系状态：稳定 / 约会 / 临时 / 其他，仅作为用户自定义上下文。
- 边界与偏好：喜欢 / 不喜欢 / 需要避免的行为。
- 沟通与反馈：事后反馈、沟通重点、需要下次注意的事项。
- 健康与恢复背景：与恢复、睡眠、压力、满意度有关的中性提示。
- 周期与性健康关怀：周期追踪开关、当前生殖目标、经期不适和保护措施沟通提示。
- 私密备注：只在伴侣详情中显示，不默认进入报告或 CSV。

系统生成标签应保持中性。用户自填内容可以保留成人表达，但系统不生成年龄、婚姻或能力评价式标签。

## 复盘接入

关系上下文进入复盘时，只能做中性解释：

- “本次记录包含关系上下文。”
- “本次记录包含周期 / 性健康关怀上下文。”
- “最近沟通类目标有 check-in。”
- “伴侣反馈样本不足，只展示事实。”
- “周期估算仅作上下文，请以伴侣实际感受为准。”
- “这条复盘包含 legacy 伴侣名，尚未关联 stable id。”

不得输出：

- “某个伴侣让你表现更好。”
- “某个伴侣导致表现变差。”
- “你和某个伴侣更匹配。”
- “这个伴侣排名更高。”
- “窗口期适合安排性生活。”
- “根据记录判断伴侣怀孕 / 排卵。”

## 训练历史接入

0.2.7 可以让 0.2.5 的目标历史读取关系沟通类目标：

- 目标 category 为关系沟通时，展示历史 check-in。
- 展示沟通是否持续、反馈是否有记录、缺口是否减少。
- 不把关系沟通转成伴侣评分。
- 不把满意度变化归因到某个伴侣。

## UI 入口

候选入口：

- 伴侣管理详情页。
- 性行为记录中的关系上下文区。
- 月经 / 生殖健康面板中的关怀提示。
- 复盘后补充上下文。
- 成人行为复盘中的关系上下文片段。
- 训练目标历史中的关系沟通目标详情。

不新增顶级“关系中心”导航。

## 空状态

空状态应鼓励记录上下文，而不是制造压力：

- “可以记录沟通重点，方便下次复盘。”
- “可以记录舒适度和需要照顾的事项，方便下次更好沟通。”
- “没有关系上下文时，复盘只显示事实数据。”
- “伴侣资料是本地私密数据，不会上传。”

不使用：

- “补全伴侣档案才能获得准确结论。”
- “记录越多越能判断谁更适合你。”
- “记录周期后系统会告诉你最佳性生活时间。”
- “分享给伴侣看看。”

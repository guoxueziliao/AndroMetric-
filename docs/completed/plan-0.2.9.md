- 核心基调：解释层只提出“可能一起出现的上下文”，不下因果结论。
- 默认不新增 schema，优先复用现有 logs、adult behavior events、review facts、training history、relationship context 和 personal normal result。

## 核心决策

- 0.2.9 回答：“最近变化附近，有哪些上下文值得一起看？”
- 0.2.9 不回答：“是什么导致了变化？”
- 输入优先来自 0.2.8 的 personal normal result。
- 对外不做独立“解释层工作台”，只作为 0.2.8 个人常态详情中的上下文解释能力。
- 第一层上下文：睡眠、压力、性负荷、恢复、色情使用、运动。
- 条件上下文：目标历史只在 0.2.5 落地后读取；关系 / 周期上下文只在 0.2.7 落地后读取。
- 输出形式是解释卡片，不是诊断报告、风险等级或训练计划。
- 所有卡片必须显示窗口、样本、限制说明和“仅供观察”的语气。
- 不新增外部 AI、云端模型、人群标准或医学阈值。

## 明确不做

- 强因果结论。
- 未来预测。
- 医学解释或诊断。
- 自动生成训练目标。
- 自动建议减少 / 增加性行为。
- 伴侣归因。
- 色情内容道德评价。
- 分享图或对外报告。

## 开发刀序

后续实现按以下顺序：

1. 刀 1：代码校准与依赖版本确认。
2. 刀 2：上下文窗口构建。
3. 刀 3：解释卡片引擎。
4. 刀 4：并入 0.2.8 详情页的 UI 与 Dashboard 轻提示。
5. 刀 5：Safety / Privacy 审计。

实现窗口必须先完成刀 1；若 0.2.8 尚未真实落地，0.2.9 应暂停实现。

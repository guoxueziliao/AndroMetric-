# 0.2.3 复盘中心与报告

> 本文从 0.2.3 入口拆出，保存成人行为复盘中心、周报/月报和 Markdown 导出方向。

## 已定复盘中心

0.2.3 选择：

> 做综合“成人行为复盘”入口，但首屏主指标围绕硬度 / 勃起质量和恢复。

原因：

- 0.2.3 不能变成单纯色情使用统计器。
- 男性健康主轴仍然是硬度、勃起质量、恢复、睡眠和行为负荷。
- 色情使用、自淫 / 自慰、性行为、射精、边缘控制等数据是解释健康状态的关键上下文。

### 入口定位

建议入口名称方向：

- 成人行为复盘
- 性健康复盘
- 行为与恢复
- 硬度与恢复

最终命名等 0.2.1 / 品牌语言确定后再定。

入口形态已定：

- 嵌入现有统计 / 洞察区域。
- 不新增顶级导航。
- 可以做成现有统计页内的 tab、section 或二级页面，具体形态按实现时现有路由和组件结构决定。
- Dashboard 只允许放轻入口或摘要提示，不承载完整复盘中心。

原因：

- 0.2.3 是“记录之后怎么看”的统计复盘能力，信息架构上接近 stats / insights。
- 新增顶级导航会过早把 0.2.3 做成独立产品区。
- Dashboard 是快速状态面板，不适合承载完整成人行为复盘和报告。

### 首屏主指标

首屏优先展示：

- 硬度 / 勃起质量趋势。
- 恢复状态。
- 近 7 / 14 / 30 天性活动负荷。
- 射精次数。
- 自淫 / 自慰频率。
- 色情使用时长 / 频率。
- 睡眠影响。
- 疲劳 / 满意度。

首屏不以色情内容类型、来源平台或偏好标签为主指标。这些可以进入二级视图。

首屏信息架构见 [`review-home-information-architecture.md`](./review-home-information-architecture.md)。

### 复盘结构

建议分层：

1. **总览**
   - 硬度、恢复、活动负荷、睡眠影响。

2. **行为链路**
   - Porn use → Masturbation / Sex → Ejaculation → Sleep → Next-day hardness。

3. **色情使用**
   - 时长、频率、动机、控制感、超时、事后状态。

4. **自淫 / 自慰**
   - 频率、射精、边缘控制、硬度、满意度、疲劳。

5. **性行为**
   - 性行为频率、硬度、射精、满意度、色情内容参与、恢复。

6. **洞察提示**
   - 带样本量和可信度的弱相关观察。

第一批弱相关观察见 [`weak-correlation-insights.md`](./weak-correlation-insights.md)。

实现顺序：

1. 先做事实时间线和窗口聚合。
2. 再做弱相关观察。
3. 最后做周报 / 月报和 Markdown 导出。

原因：

- 时间线和窗口聚合是所有洞察的事实基础。
- 先做相关性会让 UI 和文案容易过早承诺结论。
- 报告导出必须基于稳定的复盘结构。

事实时间线和窗口聚合契约见 [`review-input-and-timeline.md`](./review-input-and-timeline.md)。

### 表达边界

- 可以说“最近高色情使用时长的次日硬度均值更低，但样本少，仅供观察。”
- 不说“色情导致你硬度下降。”
- 可以说“近 14 天射精次数较上个窗口增加，疲劳自评也更高。”
- 不说“你射精过度。”
- 可以说“睡眠不足后的硬度记录偏低。”
- 不说“你有性功能障碍。”

## 已定周报 / 月报方向

0.2.3 做周报和月报，同时保留滚动窗口复盘。

定位：

- 周报 / 月报是本地复盘报告，不是分享内容。
- 报告允许成人、露骨、直白词，但表达仍是健康复盘和行为观察。
- 永远不做分享图，避免成人隐私数据外泄风险。

### 时间窗口

0.2.3 支持：

- 滚动 7 天。
- 滚动 14 天。
- 滚动 30 天。
- 自然周周报。
- 自然月月报。

默认：

- 首页复盘使用滚动 7 / 14 / 30 天。
- 报告入口提供自然周 / 自然月。

原因：

- 滚动窗口适合日常观察。
- 自然周 / 自然月适合阶段复盘和导出。

### 报告结构

周报 / 月报建议包含：

1. **本期摘要**
   - 记录天数。
   - 硬度 / 勃起质量均值与趋势。
   - 恢复状态。
   - 性活动负荷。

2. **成人行为摘要**
   - Porn use 次数 / 总时长。
   - Masturbation 次数。
   - Sex 次数。
   - 射精次数。
   - 边缘控制次数。

3. **硬度与恢复**
   - 晨间硬度。
   - 性行为 / 自慰中的硬度。
   - 疲劳。
   - 睡眠影响。

4. **可能关联**
   - 带样本量和可信度的弱相关观察。
   - 不输出强因果。

5. **样本不足提示**
   - 哪些指标只能展示事实。
   - 哪些结论暂时不能判断。

6. **本期记录缺口**
   - 缺少睡眠记录。
   - 缺少硬度记录。
   - 成人行为事件样本不足。
   - 关联事件未补全。

### Markdown 导出

0.2.3 允许周报 / 月报导出 Markdown。

模板结构见 [`report-markdown-template.md`](./report-markdown-template.md)。

规则：

- 本地生成。
- 不上传。
- 不做分享图。
- 文案允许成人、露骨、直白。
- 导出前必须明确提醒包含敏感成人健康数据。
- 文件名不应默认包含过于露骨内容，可使用产品名 + 日期 + report 类型。

导出内容可以包含：

- 摘要指标。
- 表格。
- 文本洞察。
- 样本量 / 可信度。
- 限制说明。

不包含：

- 原始备注全文，除非用户显式选择。
- 具体平台名，除非用户显式选择。
- 任何图片 / 视频 / 内容本体。

### 分享图

0.2.3 永远不做分享图。

原因：

- 成人行为、色情使用、硬度、射精、伴侣等数据高度敏感。
- 分享图容易误触传播。
- 与本地私密工具定位冲突。

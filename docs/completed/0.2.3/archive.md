# 0.2.3 候选能力归档

> 本文从 0.2.3 入口拆出，保存候选能力和已完成的早期判断问题。

## 候选能力

### 方向 A：成人行为复盘时间线

把 Porn use、Masturbation、Sex、Morning hardness、Sleep、Alcohol、Exercise 等事件放进同一时间线。

可能内容：

- 生理日时间线。
- 周复盘 / 月复盘。
- 事件链路展示：色情使用 → 自慰 → 射精 → 睡眠 → 次日硬度。
- 同一天多事件密度展示。
- 关联事件高亮。

### 方向 B：硬度与行为因素关联

围绕硬度和勃起质量，展示可能相关的行为因素。

可能因素：

- 睡眠时长 / 睡眠质量。
- 酒精 / 咖啡因。
- 运动。
- 压力 / 心情。
- 性行为频率。
- 自慰频率。
- 色情使用时长 / 强度。
- 射精次数。
- 边缘控制。
- 疲劳。

### 方向 C：色情使用复盘

围绕色情使用本身，做更清晰的趋势和自评复盘。

可能内容：

- 使用频率。
- 使用时长。
- 来源类型。
- 内容类型。
- 动机。
- 控制感。
- 超时次数。
- 使用后状态。
- 与自慰、射精、睡眠和次日硬度的关系。

### 方向 D：性活动负荷与恢复

将 Sex、Masturbation、Porn use、射精、边缘控制、疲劳和睡眠整合成恢复复盘。

可能内容：

- 近 7 / 14 / 30 天性活动负荷。
- 射精频率。
- 边缘控制频率。
- 疲劳趋势。
- 恢复窗口。
- 次日硬度变化。

### 方向 E：洞察可信度与样本量提示

0.2.3 必须避免“看起来很聪明但实际样本不足”的伪洞察。

可能内容：

- 样本量提示。
- 置信度 / 可信等级。
- “仅展示事实，不做结论”的状态。
- 最近数据 vs 长期数据区分。
- 异常点解释。

## 早期判断问题

以下问题已经转入 0.2.3 执行草案：

1. 0.2.3 允许新增轻量 adult behavior review engine。
2. 先做事实时间线和窗口聚合，再做弱相关洞察。
3. 首屏以硬度、勃起质量和恢复为中心，色情使用 / 性活动作为上下文。
4. 做滚动 7 / 14 / 30 天复盘、自然周周报和自然月月报。
5. 入口嵌入现有统计 / 洞察区域，不新增顶级导航。
6. 样本不足时只展示事实，不输出判断。
7. 禁止医学诊断、成瘾判定、强因果、道德化评价和羞辱语言。
8. 新增 adult behavior review engine，但与现有 StatsEngine 共享可复用 helper。

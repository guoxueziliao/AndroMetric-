# 0.2.4 入口、UI 与待定项

> 本文从 0.2.4 入口拆出，保存入口形态、轻目标 UI 细节、剩余待定和暂不做事项。

## 已定入口与 UI 形态

0.2.4 选择：

> 成人行为复盘作为主入口，Dashboard 做轻提示，不单独做 Training 页面。

原因：

- 0.2.4 的核心链路是“复盘之后怎么改进”，训练建议应该直接跟在成人行为复盘之后。
- Dashboard 适合展示当前目标、到期 check-in 和轻提示，不适合承载完整训练系统。
- 0.2.4 目标数量少，单独 Training 页面会显得过重，也会提前把轻目标做成完整计划系统。
- 后续如果目标、建议历史、训练周期明显增加，再考虑 0.2.5+ 独立 Training 页面。

### 主入口：成人行为复盘

成人行为复盘页 / 区块中加入：

- 本地规则建议。
- 1-3 个候选轻目标。
- 每个候选目标的触发原因。
- 样本量 / 可信度提示。
- “开始 / 忽略 / 稍后”操作。

主入口必须满足：

- 不自动创建目标。
- 不把建议命名为 AI。
- 不展示完整训练课程。
- 不打断复盘阅读。
- 不把候选目标做成强迫式任务。

### Dashboard 轻提示

Dashboard 只展示轻量状态：

- 当前 active 目标数量。
- 最近一个 active 目标。
- 即将到期的 check-in。
- 低恢复 / 高负荷时的恢复提示。
- 一条简短的“本周关注”。

Dashboard 不展示：

- 完整目标列表。
- 完整建议列表。
- 训练课程。
- 任务面板。
- 连续打卡。
- 性次数 / 射精次数 / 色情使用时长挑战。

### 不做独立 Training 页面

0.2.4 不新增独立 Training 页面。

不做原因：

- 当前版本重点是建立闭环，不是建立完整训练中心。
- 独立页面会诱导做更多目标、更多计划、更多管理状态。
- 训练建议来源于复盘，主入口离开复盘会削弱上下文。

后续只有在以下条件满足时，才重新讨论独立 Training 页面：

- 用户同时存在多个 active / paused / archived 目标。
- 需要查看目标历史。
- 需要跨周期比较 check-in。
- 需要训练计划或长期周期。
- Dashboard 轻提示和复盘入口承载不住。

## 已定轻目标与 Check-in UI 细节

0.2.4 选择：

> `TrainingGoal.description` 不在快速创建里显示，只在完整编辑里显示；`GoalCheckin.note` 不在快速 check-in 里显示，只在完整编辑里显示；`cycleFeeling` 非必填。

### TrainingGoal.description

0.2.4 支持 `TrainingGoal.description`，但快速创建不显示。

规则：

- 快速创建只显示默认标题、目标类型、周期和触发原因。
- 用户点击完整编辑时，才展示 description。
- description 用于用户补充目标背景，不参与规则建议。
- description 不能用于生成被 Safety Rails 禁止的目标。

原因：

- 手机 Chrome 是高频场景，快速创建必须短。
- 目标说明是补充信息，不应该挡住“开始 / 忽略 / 稍后”的主决策。

### GoalCheckin.note

0.2.4 支持 `GoalCheckin.note`，但快速 check-in 不显示。

规则：

- 快速 check-in 只显示状态选择：继续、暂停、完成、调整。
- 完整编辑才显示 note。
- note 用于记录本周期背景，不参与自动诊断。
- note 不用于生成伴侣评分、能力评分或强因果结论。

原因：

- 到期 check-in 应该是低摩擦动作。
- 备注适合事后补充，不适合作为必填项。

### cycleFeeling

`cycleFeeling` 表示 1-5 的“本周期感受”，但不是必填。

规则：

- 用户可以只选择状态，不填写本周期感受。
- 如果填写，展示文案使用“本周期感受”。
- 不叫评分，不叫表现评分，不叫能力评分。
- 不用于排名、惩罚或失败判断。

原因：

- 主观感受有价值，但不应把 check-in 变成考试。
- 用户在疲劳、低恢复、关系压力高时，仍应能快速完成回顾。

## 剩余待定

## 暂不做

- 完整训练计划系统。
- 完整关系管理系统。
- 技巧库和训练课程。
- 云端 AI / 外部大模型 API。
- 分享图。

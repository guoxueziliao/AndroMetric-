# 0.2.4 Safety Rails

> 本文从 0.2.4 入口拆出，保存目标、规则、文案和指标展示四层限制。

## 已定 Safety Rails

0.2.4 选择：

> 建立 Safety Rails：目标类型、建议规则、文案、指标展示四层限制。

Safety Rails 是 0.2.4 的开发验收条件，不是普通文案建议。任何目标、建议、卡片、指标、空状态、toast、confirm 或复盘文案，只要诱导更多性行为、更多射精、更高色情刺激、伴侣比较或忽视恢复，都不能进入版本。

### 第一层：目标类型限制

允许目标：

- 记录质量。
- 恢复。
- 硬度稳定。
- 性表现稳定性观察。
- 射精控制观察。
- 关系沟通。
- 伴侣反馈记录。

禁止目标：

- 性次数目标。
- 射精次数目标。
- 伴侣数量目标。
- 色情使用时长目标。
- 色情刺激强度目标。
- 做爱最长时间挑战。
- 更多约炮 / 更多伴侣目标。
- 忽视同意、安全、恢复的目标。

高负荷 / 低恢复时：

- 只能推荐恢复类目标。
- 只能推荐记录质量类目标。
- 可以提示暂停追求表现数据。
- 不能推荐表现提升、做爱时长、射精控制挑战或更多性行为相关目标。

### 第二层：建议规则限制

建议规则必须满足：

- 恢复优先。
- 样本不足时只提示继续记录，不下结论。
- 所有建议必须带触发原因。
- 所有趋势判断必须考虑样本量和可信度。
- 做爱时长、射精控制、性次数必须和满意度、硬度、恢复一起看。
- 色情使用相关建议只能表达为观察窗口，不能直接判定因果。
- 伴侣 / 关系相关建议只能表达为沟通、边界、反馈和上下文补充。

建议规则禁止：

- 医学诊断。
- 成瘾判定。
- 强因果结论。
- 自动判断某个伴侣导致表现好或差。
- 建议增加性次数、射精次数、伴侣数量或色情使用时长。
- 用羞辱、焦虑、竞争、征服感驱动用户。

### 第三层：文案限制

允许文案风格：

- 直白成人健康。
- 自我管理。
- 观察式。
- 可执行。
- 不道德化。
- 不羞辱。

允许例子：

- “近期恢复偏慢，建议优先记录睡眠和疲劳。”
- “样本不足，继续记录后再看趋势。”
- “射精控制需要和疲劳、满意度、恢复一起观察。”
- “伴侣反馈缺失时，满意度变化暂时不建议下结论。”

禁止文案风格：

- 挑逗化。
- 羞辱。
- 竞争。
- 征服感。
- 能力排名。
- 贩卖焦虑。
- 伪医学诊断。

禁止例子：

- “升级成更强男人。”
- “挑战更多次性爱。”
- “击败上一次的做爱时长。”
- “找出最适合你的伴侣。”
- “色情刺激越强越能训练能力。”
- “恢复慢说明能力差。”

### 第四层：指标展示限制

指标展示必须满足：

- 硬度和恢复优先于次数、时长、射精数量。
- 做爱时长不做最高纪录。
- 射精控制不做挑战榜。
- 性次数不做连续打卡。
- 色情使用时长不做累计成就。
- 伴侣维度不做排名、评分或胜负比较。
- 表现指标必须和满意度、硬度、恢复一起展示。
- 低恢复时，表现指标降级展示，恢复提示优先。

禁止展示：

- “最长做爱纪录。”
- “本月性次数挑战。”
- “射精次数统计成就。”
- “色情使用时长排行。”
- “最佳伴侣 / 最差伴侣。”
- “让你表现最好的伴侣。”

### 验收条件

0.2.4 每个相关刀必须检查：

- 是否引入了被禁止的目标类型。
- 是否存在诱导更多性行为、更多射精或更高色情刺激的建议。
- 是否把伴侣做成排名、评分或优劣判断。
- 是否在低恢复状态下继续强调表现提升。
- 是否出现羞辱、竞争、征服、能力排名或焦虑文案。
- 是否用强因果、诊断或成瘾判定解释数据。

只要任一项命中，不能合入 0.2.4。

# 0.2.4 刀 7 Safety Rails 审计

> 本文只规划刀 7：0.2.4 收口前的 Safety Rails 审计。它不新增产品能力。

## 状态

- 所属版本：0.2.4。
- 当前阶段：待刀 6 完成后执行。
- 前置：[`safety-rails.md`](./safety-rails.md)。
- 实现边界：目标、建议、文案、指标、空状态、toast、confirm、报告入口审计。

## 目标

刀 7 完成后，0.2.4 所有新增内容都必须通过四层限制：

- 目标类型。
- 建议规则。
- 文案。
- 指标展示。

任一命中禁止项，不能合入 0.2.4。

## 审计范围

必须审计：

- `TrainingGoal` category。
- `GoalCheckin` 文案。
- 规则建议。
- 候选轻目标。
- 训练区卡片。
- Dashboard 轻提示。
- 空状态。
- toast。
- confirm。
- 导入 warning。
- snapshot integrity 文案。
- 关系上下文文案。

## 禁止目标审计

不得出现：

- 性次数目标。
- 射精次数目标。
- 伴侣数量目标。
- 色情使用时长目标。
- 色情刺激强度目标。
- 做爱最长时间挑战。
- 更多约炮 / 更多伴侣目标。
- 忽视同意、安全、恢复的目标。

## 建议规则审计

不得出现：

- 医学诊断。
- 成瘾判定。
- 强因果结论。
- 自动判断某个伴侣导致表现好或差。
- 建议增加性次数、射精次数、伴侣数量或色情使用时长。
- 高负荷 / 低恢复时继续强调表现提升。

必须确认：

- 恢复优先。
- 样本不足只提示继续记录。
- 所有建议有触发原因。
- 所有趋势判断考虑样本量和可信度。
- 色情使用相关建议只表达观察窗口。
- 伴侣 / 关系建议只表达沟通、边界、反馈和上下文补充。

## 文案审计

不得出现：

- 羞辱。
- 焦虑推动。
- 竞争。
- 征服感。
- 能力排名。
- 挑逗化建议。
- 伪医学诊断。
- 道德化成人偏好评价。

禁止例子：

- “升级成更强男人。”
- “挑战更多次性爱。”
- “击败上一次的做爱时长。”
- “找出最适合你的伴侣。”
- “色情刺激越强越能训练能力。”
- “恢复慢说明能力差。”

## 指标展示审计

不得出现：

- 最高做爱纪录。
- 性次数连续打卡。
- 射精次数成就。
- 色情使用时长累计成就。
- 伴侣评分。
- 伴侣排名。
- 最佳 / 最差伴侣。

必须确认：

- 硬度和恢复优先于次数、时长、射精数量。
- 表现指标和满意度、硬度、恢复一起展示。
- 低恢复时，恢复提示优先。
- 样本不足时，不展示判断。

## 导入与数据审计

必须确认：

- 禁用 category 不能进入 active 数据。
- 非 7 / 14 周期不能创建目标。
- unknown status 不能自动改成 active。
- import warning 不带羞辱或诊断语气。
- orphan check-in 作为完整性提示，不责备用户。

## 验收输出

刀 7 完成时应在 PR 或实现记录中列出：

- 审计范围。
- 发现的问题。
- 修正方式。
- 剩余风险。
- 验证命令。

## 验证命令

刀 7 完成时至少跑：

```bash
npm run test
npm run typecheck
npm run build
git diff --check
```

## 非目标

- 不新增目标类型。
- 不新增建议规则。
- 不新增 UI 入口。
- 不重写 0.2.4 范围。
- 不提前做 0.2.5 训练中心。

## 版本收口

刀 7 通过后，0.2.4 才能进入版本收口：

- 更新 CHANGELOG。
- 更新 roadmap。
- 移动 planned / completed 状态。
- 确认 0.2.4 没有越界到完整训练系统。

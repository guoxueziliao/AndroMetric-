# 0.2.4 性表现训练与关系维度

> 本文从 0.2.4 入口拆出，保存性表现训练、关系与伴侣维度的已定边界。

## 已定性表现训练维度

0.2.4 选择：

> 首批覆盖硬度稳定、做爱时长、射精控制观察、满意度、事后恢复，但展示优先级不同。

这意味着 0.2.4 不是只做单一硬度训练，也不是直接做完整性技巧课程。首批目标是把做爱表现中最核心、最可记录、最能和既有健康数据联动的维度接起来。

### 展示优先级

#### 1. 硬度稳定

硬度稳定是 0.2.4 性表现训练的第一优先级。

关注：

- 做爱过程中硬度是否稳定。
- 是否出现中途变软、需要暂停、需要更强刺激才能维持。
- 硬度波动与睡眠、疲劳、酒精、压力、色情使用、自慰、射精频率之间的关系。
- 与 0.2.3 的硬度 / 勃起质量复盘打通。

表达方向：

- “近期硬度稳定性较低，优先观察睡眠、疲劳和色情使用后的表现变化。”
- “样本不足，继续记录 3 次性行为后再判断硬度趋势。”

不表达：

- “硬度越高越好。”
- “挑战更强硬度。”
- “某个伴侣导致你表现差。”

#### 2. 事后恢复

事后恢复是第二优先级，用来防止训练系统诱导透支。

关注：

- 性行为后的疲劳。
- 次日晨勃。
- 次日精神状态。
- 睡眠质量。
- 连续性行为 / 自慰 / 色情使用后的恢复压力。
- 高负荷后是否需要暂停或降低强度。

表达方向：

- “高负荷后恢复偏慢，本周建议优先观察睡眠和疲劳。”
- “连续记录显示事后疲劳较高，建议暂停追求表现数据，先看恢复。”

不表达：

- “继续挑战更多次。”
- “恢复越快说明能力越强。”
- “低恢复就是失败。”

#### 3. 射精控制观察

射精控制进入首批，但定位为观察，不做挑战。

关注：

- 射精时间。
- 是否过快或过久。
- 主观控制感。
- 边缘控制是否使用。
- 边缘控制后的疲劳、硬度、满意度和恢复。
- 与自慰、色情使用、压力、睡眠的关系。

表达方向：

- “射精控制波动较大，建议记录控制感、疲劳和恢复，而不是单看时长。”
- “边缘控制后疲劳偏高，建议观察恢复窗口。”

不表达：

- “挑战更久。”
- “坚持越久越好。”
- “提高射精次数。”
- “训练成某种固定表现标准。”

#### 4. 做爱时长

做爱时长进入首批，但不作为越长越好的目标。

关注：

- 时长稳定性。
- 是否明显短于自身常态。
- 是否明显长到影响疲劳或满意度。
- 时长与硬度、射精控制、满意度、恢复的匹配。

表达方向：

- “本阶段更适合观察时长稳定性，而不是追求更长。”
- “时长变化需要和满意度、硬度、恢复一起看。”

不表达：

- “越久越好。”
- “挑战最长做爱时间。”
- “把时长作为核心能力排名。”

#### 5. 满意度

满意度进入首批，作为表现质量和关系体验的补充指标。

关注：

- 自己的主观满意度。
- 伴侣反馈。
- 关系氛围。
- 是否有沟通、边界、偏好和事后反馈。
- 满意度与硬度、时长、射精控制、恢复之间的关系。

表达方向：

- “满意度需要和关系氛围、沟通、恢复一起看。”
- “本次表现数据正常，但满意度较低，建议记录沟通和偏好因素。”

不表达：

- 伴侣评分。
- 伴侣排名。
- “谁让你表现更好 / 更差。”
- 对关系形态做道德判断。

### 首批不做

0.2.4 首批不做：

- 完整性技巧库。
- 系统训练课程。
- 固定周期训练营。
- 以时长、次数、射精数量为目标的挑战。
- 面向伴侣的评分系统。
- 医学治疗建议。

原因：

- 当前版本更需要打通记录、建议、轻目标和回顾闭环。
- 技巧库和训练课程内容量大，容易偏离本地健康管理主线。
- 性表现训练必须优先避免诱导透支、比较和风险行为。

## 已定关系与伴侣维度

0.2.4 选择：

> 关系与伴侣维度进入 0.2.4 主线，但只做“关系上下文 + 沟通/反馈 + 表现影响观察”，不做完整关系管理系统。

这个决定的核心是：性表现不是单纯身体数据。硬度、做爱时长、射精控制、满意度和事后恢复都会受到关系氛围、沟通、边界、偏好、熟悉度、压力和伴侣反馈影响。0.2.4 如果完全不处理关系上下文，训练建议会缺少重要背景。

但 0.2.4 也不能扩张成完整关系管理产品。本版本只把关系维度接入性健康、性表现和养成闭环。

### 进入主线的内容

0.2.4 可以做：

- 性行为记录关联关系 / 伴侣上下文。
- 多伴侣、多性别伴侣、多关系形态的中性表达。
- 性表现复盘中展示关系上下文。
- 满意度拆成自我满意度、伴侣反馈、沟通质量等轻量字段。
- 训练目标支持“关系沟通”类目标。
- 建议规则可以提示沟通、边界、偏好和事后反馈，但不能下判断。

### 表现影响观察

0.2.4 可以观察：

- 不同关系上下文下的硬度稳定性。
- 沟通质量与满意度之间的关系。
- 伴侣反馈与自我满意度是否一致。
- 压力、紧张、关系氛围与射精控制、硬度波动之间的关系。
- 事后反馈与恢复状态之间的关系。

表达方式必须是观察式：

- “在沟通记录更完整的性行为中，满意度样本更稳定。”
- “近期关系氛围备注较少，满意度变化暂时无法解释。”
- “伴侣反馈和自我满意度不一致时，建议补充沟通记录。”

不允许表达成强因果或评判：

- “某个伴侣导致你表现变差。”
- “某类伴侣更适合你。”
- “这个关系不健康。”
- “你应该更换伴侣。”

### 多伴侣与多性别伴侣表达

0.2.4 必须避免默认异性恋、单偶、一男一女关系。

文案和字段命名应支持：

- 单一伴侣。
- 多伴侣。
- 多性别伴侣。
- 固定关系。
- 非固定关系。
- 开放关系 / 多偶关系。
- 未标记关系。

推荐表达：

- “伴侣 / 关系上下文”
- “本次关系氛围”
- “伴侣反馈”
- “沟通与边界”

避免表达：

- “女友 / 老婆 / 女性伴侣”作为默认。
- “正经关系 / 不正经关系”。
- “主伴侣更好 / 临时伴侣更差”。
- 对关系形态做道德评价。

### 0.2.4 不做完整关系管理系统

0.2.4 不做：

- 完整伴侣档案系统重做。
- 关系阶段管理。
- 关系协议管理。
- 关系事件时间线。
- 冲突 / 修复记录系统。
- 伴侣偏好资料库。
- 伴侣评分。
- 伴侣排名。
- 伴侣优劣判断。
- 自动判断“适不适合某伴侣”。

这些内容可以作为 0.2.5+ 的候选方向，但不进入 0.2.4 的开发范围。

### 与轻目标的关系

关系维度可以进入轻目标，但只能是健康、沟通和复盘导向。

允许目标：

- “本周补充 2 次事后反馈。”
- “记录关系氛围和沟通质量。”
- “观察满意度和沟通记录是否一致。”
- “记录边界、偏好或不适点。”

不允许目标：

- “尝试更多伴侣。”
- “比较不同伴侣表现。”
- “找出让你表现最好的伴侣。”
- “提高征服感。”
- “用伴侣反馈排名关系质量。”

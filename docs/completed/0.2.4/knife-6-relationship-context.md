# 0.2.4 刀 6 关系上下文接入

> 本文只规划刀 6：把关系上下文接入性表现复盘、训练建议和轻目标。它不重做完整关系管理系统。

## 状态

- 所属版本：0.2.4。
- 当前阶段：待刀 5 完成后执行。
- 前置：[`performance-and-relationship.md`](./performance-and-relationship.md)。
- 实现边界：关系上下文、沟通 / 反馈、表现影响观察。

## 目标

刀 6 完成后，0.2.4 可以在训练建议和表现复盘中使用关系上下文，但不评价伴侣、不排名伴侣、不判断关系好坏。

允许：

- 关系上下文进入复盘和建议。
- 沟通、边界、偏好和事后反馈进入建议触发条件。
- 关系沟通类轻目标。
- 多伴侣、多性别伴侣、多关系形态的中性表达。

禁止：

- 伴侣评分。
- 伴侣排名。
- 伴侣优劣判断。
- 判断某个伴侣导致表现好或差。
- 完整关系管理系统。

## 输入边界

允许输入：

- 性行为记录中的关系 / 伴侣上下文。
- 沟通记录是否存在。
- 伴侣反馈是否存在。
- 关系氛围结构化字段。
- 满意度、硬度、恢复、疲劳等结构化指标。

不允许输入：

- 伴侣私密备注全文用于自动判断。
- 伴侣排名数据。
- 伴侣评分数据。
- 关系审判或道德标签。

## 建议规则

允许建议：

- “伴侣反馈记录较少，可以选择一个关系沟通观察目标。”
- “满意度变化暂时缺少关系上下文，建议补充沟通或边界记录。”
- “伴侣反馈和自我满意度不一致时，建议补充事后反馈。”

禁止建议：

- “某个伴侣导致你表现变差。”
- “某类伴侣更适合你。”
- “这个关系不健康。”
- “你应该更换伴侣。”
- “找出最适合你的伴侣。”

## 轻目标边界

允许 category：

- `relationship_communication`

允许目标方向：

- 记录关系氛围。
- 补充沟通和边界记录。
- 补充事后反馈。
- 观察满意度和沟通记录是否一致。

禁止目标方向：

- 比较不同伴侣。
- 伴侣排名。
- 伴侣评分。
- 增加伴侣数量。
- 提高征服感。

## 文案要求

必须支持：

- 单一伴侣。
- 多伴侣。
- 多性别伴侣。
- 固定关系。
- 非固定关系。
- 开放关系 / 多偶关系。
- 未标记关系。

推荐表达：

- “伴侣 / 关系上下文”
- “本次关系氛围”
- “伴侣反馈”
- “沟通与边界”

避免表达：

- “女友 / 老婆 / 女性伴侣”作为默认。
- “正经关系 / 不正经关系”。
- “主伴侣更好 / 临时伴侣更差”。
- 对关系形态做道德评价。

## 测试边界

必须覆盖：

- 关系沟通建议不评价伴侣。
- 多关系形态文案中性。
- 缺少伴侣反馈时只提示缺口。
- 不出现伴侣评分、排名或优劣判断。
- 不自动判断某个伴侣导致表现变化。
- `relationship_communication` 目标通过 Safety Rails。

## 非目标

- 不重做伴侣档案系统。
- 不做关系阶段管理。
- 不做关系协议管理。
- 不做冲突 / 修复记录系统。
- 不做伴侣偏好资料库。
- 不做伴侣评分或排名。

## 交接给刀 7

刀 7 必须审计所有关系上下文文案、目标和指标，确认没有伴侣评价、排名、优劣判断或关系道德化。

# 0.2.5 用户反馈接入清单

> 本文记录 0.2.4 完成后暴露、且会影响 0.2.5 开发判断的体验问题。它不是新增大范围需求清单。

## 进入 0.2.5 的问题

### 复盘入口差异不清

用户反馈：状态、历史统计、复盘里的 7 天、14 天、30 天、周报、月报点击后看不出区别。

0.2.5 处理原则：

- 刀 1 校准当前入口和页面真实差异。
- 刀 4 做入口 UI 时必须避免 Training Center 继续增加同类混乱。
- 复盘 / 统计 / 目标历史入口必须有清晰标题、时间范围和内容差异。

### 训练建议重复开始

用户反馈：训练建议下方有“开始 / 忽略”，连续点击 5 次“开始”出现 5 个内容，不清楚是什么意思。

0.2.5 处理原则：

- 刀 1 必须确认 0.2.4 是否允许同一建议重复创建多个 goal。
- 刀 2 目标历史工作台必须能解释这些重复目标，并提供安全的去重 / 归档入口。
- 若需要从源头禁止重复创建，应回到 0.2.4 修复，不在 0.2.5 用历史工作台掩盖。

### 报告入口误解

用户反馈：报告部分以为是直接查看，结果是导出。

0.2.5 处理原则：

- 刀 4 入口 UI 不能把“查看”和“导出”混在同一个预期里。
- 如果是查看，应进入页面；如果是导出，按钮文案必须明确写导出。
- 0.2.12 的“本月回看”也要继承这个边界，避免被理解成导出报告。

## 不直接进入 0.2.5 的问题

### 备份状态不可信

用户反馈：我的 - 设置与数据界面一直提示未备份，但已经备份。

归属：0.2.6 数据安全与长期数据承诺。

原因：这是备份状态、写后读确认和数据安全反馈问题，不应塞进 0.2.5 目标历史。

### 色情使用入口缺失

用户反馈：色情使用没有入口，只能从日期表单进入；生活里面还有看片，需要修改。

归属：后续入口信息架构反馈，优先作为成人行为 / 复盘 / 个人常态主线的入口债处理。

原因：此前已决定性健康 / 色情使用增强不独立成新版本；但入口缺失是有效体验债，需要在后续入口整理中跟踪。

### 补剂模型写死

用户反馈：补剂是直接写死的；实际补剂可能集中一起，然后按照周期来。

归属：后续数据模型候选，暂不进入 0.2.5。

原因：补剂周期模型可能涉及 schema / 记录模型变化，不属于目标历史工作台的默认复用范围。

## 0.2.5 开发底线

- 0.2.5 可以修正目标历史、入口解释和重复目标可理解性。
- 0.2.5 不替 0.2.6 修备份状态。
- 0.2.5 不新增色情使用独立版本。
- 0.2.5 不新增补剂周期 schema。

# 0.2.5 刀 2 目标历史工作台

> 本文只规划刀 2：基于 0.2.4 的目标和 check-in 数据，实现可回看的目标历史工作台。

## 状态

- 所属版本：0.2.5。
- 当前阶段：待刀 1 完成后执行。
- 前置：[`knife-1-code-and-data-calibration.md`](./knife-1-code-and-data-calibration.md)。
- 实现边界：目标列表、详情、筛选、归档 / 恢复和空状态。

## 目标

刀 2 完成后，用户可以查看自己长期关注过的轻目标，并理解每个目标的状态和 check-in 历史。

必须完成：

- active / paused / completed / archived 目标列表。
- 按 status 筛选。
- 按 category 筛选。
- 按 startDate / updatedAt 排序。
- 最近 check-in 摘要。
- 目标详情中的 check-in 历史。
- archived 目标保留历史。
- orphan check-in 完整性提示。
- 重复创建的同源目标可识别、可解释，并可安全归档。

## 信息结构

目标历史工作台建议包含：

1. 当前 active 目标。
2. 到期或最近 check-in。
3. 目标历史列表。
4. status / category 筛选。
5. 目标详情。
6. 缺失数据和完整性提示。

移动端优先展示：

1. 到期 check-in。
2. 当前 active 目标。
3. 最近完成或暂停的目标。
4. 筛选入口。
5. 历史详情。

## 操作边界

允许：

- 暂停 active 目标。
- 恢复 paused 目标。
- 归档 paused / completed 目标。
- 谨慎恢复 archived 目标。
- 查看历史 check-in。

默认不做：

- 删除目标。
- 批量归档。
- 自动完成目标。
- 自动创建新目标。
- 编辑历史 check-in 的业务语义。

## 空状态

没有目标时：

- 展示继续从复盘建议创建轻目标。
- 不展示空任务中心。
- 不催促用户开始训练。

没有 check-in 时：

- 展示暂无回顾记录。
- 不判断用户失败。

存在 orphan check-in 时：

- 展示完整性提示。
- 不删除历史。
- 不责备用户。

## 禁止展示

不得出现：

- 能力分。
- 失败次数。
- 连续打卡。
- 排行榜。
- 伴侣评分 / 排名。
- 性次数 / 射精次数 / 色情使用时长挑战。

## 验收

- 用户能按 status 查看目标。
- 用户能按 category 筛选目标。
- 用户能查看单个目标的 check-in 历史。
- archived 目标仍保留 check-in。
- orphan check-in 被提示但不丢失。
- 重复点击“开始”产生的多个目标不会变成不可理解的列表噪音。
- 没有任何评分、排名、打卡压力或挑战文案。

## 交接给刀 3

刀 2 结束后，刀 3 可以基于目标历史和 check-in 生成跨周期反馈，但仍只能生成事实、稳定性和缺口提示。

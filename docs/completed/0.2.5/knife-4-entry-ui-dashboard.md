# 0.2.5 刀 4 入口 UI 与 Dashboard

> 本文只规划刀 4：把目标历史工作台接入现有复盘 / 统计体验，并保持 Dashboard 轻量。

## 状态

- 所属版本：0.2.5。
- 当前阶段：待刀 3 完成后执行。
- 前置：[`knife-3-cross-cycle-feedback.md`](./knife-3-cross-cycle-feedback.md)。
- 实现边界：二级入口、工作台页面结构、Dashboard 轻入口和移动端信息密度。

## 入口结论

0.2.5 不新增顶级 Training 导航。

默认入口：

- 成人行为复盘内二级入口。
- Stats / Insights 内 tab。
- Dashboard 轻入口。

实现窗口可根据真实代码选择其中一个主入口，但不能新增独立训练中心顶级导航。

入口文案必须区分：

- 查看类入口：进入页面查看状态、历史、复盘或目标历史。
- 导出类入口：明确写“导出”，不能只写“报告”。
- 时间范围入口：7 天、14 天、30 天、周报、月报必须让用户看出范围和内容差异。

## 页面结构

目标历史工作台建议包含：

1. 当前目标。
2. 到期 check-in。
3. 目标历史。
4. 周期摘要。
5. 缺失数据 / 完整性提示。

页面不包含：

- 训练课程。
- 每日任务。
- 打卡日历。
- 排行榜。
- 成就墙。

## Dashboard

Dashboard 允许展示：

- 最近 active goal。
- 即将到期 check-in。
- 本周关注。
- 低恢复提示。
- 进入目标历史工作台的轻入口。

Dashboard 禁止展示：

- 完整目标列表。
- 完整目标历史。
- 课程入口。
- 连续打卡。
- 成就进度。

## 移动端优先级

手机 Chrome 优先：

1. 到期 check-in。
2. 当前 active goal。
3. 本周关注。
4. 历史摘要。
5. 筛选和归档。

避免：

- 首屏塞满历史图表。
- 目标卡片过多。
- 长文案压迫操作。
- 打卡式 UI。

## 验收

- 目标历史工作台可以从复盘 / 统计区域进入。
- Dashboard 只有轻提示和轻入口。
- 查看和导出不会被同一个“报告”文案混淆。
- 7 / 14 / 30 天、周报、月报入口有明确差异。
- 移动端首屏不被历史和筛选挤满。
- 不出现顶级 Training 导航。
- 不出现课程中心、任务中心或打卡中心。

## 交接给刀 5

刀 4 结束后，刀 5 对目标历史、跨周期反馈、Dashboard、导出提示和文案做 Safety / Privacy 审计。

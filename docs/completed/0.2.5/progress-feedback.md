# 0.2.5 长期进度反馈

> 本文负责跨周期摘要、趋势表达、稳定性反馈和样本边界。

## 目标

长期进度反馈不是评分系统，而是帮助用户回看：

- 记录是否更完整。
- 恢复是否更稳定。
- 硬度波动是否更清楚。
- 关系沟通记录是否更持续。
- 哪些目标值得继续。

## 允许的反馈

允许：

- “近 4 个周期里，你更常选择恢复相关目标。”
- “硬度稳定目标有 3 次 check-in，样本仍有限。”
- “记录质量目标完成后，后续复盘缺口减少。”
- “关系沟通目标有 check-in，但伴侣反馈样本仍少。”
- “本阶段更适合继续记录，而不是判断趋势。”

## 禁止的反馈

禁止：

- “你的能力提升了 X 分。”
- “你训练失败。”
- “你比之前更强。”
- “某个伴侣让你表现更好。”
- “色情使用导致你变差。”
- “你应该增加性次数 / 射精次数。”

## 指标来源

允许来源：

- `TrainingGoal` 状态。
- `GoalCheckin` 状态。
- `cycleFeeling` 分布，但不命名为评分。
- 0.2.3 review facts。
- 0.2.4 建议维度和目标 category。

不允许来源：

- notes 全文自动判断。
- 伴侣私密备注全文自动判断。
- 色情内容本体。
- 云端模型输出。

## 展示方式

建议展示：

- 周期摘要。
- 目标分类分布。
- check-in 完成情况。
- 缺失数据提示。
- 稳定性趋势。

避免展示：

- 分数。
- 排名。
- 失败率。
- 连续打卡。
- 最高纪录。

## 样本边界

样本不足时：

- 只展示事实。
- 提示继续记录。
- 不输出趋势判断。
- 不生成表现结论。

长期反馈必须继续继承 0.2.3 / 0.2.4 的 confidence 和 limitation 规则。

# 0.2.11 刀 5 Safety / Privacy 审计

> 本文定义 0.2.11 收口前的安全、隐私和文案审计。

## 审计范围

必须检查：

- 保存入口。
- 编辑确认。
- 经验卡列表。
- 经验卡详情。
- 空状态。
- 导出相关文案。

## 禁止词

不得出现：

- 已经证明。
- 规律成立。
- 原因是。
- 以后应该。
- 诊断结果。
- 伴侣导致。

这些词可以在开发文档禁止项中出现，不能出现在用户 UI。

## 隐私检查

必须确认：

- 不展示 notes 全文。
- 不展示色情标题、平台名或 URL。
- 不展示伴侣私密备注全文。
- 不新增分享入口。
- 不新增可读导出。

## 验证底线

实现完成后至少跑：

```bash
npm run test
npm run typecheck
git diff --check
```

触达 UI 时也跑：

```bash
npm run build
```

# 0.2.13 每日记录接入

## 当前问题

每日记录里的补剂来自固定列表：

- 维生素D。
- 锌。
- 镁。
- 鱼油。
- 辅酶Q10。
- 益生菌。

这让补剂像一次性标签，而不是长期项目。

## 新入口

每日记录健康区应改为：

- 今日健康项目。
- 计划内项目列表。
- 完成 / 跳过 / 备注。
- 旧补剂记录的兼容展示。
- 管理健康项目入口。

## 入口位置

- 主入口：每日记录健康区。
- 次入口：Dashboard / 首页只显示轻提示。
- 不放入“我的 / 设置”。
- 不新增顶级导航。

## 用户动作

用户在每日记录里可以：

- 标记今天已完成。
- 标记今天跳过。
- 临时记录备注。
- 跳转到项目详情。
- 看到旧补剂事实，但不把它误解成计划项。

用户不应该在每日记录里：

- 维护完整项目档案。
- 设置复杂周期。
- 进行剂量安全判断。
- 创建用药计划。

## 空状态

如果没有 active 项目：

- 不显示写死补剂按钮。
- 显示轻入口：“管理健康项目”。
- 不制造用户必须吃补剂的压力。

## Dashboard 轻提示

Dashboard 只展示今日状态：

- 今天待记录几个健康项目。
- 今天已完成几个健康项目。
- 不展示趋势图。
- 不展示补剂备注全文。
- 不展示连续打卡、评分或效果分析。

## 旧记录展示

旧 `supplements` 记录：

- 可以在当天历史里展示。
- 不自动变成 active 项目。
- 不计入计划执行率。
- 不用于补剂推荐。
- 导入旧数据后仍应保留在当天记录里。

# 0.2.13 周期计划契约

## 支持周期

第一版支持：

- 每日。
- 隔日。
- 每周若干天。
- 连续 N 天。
- 开始日期。
- 可选结束日期。
- 暂停期。
- 暂停 / 恢复。
- 手动结束。

默认周期：

- 新补剂项目默认每日。
- 新健康习惯默认每日或每周若干天，由用户选择。
- 不根据项目类型自动推断周期。

服用时间：

- 第一版只做时间标签。
- 不做精确闹钟。
- 不做后台提醒。
- 一个项目可以有多个时间标签。

暂不支持：

- 复杂轮班周期。
- 每月第几天。
- 按月第几个星期几。
- 按症状自动调整。
- 和外部日历同步。
- 后台提醒通知。
- 根据补剂类型自动推荐周期。

## 计划状态

项目状态：

- `active`
- `paused`
- `completed`
- `archived`

每日执行状态：

- `due`
- `done`
- `skipped`
- `not_due`

## 生成规则

- 每日记录页面只显示当天应执行的 active 项目。
- paused 项目不生成 due 项。
- completed / archived 项目不生成 due 项。
- 用户可以对当天项目标记完成或跳过。
- 跳过不等于失败，只是事实记录。
- 同一天重复生成必须去重，同一 project 只出现一个 due 项。
- 系统只根据用户设置生成“今天是否应执行”，不判断用户该不该执行。

## 结束和暂停

- 暂停必须保留历史执行记录。
- 结束后不再生成新 due 项。
- 归档只影响列表展示，不删除历史。
- 删除项目需要单独确认，优先考虑归档。

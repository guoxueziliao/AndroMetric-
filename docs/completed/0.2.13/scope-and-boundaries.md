# 0.2.13 范围与边界

## 核心定位

0.2.13 把“今天勾了哪些补剂”升级为“我正在管理哪些健康项目，以及这些项目今天是否执行”。

健康项目不是医疗方案。它只承载用户自己定义的补剂和非医疗健康习惯。

第一版只支持六类：

- 补剂。
- 晒太阳。
- 拉伸。
- 康复。
- 作息干预。
- 其他健康习惯。

## 做什么

- 健康项目档案：名称、类型、目标、备注、状态。
- 补剂计划：剂量、单位、服用时间、周期、开始 / 结束 / 暂停。
- 健康习惯计划：晒太阳、拉伸、康复、作息干预和其他健康习惯。
- 每日执行记录：完成、跳过、备注。
- 历史回看：执行事实、持续情况、记录缺口。
- 旧每日补剂记录的兼容读取。
- 不把旧补剂事实自动迁移成长期计划。

## 历史回看范围

- 项目详情页必须展示执行历史。
- 项目详情页可以展示最近记录、近 7 / 30 天完成次数和跳过次数。
- Dashboard 只提示今天待记录和已完成数量。
- 0.2.12 阶段回顾后续只读取概况和记录缺口。
- 不展示补剂备注全文。
- 不做连续打卡、评分、排名或效果分析。

## 不做什么

- 药物计划。
- 医嘱提醒。
- 疾病治疗方案。
- 自动推荐补剂。
- 剂量安全判断。
- 备孕成功率、性表现或健康结果的强因果判断。
- 成就、连续打卡、排名、评分或补剂效果分析。
- 医学诊断或替代医生意见。

## 用药边界

用药只保留为每日健康事实或备注：

- 可以记录“今天吃了某药”。
- 不生成用药计划。
- 不提醒服药。
- 不判断剂量是否安全。
- 不评价疗效。

## 与后续版本关系

- 0.2.8 个人常态可以读取健康项目执行事实，但不能把它当成诊断依据。
- 0.2.10 观察计划可以引用健康项目作为观察背景，但不能自动建议补剂。
- 0.2.12 阶段回顾可以展示执行概况和记录缺口。

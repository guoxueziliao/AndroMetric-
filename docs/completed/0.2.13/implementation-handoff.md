# 0.2.13 实现交接摘要

> 本文是后续进入实现窗口前的短交接单。当前是执行草案 / 待实现。

## 版本范围

0.2.13 做健康项目与补剂周期系统 v1：

- 补剂作为第一批健康项目。
- 有周期计划的健康习惯可以进入第一版。
- 用药不进入周期计划系统。
- 每日记录从写死补剂列表改为计划生成的待记录项。

## 推荐数据流

```text
HealthProject
  -> HealthProjectPlan
  -> today due items
  -> HealthProjectLog
  -> daily log / dashboard / later review
```

## 实现前必须校准

- 当前 `DailyLog.supplements` 的类型和导入导出覆盖。
- 当前每日记录健康区的 UI 入口。
- 当前 reproductive readiness 是否读取 `supplements`。
- 0.2.6 数据安全实现后，JSON backup / integrity 的扩展方式。
- 是否已有 0.2.6 import preview 和 snapshot integrity 扩展点可复用。

## 推荐实现顺序

1. 代码校准。
2. 数据模型与 migration。
3. 项目管理。
4. 每日记录接入。
5. 数据安全接入。
6. Safety / Privacy 审计。

## 必须停下来重谈

- 需要把用药做成计划。
- 需要自动推荐补剂。
- 需要提醒通知。
- 需要剂量安全判断。
- 需要把旧补剂记录自动推断成长期计划。
- 需要外部 API、账号或云同步。

## 验收底线

- 用户能创建、暂停、结束和归档补剂项目。
- 补剂项目支持可选剂量、单位、服用时间标签和备注。
- 用户能设置简单周期。
- 周期支持每日、隔日、每周若干天、连续 N 天、暂停 / 恢复和手动结束。
- 不支持每月复杂周期、轮班周期、提醒通知、外部日历或自动推荐周期。
- 每日记录只显示当天应执行项目。
- 每日记录健康区提供管理健康项目入口。
- Dashboard 只提供轻提示，不承载完整管理。
- 项目详情提供执行历史、近 7 / 30 天完成次数和跳过次数。
- 不做连续打卡、评分、排名或补剂效果分析。
- 用户能完成或跳过当天项目。
- 旧 `supplements` 记录继续可读。
- 旧 `supplements` 不被自动迁移成长期计划。
- import preview 区分 legacy supplements 和新健康项目数据。
- JSON backup 覆盖新数据。
- 不出现医疗建议、剂量安全判断或服药提醒。

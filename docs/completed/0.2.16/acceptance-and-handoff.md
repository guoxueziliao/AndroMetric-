# 0.2.16 验收与交接

## 推荐实现顺序

1. 代码校准：确认现有 stats、state、dashboard、data health、training 和 review 能力。
2. 长期数据质量：用现有数据计算覆盖率、缺口和低价值字段提示。
3. 洞察可信度：为趋势、复盘、建议和阶段回顾补可信度说明。
4. 首页 / 状态页：调整长期用户重点，不做新手引导。
5. 开发前校准清单：形成后续版本实现前的固定检查模板。
6. Safety / Privacy 审计：避免诊断、强因果和过度提醒。

## 验收底线

- 长期用户能看出哪些数据可靠。
- 样本不足的洞察会解释限制。
- 首页不再以新手启动为主。
- 状态页能突出长期趋势和可信度。
- 后续版本进入开发前有校准清单。
- 不新增 schema / migration。
- 不恢复新手引导方向。

## 开发会话必读

- 先读 [`../plan-0.2.16.md`](../plan-0.2.16.md)。
- 再读本目录全部专题文档。
- 实现前必须按 [`pre-implementation-calibration.md`](./pre-implementation-calibration.md) 输出代码校准结果。
- 若真实代码需要 schema / migration，停下来重谈。

## 停下来重谈

- 需要新增数据表或迁移。
- 需要重做 Dashboard。
- 需要新增统计大模型。
- 需要医学风险判断。
- 需要 AI 教练。
- 需要改变 0.2.15 主导航。

# 0.2.16 洞察可信度

## 目标

长期用户看到趋势、复盘和建议时，应该知道哪些可以认真看，哪些只是轻提示。

## 可信度层级

- 可看：样本足够、字段覆盖稳定、时间窗口清楚。
- 谨慎看：样本有限、缺口明显、只适合观察方向。
- 暂不看：记录太少、字段缺失或窗口不成立。

## 判定依据

实现可以按真实代码校准阈值，但必须至少考虑：

- 时间窗口是否成立。
- 样本量是否足够。
- 关键字段覆盖率是否稳定。
- 最近记录是否存在突然缺口。
- 结论是否依赖敏感或低覆盖字段。

## 适用位置

- 个人常态。
- 长期趋势。
- 变化解释层。
- 复盘报告。
- 训练建议。
- 阶段回顾。

## 表达要求

可信度不能只用颜色或分数。

每个提示至少说明：

- 样本来自哪个窗口。
- 哪些字段缺失。
- 为什么只能谨慎看。
- 用户继续记录什么会让结论更稳定。

## UI 要求

- 可信度标签必须靠近对应洞察。
- 限制说明必须可见，不藏在导出文件里。
- 同一条洞察如果可信度不足，应弱化排序或放到次级区域。
- `暂不看` 不等于报错；页面应说明继续记录后何时可看。

## 不做

- 不做强因果。
- 不把可信度包装成医学准确率。
- 不输出确定性预测。
- 不用“异常”“风险”“病变”等医疗化词汇。

# 0.2.8 刀 5 Safety / Privacy 审计

> 本文定义 0.2.8 收口前的安全、隐私和文案审计。刀 5 只审计和修正，不扩产品范围。

## 审计范围

必须检查：

- Stats 个人常态入口。
- 指标卡片。
- 指标详情层。
- Dashboard 轻提示。
- 空状态。
- record gaps。
- toast / confirm。
- 导出相关文案。

## 文案禁词

不得出现：

- 异常。
- 疾病风险。
- 性功能障碍。
- 成瘾。
- 正常男人。
- 达标 / 不达标。
- 预测未来表现。
- 某个伴侣导致你变差。

如需提及“风险”，只能用于“隐私风险”“误读风险”等产品安全语境。

## 隐私审计

必须确认：

- 派生结果不写入 JSON backup。
- 派生结果不默认写入 CSV。
- Dashboard 不展示敏感指标值。
- 详情层不暴露 notes 全文。
- 关系、周期、色情内容只作为隐藏或二级上下文。

JSON backup 仍只负责原始数据完整恢复。

## UI 审计

必须确认：

- 没有总分、等级、排行。
- 没有人群比较。
- 没有医学阈值。
- 没有 high confidence。
- 没有红绿好坏评价。
- 空状态鼓励继续记录，不制造失败感。

## 版本验证

实现完成后至少跑：

```bash
npm run test
npm run typecheck
git diff --check
```

如果触达 UI 或导出入口，也跑：

```bash
npm run build
```

## 审计结论

收口时必须产出：

- 禁词检查结果。
- 入口和 Dashboard 检查结果。
- 导出边界检查结果。
- 仍需后续版本处理的问题。

## 停下来重谈

- 需要可读导出个人常态结果。
- 需要分享图或对外报告。
- 需要医学提示、风险等级或预测。
- 需要把关系 / 周期 / 色情内容放进首屏判断。

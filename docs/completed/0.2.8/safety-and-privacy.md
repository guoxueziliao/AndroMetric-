# 0.2.8 Safety / Privacy Rails

> 本文负责 0.2.8 个人基线和长期趋势的安全、隐私、文案和导出边界。

## 核心风险

个人基线容易被误读成：

- 医学正常范围。
- 异常检测。
- 健康风险等级。
- 性表现评分。
- 自我比较压力。
- 伴侣或行为归因。

0.2.8 必须把这些风险前置为验收条件。

## 禁止文案

不得出现：

- “异常。”
- “疾病风险。”
- “性功能障碍。”
- “成瘾。”
- “正常男人。”
- “达标 / 不达标。”
- “某个伴侣导致你变差。”
- “预测你未来表现。”

## 允许文案

可以出现：

- “偏离你的个人常态范围。”
- “仍在你的个人常态范围内。”
- “数据不足，先只展示事实。”
- “样本有限，仅供观察。”
- “缺少记录，暂不输出趋势判断。”
- “建议结合恢复、睡眠和压力一起看。”

## 隐私边界

长期趋势默认本地展示。

CSV / 报告类可读导出默认不包含：

- notes 全文。
- 伴侣私密备注。
- 关系沟通全文。
- 色情内容本体。
- 具体平台名或内容标题。

JSON backup 仍应完整保留原始数据和恢复所需字段。

## 指标边界

可以展示：

- 个人历史中位数。
- 个人历史范围。
- 当前窗口值。
- 样本量和缺失天数。
- confidence。
- limitations。

不能展示：

- 人群百分位。
- 医学风险等级。
- 性表现能力分。
- 伴侣排名。
- 自动诊断标签。

派生结果不得写入 JSON backup、CSV 导出或 IndexedDB store；详见 [派生数据契约](./derived-data-contract.md)。

## 审计要求

0.2.8 收口前必须审计：

- 长期趋势卡片。
- Dashboard 轻提示。
- 空状态。
- 导出提示。
- report / CSV 文案。
- toast / confirm。
- 所有 “偏离” 相关文案。

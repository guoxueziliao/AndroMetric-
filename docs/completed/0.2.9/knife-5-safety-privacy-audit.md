# 0.2.9 刀 5 Safety / Privacy 审计

> 本文定义 0.2.9 收口前的安全、隐私和文案审计。

## 审计范围

必须检查：

- 解释卡片。
- 解释详情层。
- Dashboard 轻提示。
- 空状态。
- record gaps。
- toast / confirm。
- 导出相关文案。

## 禁止词

不得出现：

- 导致。
- 主要原因。
- 疾病风险。
- 成瘾。
- 性功能障碍。
- 某个伴侣导致。
- 你应该停止。

“导致”只能出现在禁止说明或开发文档中，不出现在用户 UI。

## 隐私检查

必须确认：

- 不展示 notes 全文。
- 不展示色情标题、平台名、URL。
- 不展示伴侣私密备注全文。
- 不展示关系沟通全文。
- 不新增导出字段。

## 验证底线

实现完成后至少跑：

```bash
npm run test
npm run typecheck
git diff --check
```

触达 UI 时也跑：

```bash
npm run build
```

## 审计结论

收口时必须产出：

- 禁词检查结果。
- 隐私检查结果。
- Dashboard 检查结果。
- 仍需后续版本处理的问题。

## 停下来重谈

- 需要用户可读导出。
- 需要外部 AI。
- 需要预测。
- 需要医学解释。
- 需要把上下文写成原因。

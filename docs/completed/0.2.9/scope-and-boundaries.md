# 0.2.9 范围与边界

> 0.2.9 是个人变化解释层 v1。它作为单独实现版本排期，但产品上归属 0.2.8 个人常态 / 长期趋势详情。

## 核心定位

0.2.9 帮用户回答：

- 最近变化附近，哪些上下文值得一起看？
- 这些上下文记录是否完整？
- 是否有需要继续观察的方向？

它不回答：

- 到底是什么导致了变化。
- 用户是否有医学风险。
- 用户应该立刻改变什么行为。

## 产品归属

- 0.2.9 不做独立工作台。
- 0.2.9 不新增顶级入口。
- 解释卡片只出现在 0.2.8 个人常态 / 长期趋势详情中。
- Dashboard 只做“有变化可回看”的轻提示，不直接展示解释卡。

## 可做范围

- 从 0.2.8 读取变化状态。
- 汇总变化窗口附近的上下文。
- 生成温和解释卡片。
- 展示上下文样本量和缺口。
- 提醒用户继续观察。

## 第一层上下文

优先考虑：

- 睡眠。
- 压力。
- 性负荷。
- 恢复 / 疲劳。
- 色情使用。
- 运动。

这些上下文已在既有记录和复盘中较容易获得。

## 条件上下文

仅在真实落地后读取：

- 0.2.5 目标历史。
- 0.2.7 关系上下文。
- 0.2.7 周期 / 女性性健康关怀上下文。

未落地时必须隐藏，不做占位结论。

## 明确不做

- 相关性变因果。
- 医学解释。
- 人群比较。
- 伴侣归因。
- 色情内容评价。
- 自动生成目标。
- 外部 AI 分析。
- 独立解释层工作台。

## 停下来重谈

出现以下需求应回到规划：

- 需要新字段才能解释第一层上下文。
- 需要预测未来变化。
- 需要输出“主要原因”。
- 需要给出强行动建议。
- 需要把解释卡片写成诊断。

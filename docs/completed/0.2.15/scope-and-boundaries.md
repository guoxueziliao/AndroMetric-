# 0.2.15 范围与边界

## 核心定位

0.2.15 解决“功能越来越多后，用户不知道该从哪里进、同一个能力散落在多个地方、入口命名不稳定”的问题。

它不是新功能版本，而是导航与信息架构整理版本。

## 做什么

- 梳理主导航职责。
- 梳理二级入口层级。
- 给 0.2.7 - 0.2.14 的新增能力指定默认归属。
- 收束重复入口，让快捷入口回到同一数据口径。
- 统一入口文案，减少“点进去才知道是什么”的情况。

## 默认边界

- 默认不新增顶级导航。
- 默认不新增 schema / migration。
- 默认不新增新页面类型；优先整理现有页面和入口。
- 如果需要新页面，也只作为导航容器或聚合页，不新增业务数据能力。

## 不做什么

- 不做隐私模式 / 屏幕遮挡 / 隐私模糊。
- 不做全局搜索 / 快速跳转。
- 不做品牌命名和 PWA 资产统一。
- 不新增复盘算法、趋势算法、训练模型或补剂模型。
- 不做账号、云同步、后端或外部 API。

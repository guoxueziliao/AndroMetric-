# 0.2.15 执行计划

## 目标

整理导航与信息架构，让核心模块的入口、命名和层级更清楚。开发完成后，用户应能从主导航理解“日历 / 状态 / 性生活 / 我的”的分工，且不会出现隐私模糊或全局搜索方向。

## 进入开发前校准

先读这些文件：

- `app/viewTypes.ts`：主视图枚举。
- `app/AppContent.tsx`：`activeMainView` 状态和桌面布局。
- `app/MainViewRouter.tsx`：各主视图渲染和跳转。
- `app/BottomNav.tsx`：移动端底部导航。
- `app/SidebarNav.tsx`：桌面侧边导航。
- `features/dashboard/Dashboard.tsx`：日历 / 今日 / 历史入口。
- `features/state/StateView.tsx`：状态页。
- `features/sex-life/SexLifeView.tsx`：性生活页。
- `features/profile/MyView.tsx`：我的 / 设置 / 数据入口。

## 信息架构决策

四个主入口保持轻量：

- 日历：今天、历史日期、每日记录、时间线。
- 状态：身体状态、趋势、个人常态、复盘入口。
- 性生活：性事件、自慰、色情使用、关系上下文、女性性健康关怀。
- 我的：设置、数据安全、标签、健康项目管理、版本信息。

## 实现切片

### 切片 1：入口矩阵落地

- 按 `module-placement-matrix.md` 逐项检查现有入口。
- 每个模块只能有一个主入口，其他地方只做快捷入口或上下文入口。
- 同一功能的命名必须一致，例如“色情使用”不要在另一处写成“看片”。

### 切片 2：导航代码整理

- 如果新增主入口，必须同步 `AppView` / `MainView`、`BottomNav`、`SidebarNav`、`MainViewRouter`。
- 第一版不建议新增第五个主 tab；优先在现有四 tab 内整理。
- 图标使用 lucide，不手写 SVG。

### 切片 3：我的页整理

- 数据安全、备份、导入导出放在数据区域。
- 标签管理、健康检查放在数据生态区域。
- 健康项目管理可放在我的页的健康管理区域。
- 隐私模糊 / 屏幕遮挡入口直接删除，不保留空开关。

### 切片 4：状态页与统计入口

- 状态页优先承接复盘、长期趋势和训练目标。
- 不把状态页变成所有功能的二级菜单。
- 从日历跳到复盘时，目标应明确到状态页中的复盘区。

## 实现顺序

1. 盘点现有入口并列出重复 / 命名不一致项。
2. 先改命名和重复入口，不改数据模型。
3. 再改导航组件和路由映射。
4. 再整理我的页分组。
5. 最后做移动端 / 桌面端路径走查。

## 验收步骤

- 底部导航和桌面侧栏项目一致。
- 四个主入口的功能归属能从页面标题和内容看出来。
- 色情使用入口归属性生活，不再只藏在日期表单。
- 隐私模糊入口不存在。
- 没有全局搜索 / 快速跳转入口。
- 从 Dashboard 的复盘跳转到状态页后，用户能看到复盘相关区域。

## 测试建议

- TypeScript 检查能发现遗漏的 `MainView` 分支。
- 手动验证移动端底部导航、桌面侧栏、表单返回。
- 用 `rg` 检查旧文案：`看片`、`隐私模糊`、`全局搜索`、`快速跳转`。

## 停线项

- 需要新增第五个主导航。
- 需要重做 Dashboard 或 StateView 的整体视觉结构。
- 需要恢复隐私模糊 / 屏幕遮挡。
- 需要全局搜索 / 全文搜索 / 快速跳转。

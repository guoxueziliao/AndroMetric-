## 核心决策

- 0.2.7 的核心基调：帮助男性用户在性生活和亲密关系中更细致地关怀女性伴侣的周期、感受、边界和恢复；不把女性伴侣变成被管理、被评分或被监控的对象。
- 关系上下文系统 v1 包含四类核心信息：关系情境、沟通状态、边界 / 偏好、事后反馈 / 恢复背景。
- 女性性健康关怀 v1 复用现有月经 / 生殖健康内容，把经期、预计窗口期、痛经、经量、备孕 / 避孕目标等作为亲密计划和复盘的尊重性上下文。
- 伴侣资料是关系上下文的长期来源之一；单次记录也必须能填写本次上下文，不能强迫用户先维护完整伴侣档案。
- 新能力优先服务用户自我理解：哪些沟通、边界和情境需要被记录，哪些信息可能影响恢复、满意度和目标回顾。
- 系统生成内容不得输出“哪个伴侣更好”“某个伴侣导致表现好 / 差”“伴侣排名”。
- 新流程优先使用 stable partner id；legacy name 记录只能兼容读取，不能静默猜测或自动合并。
- 伴侣私密备注、偏好、敏感点和沟通记录默认不进入 CSV / 报告类可读导出。

## 明确不做

- 完整关系管理系统。
- 多伴侣关系图谱。
- 伴侣评分、排名、能力比较或优劣判断。
- 面向伴侣的分享报告。
- 情感咨询、关系诊断或冲突裁决。
- 医疗诊断、怀孕结论、排卵确定性判断或替代医生建议。
- 把月经 / 排卵 / 性欲状态当成要求性生活的理由。
- 自动合并疑似同一伴侣。
- 云端同步或外部 AI 分析。

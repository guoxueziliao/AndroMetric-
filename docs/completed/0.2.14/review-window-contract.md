# 0.2.14 复盘窗口契约

## 目标

7 天、14 天、30 天、周报和月报不能只是按钮不同。用户点击后必须看出差异。

## 窗口定义

- `7 天`：近期状态快照，重点看最近变化、记录完整度和明显异常。
- `14 天`：短周期对比，重点看前后两周变化，适合训练目标和生活节律。
- `30 天`：月内趋势，重点看稳定性、波动、样本量和长期状态。
- `周报`：自然周回顾，固定周一到周日。
- `月报`：自然月回顾，适合阶段性回看和归档。

## UI 要求

- 页面标题必须显示当前窗口。
- 副标题必须解释窗口含义。
- 时间范围必须可见。
- 主要模块必须有差异。
- 如果复用同一组件，也必须有不同解释文案和不同重点模块。

## 不做

- 不新增复盘算法。
- 不输出新的医学结论。
- 不把窗口差异做成评分或排名。

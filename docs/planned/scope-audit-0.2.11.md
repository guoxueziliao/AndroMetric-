# 0.2.11 范围审计

> 本文记录 0.2.11 在 0.2.8 - 0.2.12 范围审计中的结论。

## 审计结论

保留为单独实现版本，但产品上是 0.2.10 观察结束后的经验保存能力，不成为独立知识库。

## 理由

- 0.2.11 的价值来自“用户主动留下经验”，不是系统生成结论。
- 经验卡如果要以后回看，必须有可靠持久化位置。
- 0.2.11 默认复用 0.2.5 / 0.2.10 history 或 note-like 结构，但这需要真实代码校准。
- 如果没有可复用存储，继续坚持“不新增 schema”会让经验卡变成假能力。
- 独立经验知识库会过早扩大概念和入口。

## 保留范围

0.2.11 继续保留：

- 观察结束后手动保存。
- 系统预填事实摘要。
- 用户手动填写心得。
- 关联观察窗口、指标、上下文和 limitations。
- 历史回看和基础筛选。
- 不自动总结规律、不生成建议、不做诊断。

## 已同步方向

0.2.11 文档已按以下方向收口：

- 对外弱化为“保存这次观察经验”，不强调独立经验系统。
- 主入口只在 0.2.10 观察结束回看页。
- 历史回看优先归入 0.2.5 / 0.2.10 历史；不依赖已删除的 0.2.13。
- 刀 1 必须明确存储结论：可复用存储、需要新增 schema、或暂停实现。
- 如果需要新增 schema，应回到规划重谈，但不能假装无存储也能完成经验沉淀。

## 已确认

- 0.2.11 继续作为单独实现版本。
- 0.2.11 产品上作为 0.2.10 观察结束后的经验保存能力。
- 不并入 0.2.10。
- 不推迟到 0.2.10 和存储完全落地后再重新讨论，但实现时若没有可靠存储必须暂停或回到规划。

# 0.2.5 刀 3 跨周期进度反馈

> 本文只规划刀 3：从目标历史和 check-in 中派生长期反馈。它不是评分系统。

## 状态

- 所属版本：0.2.5。
- 当前阶段：待刀 2 完成后执行。
- 前置：[`knife-2-goal-history-workbench.md`](./knife-2-goal-history-workbench.md)。
- 实现边界：周期摘要、category 分布、check-in 覆盖、稳定性和缺失提示。

## 目标

刀 3 完成后，用户可以看到长期目标历史里的事实和变化方向，而不会被打分、排名或诊断。

必须完成：

- 目标 category 分布。
- active / paused / completed / archived 数量。
- check-in 覆盖情况。
- 近几个周期的目标关注方向。
- 缺失数据提示。
- 样本不足提示。
- 稳定性摘要。

## 派生指标

默认运行时派生：

- 各状态目标数量。
- 各 category 目标数量。
- 每个目标的 check-in 数量。
- 最近一次 check-in 时间。
- 最近 4 个周期的关注 category。
- 多个目标暂停或归档的提醒。
- 目标有无 source insight。
- orphan check-in 数量。

不得持久化：

- long-term aggregate cache。
- TrainingSuggestionHistory。
- GoalTemplate。
- Milestone。
- Badge。
- Level。

## 样本规则

样本不足时：

- 只展示事实。
- 显示继续记录。
- 不输出趋势判断。
- 不生成表现结论。

建议下限：

- 少于 2 个完成周期：不展示趋势，只展示事实。
- 少于 3 次 check-in：不展示稳定性判断，只展示记录数量。
- 伴侣反馈样本不足：不输出关系结论。

具体阈值可由实现窗口根据现有 review engine 规则微调，但不能放宽成强判断。

## 文案形态

允许：

- “近几个周期里，你更常关注恢复。”
- “硬度稳定目标已有 check-in，但样本仍有限。”
- “本阶段更适合继续记录，而不是判断趋势。”
- “多个目标处于暂停状态，可能适合减少目标数量。”

禁止：

- “你能力提升了 X 分。”
- “你训练失败。”
- “你比之前更强。”
- “某个伴侣让你表现更好。”
- “色情使用导致你变差。”
- “你应该增加性次数 / 射精次数。”

## 验收

- 跨周期摘要只展示事实、稳定性和缺口。
- 样本不足不输出趋势判断。
- `cycleFeeling` 不被命名为评分。
- 不展示失败率、最高纪录、连续打卡或能力分。
- 不新增 schema / migration。

## 交接给刀 4

刀 3 结束后，刀 4 将目标历史工作台和跨周期反馈接入复盘 / 统计区域，并保持 Dashboard 轻提示。

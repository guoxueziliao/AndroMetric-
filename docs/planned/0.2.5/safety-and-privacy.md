# 0.2.5 Safety / Privacy Rails

> 本文负责 0.2.5 长期目标历史的安全、隐私、文案和指标展示边界。

## 核心原则

长期历史更容易制造压力，所以 0.2.5 必须比 0.2.4 更克制。

目标历史不能变成：

- 能力评分。
- 失败记录。
- 伴侣排名。
- 性次数挑战。
- 色情使用成就。
- 训练羞辱。

## 文案边界

允许：

- “这几个周期里，你更常关注恢复。”
- “本阶段记录质量更完整。”
- “目标历史样本仍少，先继续观察。”
- “多个目标暂停，可能适合降低目标数量。”

禁止：

- “你训练失败。”
- “你能力变差。”
- “你不够自律。”
- “某个伴侣影响你的表现。”
- “挑战更多性行为。”

## 隐私边界

长期历史默认本地展示。

禁止：

- 分享图。
- 社交传播。
- 云端同步。
- 外部 AI 分析。
- 自动导出敏感目标历史。

导出时：

- 默认只走 JSON backup。
- 报告摘要必须提醒包含敏感成人健康数据。
- 不默认包含 notes 全文、伴侣私密备注或色情内容本体。

## 指标边界

允许：

- 目标数量。
- check-in 数量。
- category 分布。
- 缺失数据。
- 稳定性摘要。

禁止：

- 能力分。
- 失败率。
- 伴侣排名。
- 性次数成就。
- 射精次数成就。
- 色情使用时长累计。

## Safety Rails 继承

0.2.5 必须继承：

- 0.2.3 confidence / limitations。
- 0.2.4 Safety Rails。
- 本地优先。
- 不诊断。
- 不判定成瘾。
- 不做强因果。
- 不评价伴侣。

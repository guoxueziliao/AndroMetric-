# 0.2.5 刀 5 Safety / Privacy 审计

> 本文只规划刀 5：0.2.5 收口前的安全、隐私、文案和指标审计。它不新增产品能力。

## 状态

- 所属版本：0.2.5。
- 当前阶段：待刀 4 完成后执行。
- 前置：[`knife-4-entry-ui-dashboard.md`](./knife-4-entry-ui-dashboard.md)。
- 实现边界：目标历史、长期摘要、Dashboard、空状态、toast、confirm 和导出提示审计。

## 审计目标

长期历史更容易制造压力，所以 0.2.5 必须比 0.2.4 更克制。

刀 5 完成后，应确认所有新增内容都没有把轻目标历史变成：

- 能力评分。
- 失败记录。
- 伴侣排名。
- 性次数挑战。
- 色情使用成就。
- 训练羞辱。

## 审计范围

必须审计：

- 目标历史列表。
- 目标详情。
- check-in 历史。
- 归档 / 恢复 confirm。
- 跨周期摘要。
- category 分布。
- 缺失数据提示。
- orphan check-in 提示。
- Dashboard 轻入口。
- 空状态。
- 导出或报告摘要提示。

## 禁止文案

不得出现：

- “你训练失败。”
- “你能力变差。”
- “你不够自律。”
- “某个伴侣影响你的表现。”
- “挑战更多性行为。”
- “击败上一次。”
- “色情刺激越强越能训练。”

## 隐私边界

必须确认：

- 默认本地展示。
- 不做分享图。
- 不做社交传播。
- 不做云端同步。
- 不做外部 AI 分析。
- 不自动导出敏感目标历史。
- 报告摘要提醒包含敏感成人健康数据。
- 不默认包含 notes 全文、伴侣私密备注或色情内容本体。

## 指标边界

允许：

- 目标数量。
- check-in 数量。
- category 分布。
- 缺失数据。
- 稳定性摘要。

禁止：

- 能力分。
- 失败率。
- 伴侣排名。
- 性次数成就。
- 射精次数成就。
- 色情使用时长累计。

## 验收输出

刀 5 完成时应在实现记录中列出：

- 审计范围。
- 发现的问题。
- 修正方式。
- 剩余风险。
- 验证命令。

## 验证命令

刀 5 完成时至少跑：

```bash
npm run test
npm run typecheck
npm run build
git diff --check
```

## 版本收口

刀 5 通过后，0.2.5 才能进入版本收口：

- 更新 CHANGELOG。
- 更新 roadmap。
- 移动 planned / completed 状态。
- 确认没有新增顶级 Training 导航。
- 确认没有新增 schema / migration，除非此前重新规划过。

# 0.2.5 范围与边界

> 本文负责 0.2.5 的核心基调、版本目标和范围边界。

## 核心基调

0.2.5 的核心基调为：

> 把轻目标变成可回看的长期自我管理系统。

0.2.4 建立“建议 -> 用户确认目标 -> check-in”的闭环。0.2.5 应回答：

- 多个目标之后，用户如何回看自己在关注什么。
- 多个周期之后，哪些目标继续有意义，哪些应该暂停或归档。
- 训练建议如何从一次复盘，变成长期但克制的自我管理。

0.2.5 不应该把产品推向训练游戏、课程平台或 AI 教练。

## 版本目标

0.2.5 可以做：

- 轻量 Training Center / 目标历史工作台。
- active / paused / completed / archived 目标管理。
- 跨周期 check-in 历史。
- 目标筛选和归档。
- 长期进度反馈。
- Dashboard 当前目标轻提示。
- 目标历史的本地导出摘要候选。

## 默认数据边界

0.2.5 默认不新增 schema。

优先复用：

- `TrainingGoal`
- `GoalCheckin`
- 0.2.3 review facts / insights
- 0.2.4 rule suggestions 的即时结果

如果实现时发现必须新增 schema，应停止本版本规划并重新讨论。

## 明确不做

- AI 教练。
- 云端模型或外部 API。
- 完整技巧库。
- 系统训练课程。
- 完整游戏化等级、徽章、连续打卡。
- 性次数 / 射精次数 / 伴侣数量挑战。
- 色情使用时长挑战。
- 伴侣评分 / 排名。
- 医学诊断、成瘾判定、强因果结论。
- 分享图或社交传播。

## 0.2.4 与 0.2.5 的边界

0.2.4：

- 建立轻目标和 check-in 闭环。
- 在复盘后展示建议和候选目标。
- Dashboard 只做轻提示。

0.2.5：

- 当目标和 check-in 增多后，提供长期回看和管理。
- 明确 Training Center 作为复盘 / 统计区域内的二级工作台或 tab，不新增顶级导航。
- 可以做跨周期摘要，但仍不能做评分、排名或打卡压力。

# 0.2.18 执行工作流

## 开发前

- 读取 roadmap。
- 读取目标版本 plan。
- 读取目标版本 implementation handoff。
- 按 0.2.16 的代码校准模板确认真实代码状态。
- 标出需要停下来重谈的点。

## 开发中

- 不扩大版本范围。
- 如果发现 schema / migration 冲突，停下来重谈。
- 如果文档与代码不一致，以真实代码为准，并同步文档。

## 完成后

- 更新 roadmap 状态。
- 更新 planned / completed 索引。
- 将详细执行文档归档或删减为摘要。
- 保留开发结果、验证结果和已知后续事项。

## 不做

- 不自动提交代码。
- 不自动发版。
- 不自动创建 GitHub issue / PR。
- 不读取 GitHub PR 状态。
- 不自动 tag / release / push。
- 不替代开发会话自己的验证。
- 不把所有旧文档重新展开维护。

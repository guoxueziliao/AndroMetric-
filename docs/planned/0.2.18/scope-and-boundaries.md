# 0.2.18 范围与边界

## 核心定位

0.2.18 管理长期规划队列和开发执行状态。

它回答：

- 当前该开发哪个版本。
- 开发前要读哪些文档。
- 需要做哪些代码校准。
- 完成后如何更新文档状态。
- 哪些方向已经否决，不能恢复。

## 做什么

- 版本队列总览。
- 版本状态模型。
- 开发前校准工作流。
- 完成后归档流程。
- not doing / rejected 方向维护。

## 默认边界

- 默认只维护 docs。
- 默认不做应用内 UI。
- 默认不接 GitHub API。
- 默认不新增 schema / migration。
- 默认不自动创建 issue / PR。
- 默认不自动 tag / release / push。

## 不做什么

- 不做用户健康功能。
- 不做外部项目管理系统。
- 不做多人权限。
- 不做自动排期。
- 不做自动生成代码。
- 不做自动发版。

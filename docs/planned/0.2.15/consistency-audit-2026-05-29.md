# 0.2.15 一致性审计 2026-05-29

## 审计结论

0.2.15 已定为导航与信息架构整理 v1。

它不继承已否决的隐私模糊方向，也不继承误输入的全局搜索 / 快速跳转方向。

## 与当前代码对齐

当前代码主导航为：

- `calendar`：日历
- `state`：状态
- `sexlife`：性生活
- `my`：我的

0.2.15 第一版默认保留这四个顶级入口。

## 与前序版本对齐

- 0.2.7 关系上下文归性生活主线。
- 0.2.8 - 0.2.12 个人常态、解释、观察、经验和阶段回顾归状态主线。
- 0.2.13 健康项目和补剂周期以日历今日执行为主，管理入口可放我的或项目详情。
- 0.2.14 是入口收束版本，不作为用户可见主模块。

## 风险

- 若 0.2.7 - 0.2.14 实现后入口数量大幅增加，四入口可能需要二级分组页。
- 若“性生活”承载关系上下文、成人行为和女性性健康关怀后显得过窄，可能需要在后续版本讨论命名，但 0.2.15 默认不改顶级名称。
- 若补剂周期管理入口放入我的，必须避免用户误以为今日执行也在我的。

## 审计通过条件

- 没有把隐私模糊写回规划。
- 没有把全局搜索写成新版本。
- 没有新增 schema / migration 承诺。
- 没有扩大 0.2.7 - 0.2.14 的业务能力范围。
- 0.2.15 只处理导航、入口、命名和模块归属。

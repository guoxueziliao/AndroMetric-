# 0.2.15 验收与交接

## 推荐实现顺序

1. 代码校准：确认主导航、底部导航、侧边栏、状态页、性生活页和我的页现状。
2. 入口盘点：列出日历、状态、性生活、我的中的所有一级和二级入口。
3. 归属调整：按主导航契约移动或重命名入口。
4. 重复入口收束：保证快捷入口回到同一页面或同一数据口径。
5. 文案校准：清理语义不清的入口名。
6. 回归检查：移动端底部导航和桌面侧边栏一致。

## 验收底线

- 主导航数量没有无故增加。
- 用户能判断“记录、回看、成人行为、设置数据”分别从哪里进入。
- 0.2.7 - 0.2.14 的能力都有明确归属。
- 同一能力没有两套入口口径。
- 移动端和桌面端导航项目一致。
- “报告”“历史”“管理”等模糊词不单独作为关键动作。
- 色情使用、训练目标、补剂周期、阶段回顾都有明确主归属。

## 停下来重谈

- 需要新增顶级导航。
- 需要改品牌名、PWA 名称或应用图标。
- 需要新增 schema / migration。
- 需要做全局搜索 / 快速跳转。
- 需要恢复隐私模糊或屏幕遮挡。
- 需要新增业务能力，而不是整理入口。

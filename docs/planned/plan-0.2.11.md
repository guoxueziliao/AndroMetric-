## 当前状态

- 状态：执行草案 / 待实现。
- 前置：0.2.10 个人观察计划；0.2.5 目标历史可选。
- 主方向：观察计划结束后，让用户手动保存“这次我观察到什么”的个人经验卡。
- 产品归属：0.2.11 是单独实现版本，但用户体验上只是 0.2.10 观察结束后的经验保存能力。
- 核心基调：经验卡是用户自己的主观沉淀，不是系统结论、医学解释或因果规则。
- 默认不新增 schema；优先复用 goal/check-in/history 或本地 note-like 结构，真实实现前必须代码校准。

## 核心决策

- 0.2.11 回答：“这次观察结束后，我想留下什么经验？”
- 对外弱化为“保存这次观察经验”，不做独立经验知识库。
- 经验卡必须由用户手动保存。
- 系统可以预填事实摘要，但不能自动生成结论。
- 经验卡不能自动变成训练建议、规则或诊断。
- 经验卡默认私密本地保存，不分享、不导出可读报告。
- 经验卡可以关联观察计划、指标、上下文和日期窗口。
- 实现前必须确认可靠持久化位置；不能假装无存储也能完成经验沉淀。

## 明确不做

- 自动总结规律。
- AI 经验教练。
- 医学解释。
- 因果规则库。
- 自动推荐给未来计划。
- 独立经验知识库。
- 分享图或伴侣报告。
- 人群经验库。

## 开发刀序

1. 刀 1：代码校准与存储边界确认。
2. 刀 2：经验卡契约与来源映射。
3. 刀 3：保存入口与编辑确认流。
4. 刀 4：经验回看与筛选。
5. 刀 5：Safety / Privacy 审计。

如果 0.2.10 未真实落地，0.2.11 应暂停实现。

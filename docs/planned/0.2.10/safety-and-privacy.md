# 0.2.10 Safety / Privacy Rails

> 本文负责观察计划的安全、隐私、文案和导出边界。

## 核心风险

观察计划容易滑向：

- 医学治疗计划。
- 行为挑战。
- 打卡压力。
- 色情或性行为目标化。
- 伴侣监督。

0.2.10 必须保持“短期观察”语气。

## 禁止文案

- 治疗。
- 改善保证。
- 挑战。
- 达标 / 不达标。
- 失败。
- 戒断。
- 必须完成。

## 允许文案

- “观察一段时间。”
- “记录后再回看。”
- “样本有限，仅供观察。”
- “到期后展示事实总结。”

## 隐私边界

- Dashboard 不展示敏感主题。
- 观察计划不对外分享。
- 不生成伴侣可见报告。
- 不默认导出观察总结。

JSON backup 如复用 goal / check-in，应保持原有完整恢复语义。

## 审计要求

必须检查：

- 创建确认弹层。
- Dashboard 轻提示。
- 到期回看。
- 空状态。
- 导出文案。

## 停下来重谈

- 需要推送强提醒。
- 需要分享给伴侣。
- 需要治疗或戒断文案。
- 需要自动生成计划。

# 0.2.10 开发刀序与验收

> 本文定义 0.2.10 的候选实现顺序、验收和停线条件。

## 候选刀序

### 刀 1：代码校准与 0.2.5 依赖确认

目标：
- 确认 0.2.5 goal / check-in 是否真实落地。
- 确认 0.2.9 解释卡片是否可作为入口。
- 确认默认不新增 schema。

验收：
- 0.2.5 未落地则暂停实现。
- 输出可复用代码入口清单。

### 刀 2：观察计划契约与派生模型

目标：
- 按 [观察计划契约](./observation-plan-contract.md) 映射 goal / check-in。
- 固定 7 / 14 天窗口。

验收：
- 不新增 store。
- 不新增诊断、失败、风险字段。

### 刀 3：入口与创建确认流

目标：
- 从解释卡片和个人常态详情进入。
- 用户手动确认后创建。

验收：
- 不自动创建。
- 不使用挑战或治疗文案。

### 刀 4：回看视图与事实总结

目标：
- 到期后展示事实总结。
- 展示记录天数、缺失天数和 limitations。

验收：
- 不输出成功 / 失败。
- 不输出医学解释。

### 刀 5：Safety / Privacy 审计

目标：
- 审计创建、Dashboard、回看、空状态和导出文案。

验收：
- 禁词清除。
- 不新增分享或对外报告。
- JSON backup 地位不变。

## 版本验收

- 用户能手动创建 7 / 14 天观察计划。
- 用户能到期回看事实总结。
- 系统不自动创建、不打分、不挑战化。
- 未落地依赖会阻止实现或隐藏入口。

## 停下来重谈

- 需要新增 schema。
- 需要自动目标。
- 需要强提醒。
- 需要完成率或 streak。
- 需要医疗、戒断或治疗文案。

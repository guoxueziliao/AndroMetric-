# 0.2.10 范围与边界

> 0.2.10 是个人观察计划 v1。它承接 0.2.9 的解释卡片，但产品上只是 0.2.8 / 0.2.9 详情里的短期观察动作。

## 核心定位

0.2.10 帮用户回答：

- 我想短期观察哪个上下文？
- 接下来 7 / 14 天需要重点记录什么？
- 到期后事实上发生了什么？

它不回答：

- 我是否成功改善了健康。
- 哪个行为导致了变化。
- 我应该被系统要求做什么。

## 产品归属

- 0.2.10 不做独立观察计划中心。
- 0.2.10 不新增顶级入口或独立工作台。
- 入口只从 0.2.8 个人常态详情或 0.2.9 解释卡进入。
- 存储和历史归档优先归 0.2.5 goal / check-in。

## 可做范围

- 用户手动创建观察计划。
- 观察窗口为 7 / 14 天。
- 关联一个解释卡片或个人常态指标。
- 提醒用户关注记录缺口。
- 到期后展示事实总结。

## 观察对象

第一版允许：

- 睡眠。
- 压力。
- 性负荷。
- 恢复 / 疲劳。
- 色情使用。
- 运动。

关系 / 周期上下文只在 0.2.7 真实落地后允许观察。

## 明确不做

- 自动创建目标。
- 连续打卡系统。
- 完成率排名。
- 性行为挑战。
- 医学治疗计划。
- 外部 AI 教练。
- 伴侣共享或监督。
- 独立观察计划中心。

## 停下来重谈

- 需要新增 schema。
- 需要每日强提醒。
- 需要自动判断成功 / 失败。
- 需要把观察计划升级为训练课程。

# 0.2.22 验收与交接

## 推荐实现顺序

1. 代码校准：列出设置 Modal 现有全部分区与项，标注当前归属与标题样式。
2. 定稿目标分组（`settings-grouping-contract.md`）。
3. JSX 重排到目标分组，保持每项原有 props / handler 不变。
4. 统一分区标题与项样式（`entry-and-naming.md`）。
5. 命名校准。
6. 行为等价回归：逐项确认功能、读写、默认值、确认弹窗不变。

## 验收底线

- 设置面板分区与分组契约一致。
- 所有分区标题用同一套样式。
- 每个设置项只出现在一个分区。
- 所有设置项行为等价（功能 / 读写 / 默认值 / 确认弹窗不变）。
- `AppSettings` 字段、`defaultSettings`、数据流零改动（`git diff` 为空）。
- 危险操作仍独立、显著、需确认。
- 未引入新视觉风格；只复用现有 token 与控件。

## 停下来重谈

- 整理无法在不拆 `MyView.tsx` 的情况下完成（升级到 B）。
- 整理需要改 `AppSettings` 字段或存储（升级到 C / 需 schema）。
- 需要新增设置项或新功能。
- 需要引入新视觉风格。
- 需要改备份 / 导出的用户可见字段或 JSON 字段名。
- 某项归属存在争议且可能改变其功能。

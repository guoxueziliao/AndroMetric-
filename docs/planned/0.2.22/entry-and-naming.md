# 0.2.22 入口与命名

## 标题样式统一

现状两套分区 / 卡片标题：

- 设置分区：`text-xs font-bold text-text-muted uppercase tracking-wider`
- 卡片标题：`font-bold text-sm text-text-primary`

目标：分区级标题统一为**一套**样式。

- 建议沿用设置分区样式 `text-xs ... text-text-muted uppercase tracking-wider` 作为所有分区标题口径。
- 卡片内若需次级标题，可保留区分，但同级标题必须一致。
- 只复用现有 token，不新增颜色。

## 命名规则

- 分区名表达用户意图，不用模糊词单独成标题（如孤立的「管理」「其他」）。
- 不出现两个语义重叠的分区（如同时有「备份」和「迁移与备份」却各放一半）。
- 设置项文案与其控件行为一致，避免「点开才知道是什么」。
- 中文命名稳定，不为整理而改动用户已熟悉的关键词（如「数据快照」「清除全部数据」）。

## 项行样式

- 设置项行（标签 + 控件）统一左标签右控件的对齐与间距。
- 开关 / 选择器 / 按钮复用现有 `shared/ui` 控件，不新造样式。

## 验收

- 所有分区标题用同一套样式。
- 无语义重叠或孤立模糊的分区标题。
- 用户熟悉的关键词未被改名。

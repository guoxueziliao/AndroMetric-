# 0.2.22 范围与边界

## 核心定位

0.2.22 解决「设置面板随版本增长后分区杂乱、标题样式不统一、管理类入口散落」的问题。

它不是新功能版本，也不是组件重构版本，而是设置面板的信息架构整理版本（A 档）。目标是分区清晰、命名一致、归属明确，且行为完全等价。

## 做什么

- 把设置面板的项收束为稳定分组，明确每项归属。
- 统一分区标题与设置项的视觉样式（复用现有 token）。
- 让标签管理、数据快照、迁移与备份、危险操作归入清晰分区。
- 消除语义不清或重复的分区标题。

## 默认边界（A 档）

- 不改 `AppSettings` 字段、类型、默认值。
- 不动存储、不新增 schema / migration。
- 不拆分 `MyView.tsx`。
- 不收束偏好存储访问层。
- 只复用现有 token，不引入新视觉。
- 任何整理必须行为等价。

## 不做什么

- 不新增设置项或新功能。
- 不改备份 / 导出的用户可见字段或 JSON 字段名。
- 不改数据读写流（`onUpdateSettings` 等）。
- 不做账号、云同步、跨设备偏好同步。
- 不恢复隐私模糊 / 全局搜索。

## 暂不做（更高档位，需显式确认才升级）

- **B 档**：`MyView.tsx` 组件拆分（JSX-only 抽 section 子组件）。受 MEMORY「大组件保守拆分」约束，需显式同意 + 浏览器验证。
- **C 档**：偏好存储访问层收束。触碰存储边界，风险最高。

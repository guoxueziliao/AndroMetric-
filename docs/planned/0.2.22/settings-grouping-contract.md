# 0.2.22 设置分组契约

> 本文给出设置面板的目标分区与每项归属。开发前以 `MyView.tsx` 实测项为准核对，不臆造设置项。

## 现状（实测）

设置 Modal「设置与数据」内分区：

- 外观
- 隐私
- 备份相关 ×3（自动备份调度、备份保留策略等）
- 迁移与备份

散落的独立卡片：

- 标签管理
- 虚拟回测实验室
- 数据快照

独立弹窗：

- 危险操作（清除全部数据）

## 目标分组

| 分区 | 归入项 | 说明 |
|---|---|---|
| 外观与通知 | theme、enableNotifications、notificationTime | 展示与提醒偏好 |
| 隐私与安全 | appLock（PIN / 生物识别 / 自动锁）、hiddenFields 相关入口 | 访问控制与字段可见性 |
| 数据与备份 | 备份调度、备份保留策略、数据快照、迁移与备份、导入导出 | 全部备份 / 快照 / 导入导出收归一处 |
| 数据管理 | 标签管理（、回测实验室按归属决定） | 管理类工具 |
| 危险操作 | 清除全部数据 | 保持独立、显著、需确认，置于末尾 |

## 归属规则

- 每项只归一个分区，不在多处重复出现。
- 备份 / 快照 / 迁移 / 导入导出统一归「数据与备份」，不再散落。
- 危险操作不混入普通分区，单独成区且视觉上可区分。
- 「虚拟回测实验室」若属开发 / 实验工具，可归「数据管理」或保留现位，开发前确认；不强行塞入用户偏好分区。

## 约束

- 分组是 JSX 重排 + 标题整理，每项保持原有 props / handler 不变。
- 不因分组改变任何项的功能或默认值。
- 若某项归属存在争议，开发前在交接里确认，不擅自删除或合并功能。

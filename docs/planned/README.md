## 阅读顺序

- 看当前状态：先读本索引。
- 做某个版本：先读对应 `plan-*.md`，再读版本目录里的 `execution-plan.md`，最后读 `implementation-handoff.md` 和专题文档。
- 查 0.2.8 - 0.2.12 边界：读单版本审计。
- 查 0.2.13：读 [`plan-0.2.13.md`](./plan-0.2.13.md)。

## 维护规则

- 单篇文档尽量保持短小；过长时拆到版本目录下的专题文档。
- 0.2.13 及之后的 planned 版本必须有 `execution-plan.md`，作为开发会话可直接执行的任务书。
- 当前 planned 维护 0.2.17 - 0.2.22 和 0.2.6 后置反馈池。
- 旧 0.2.13 档案索引方向已删除，不恢复。
- 0.2.16 起不恢复隐私模糊，不承接全局搜索 / 快速跳转。
- 0.2.19 不做 1-100 精细权重，不改变健康结论算法。
- 0.2.20 只收尾视觉 token 系统，不引入新视觉风格，不改品牌资产，不新增 schema / migration。
- 0.2.21 只扩展现有数据体检 / 修复到全表与孤立关系清理，不改健康算法，不静默删数据，默认不新增 schema / migration。
- 0.2.22 范围 A 纯信息架构整理，只整理设置分区 / 归属 / 标题样式，不改字段、不动存储、不拆组件、不引入新视觉。

# 0.2.3 研究入口（洞察与复盘增强）

> 本文档用于 0.2.3 的讨论研究。
> 当前已确定主方向：洞察与复盘增强。
> 0.2.2 负责补成人行为与色情使用记录闭环；0.2.3 在此基础上让数据产生解释价值。

## 当前状态

- 状态：研究入口。
- 前置：0.2.2 三类事件模型完成或进入执行。
- 已定主方向：洞察与复盘增强。
- 已定：允许新增轻量 adult behavior review engine。
- 待定：范围边界、复盘中心、刀序、验收标准。

## 核心基调

0.2.3 的核心基调暂定为：

> 让成人行为、色情使用和健康状态形成可解释的复盘系统。

0.2.2 解决“记录什么”和“怎么结构化”；0.2.3 解决“记录之后能看见什么”。重点不是给用户下判断，而是把硬度、性行为、自淫/自慰、色情使用、射精、睡眠、酒精、运动、压力、疲劳和满意度放到同一套复盘语言里。

0.2.3 应继续坚持：

- 不做医学诊断。
- 不做成瘾判定。
- 不做道德审判。
- 不做强因果结论。
- 样本不足时明确提示。
- 本地优先，不上传数据。

## 已定统计模型边界

0.2.3 允许新增轻量 `adult behavior review engine`。

定位：

- 用于复盘成人行为、色情使用和健康状态之间的时间关系、趋势和共现。
- 输出解释性提示、事实摘要、样本量提示和弱相关观察。
- 不输出医学诊断、成瘾判定、道德判断或强因果结论。

允许做：

- 汇总 Porn use / Masturbation / Sex / Morning hardness / Sleep / Alcohol / Exercise / Mood / Stress 等数据。
- 计算近 7 / 14 / 30 天趋势。
- 计算频率、时长、射精次数、边缘控制次数、疲劳、满意度、睡眠影响。
- 计算事件链路，例如色情使用 → 自慰 → 射精 → 睡眠 → 次日硬度。
- 做简单共现和窗口比较，例如“高色情使用时长的次日硬度均值”和“低色情使用时长的次日硬度均值”。
- 给出样本量和可信等级。

不允许做：

- 医学诊断。
- ED / 成瘾 / 性功能障碍判定。
- “X 导致 Y”的强因果结论。
- 道德化评价。
- 云端模型或联网分析。
- 复杂黑箱模型。
- 在样本不足时强行输出洞察。

### Review engine 位置

建议新增：

- `features/stats/model/adultBehaviorReview.ts`

或在 0.2.3 数据形态更清楚后，放入：

- `features/stats/model/AdultBehaviorReviewEngine.ts`

规则：

- 只依赖 domain types 和纯数据输入。
- 不直接读 Dexie。
- 不依赖 React。
- 不写 UI 文案死在模型内部，模型输出结构化结果，由 UI 层负责展示文案。

### 输出原则

review engine 输出应包含：

- `metric`
- `window`
- `sampleSize`
- `confidence`
- `direction`
- `summary`
- `supportingFacts`
- `limitations`

`confidence` 建议：

- `none`：样本不足，只展示事实。
- `low`：样本少或波动大，只做提示。
- `medium`：样本初步可看。
- `high`：样本较多且趋势稳定，但仍不做因果结论。

0.2.3 默认最多做到 `medium`，除非数据量和算法都足够支撑。

## 已定复盘中心

0.2.3 选择：

> 做综合“成人行为复盘”入口，但首屏主指标围绕硬度 / 勃起质量和恢复。

原因：

- 0.2.3 不能变成单纯色情使用统计器。
- 男性健康主轴仍然是硬度、勃起质量、恢复、睡眠和行为负荷。
- 色情使用、自淫 / 自慰、性行为、射精、边缘控制等数据是解释健康状态的关键上下文。

### 入口定位

建议入口名称方向：

- 成人行为复盘
- 性健康复盘
- 行为与恢复
- 硬度与恢复

最终命名等 0.2.1 / 品牌语言确定后再定。

### 首屏主指标

首屏优先展示：

- 硬度 / 勃起质量趋势。
- 恢复状态。
- 近 7 / 14 / 30 天性活动负荷。
- 射精次数。
- 自淫 / 自慰频率。
- 色情使用时长 / 频率。
- 睡眠影响。
- 疲劳 / 满意度。

首屏不以色情内容类型、来源平台或偏好标签为主指标。这些可以进入二级视图。

### 复盘结构

建议分层：

1. **总览**
   - 硬度、恢复、活动负荷、睡眠影响。

2. **行为链路**
   - Porn use → Masturbation / Sex → Ejaculation → Sleep → Next-day hardness。

3. **色情使用**
   - 时长、频率、动机、控制感、超时、事后状态。

4. **自淫 / 自慰**
   - 频率、射精、边缘控制、硬度、满意度、疲劳。

5. **性行为**
   - 性行为频率、硬度、射精、满意度、色情内容参与、恢复。

6. **洞察提示**
   - 带样本量和可信度的弱相关观察。

### 表达边界

- 可以说“最近高色情使用时长的次日硬度均值更低，但样本少，仅供观察。”
- 不说“色情导致你硬度下降。”
- 可以说“近 14 天射精次数较上个窗口增加，疲劳自评也更高。”
- 不说“你射精过度。”
- 可以说“睡眠不足后的硬度记录偏低。”
- 不说“你有性功能障碍。”

## 已定周报 / 月报方向

0.2.3 做周报和月报，同时保留滚动窗口复盘。

定位：

- 周报 / 月报是本地复盘报告，不是分享内容。
- 报告允许成人、露骨、直白词，但表达仍是健康复盘和行为观察。
- 永远不做分享图，避免成人隐私数据外泄风险。

### 时间窗口

0.2.3 支持：

- 滚动 7 天。
- 滚动 14 天。
- 滚动 30 天。
- 自然周周报。
- 自然月月报。

默认：

- 首页复盘使用滚动 7 / 14 / 30 天。
- 报告入口提供自然周 / 自然月。

原因：

- 滚动窗口适合日常观察。
- 自然周 / 自然月适合阶段复盘和导出。

### 报告结构

周报 / 月报建议包含：

1. **本期摘要**
   - 记录天数。
   - 硬度 / 勃起质量均值与趋势。
   - 恢复状态。
   - 性活动负荷。

2. **成人行为摘要**
   - Porn use 次数 / 总时长。
   - Masturbation 次数。
   - Sex 次数。
   - 射精次数。
   - 边缘控制次数。

3. **硬度与恢复**
   - 晨间硬度。
   - 性行为 / 自慰中的硬度。
   - 疲劳。
   - 睡眠影响。

4. **可能关联**
   - 带样本量和可信度的弱相关观察。
   - 不输出强因果。

5. **样本不足提示**
   - 哪些指标只能展示事实。
   - 哪些结论暂时不能判断。

6. **本期记录缺口**
   - 缺少睡眠记录。
   - 缺少硬度记录。
   - 成人行为事件样本不足。
   - 关联事件未补全。

### Markdown 导出

0.2.3 允许周报 / 月报导出 Markdown。

规则：

- 本地生成。
- 不上传。
- 不做分享图。
- 文案允许成人、露骨、直白。
- 导出前必须明确提醒包含敏感成人健康数据。
- 文件名不应默认包含过于露骨内容，可使用产品名 + 日期 + report 类型。

导出内容可以包含：

- 摘要指标。
- 表格。
- 文本洞察。
- 样本量 / 可信度。
- 限制说明。

不包含：

- 原始备注全文，除非用户显式选择。
- 具体平台名，除非用户显式选择。
- 任何图片 / 视频 / 内容本体。

### 分享图

0.2.3 永远不做分享图。

原因：

- 成人行为、色情使用、硬度、射精、伴侣等数据高度敏感。
- 分享图容易误触传播。
- 与本地私密工具定位冲突。

## 已定样本量与可信度规则

0.2.3 所有洞察必须带样本量和可信度。

规则：

- 所有洞察必须包含 `sampleSize`。
- 所有洞察必须包含 `confidence`。
- 样本不足时只展示事实，不输出判断。
- 0.2.3 默认最多输出到 `medium` confidence，除非后续算法和样本量足够支撑。

### Confidence 等级

```ts
type ReviewConfidence = 'none' | 'low' | 'medium' | 'high';
```

含义：

- `none`：样本不足，只展示事实和记录缺口。
- `low`：样本少或波动大，只做轻提示。
- `medium`：样本初步可看，可以输出弱相关观察。
- `high`：样本较多且趋势稳定，但仍不输出强因果结论。

0.2.3 默认不主动追求 `high`，避免复盘系统过度自信。

### 允许输出

允许：

- “在你的记录里，A 和 B 经常同日出现。”
- “高色情使用时长窗口的次日硬度均值更低，但样本有限，仅供观察。”
- “近 14 天射精次数较上个窗口增加，疲劳自评也更高。”
- “睡眠不足后的硬度记录偏低。”
- “当前样本不足，建议继续记录 14 天后再看。”
- “本期缺少睡眠记录，相关复盘可信度较低。”

### 禁止输出

禁止：

- 医学诊断，例如“你有性功能障碍”。
- 成瘾判定，例如“你色情成瘾”。
- 强因果结论，例如“色情导致你硬度下降”。
- 道德化评价，例如“你射精过度”“你应该克制”。
- 羞辱性或训诫式语言。
- 样本不足时包装成洞察。
- 将用户的成人偏好评为好/坏。

### 输出结构

review engine 输出建议包含：

```ts
interface ReviewInsight {
  id: string;
  metric: string;
  window: '7d' | '14d' | '30d' | 'week' | 'month';
  sampleSize: number;
  confidence: ReviewConfidence;
  direction?: 'up' | 'down' | 'mixed' | 'flat';
  summary: string;
  supportingFacts: string[];
  limitations: string[];
}
```

UI 文案可以更自然，但不得删除样本量和限制说明。

## 候选能力

### 方向 A：成人行为复盘时间线

把 Porn use、Masturbation、Sex、Morning hardness、Sleep、Alcohol、Exercise 等事件放进同一时间线。

可能内容：

- 生理日时间线。
- 周复盘 / 月复盘。
- 事件链路展示：色情使用 → 自慰 → 射精 → 睡眠 → 次日硬度。
- 同一天多事件密度展示。
- 关联事件高亮。

### 方向 B：硬度与行为因素关联

围绕硬度和勃起质量，展示可能相关的行为因素。

可能因素：

- 睡眠时长 / 睡眠质量。
- 酒精 / 咖啡因。
- 运动。
- 压力 / 心情。
- 性行为频率。
- 自慰频率。
- 色情使用时长 / 强度。
- 射精次数。
- 边缘控制。
- 疲劳。

### 方向 C：色情使用复盘

围绕色情使用本身，做更清晰的趋势和自评复盘。

可能内容：

- 使用频率。
- 使用时长。
- 来源类型。
- 内容类型。
- 动机。
- 控制感。
- 超时次数。
- 使用后状态。
- 与自慰、射精、睡眠和次日硬度的关系。

### 方向 D：性活动负荷与恢复

将 Sex、Masturbation、Porn use、射精、边缘控制、疲劳和睡眠整合成恢复复盘。

可能内容：

- 近 7 / 14 / 30 天性活动负荷。
- 射精频率。
- 边缘控制频率。
- 疲劳趋势。
- 恢复窗口。
- 次日硬度变化。

### 方向 E：洞察可信度与样本量提示

0.2.3 必须避免“看起来很聪明但实际样本不足”的伪洞察。

可能内容：

- 样本量提示。
- 置信度 / 可信等级。
- “仅展示事实，不做结论”的状态。
- 最近数据 vs 长期数据区分。
- 异常点解释。

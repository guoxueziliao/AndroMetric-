## 初步范围

- 复盘 / 统计入口差异：7 天、14 天、30 天、周报、月报必须有可感知差异。
- 训练建议动作语义：区分“开始目标”“稍后处理”“忽略建议”。
- 训练目标管理：误创建后必须有取消、归档、隐藏或删除入口。
- 报告入口语义：区分“查看回顾”和“导出文件”。
- 色情使用入口：从日期表单独占入口，调整为更稳定的成人行为 / 生活记录入口。

## 明确不做

- 不新增训练数据模型。
- 不新增复盘算法。
- 不做完整成人内容管理器。
- 不做色情内容收藏器。
- 不做 AI 教练、课程或训练游戏。
- 不新增后端、账号、云同步或外部 API。

## 已定边界

- 默认不新增 schema / migration。
- 优先处理入口、文案、状态和管理动作。
- 若训练目标取消 / 归档 / 隐藏 / 删除需要新增数据结构，必须停下来重谈。
- 训练建议不是目标；点击“开始”后才创建目标；目标执行记录属于历史。
- 同一条建议已开始后，不再显示“开始”，改为“已开始”或“查看目标”。
- 刚误创建且没有历史记录的目标允许“取消目标”。
- 已有 check-in 或历史记录的目标优先“归档目标”，不硬删除。
- 目标列表和目标详情都必须提供管理入口。
- 色情使用入口统一到成人行为记录主线。
- “生活 / 看片”不再作为独立口径；如果保留，只作为跳转到同一色情使用记录的快捷入口。
- 日期表单仍可快速记录色情使用，但不能是唯一入口。
- 文案避免只用“看片”，统一为“色情使用”或“色情内容使用”。
- 不做色情内容收藏器，不保存具体内容文件，不做自动识别。
- “报告”默认表示查看，不默认表示导出。
- UI 必须区分“查看回顾”和“导出文件”。
- 周报 / 月报入口点击后优先进入只读查看页；导出是查看页里的二级动作。
- 导出按钮必须明确格式，例如“导出 CSV”或“导出 JSON”。
- 如果实现暂时没有查看页，文案至少要写成“导出报告文件”，不能只写“报告”。
- `7 天`：近期状态快照，重点看最近变化、记录完整度和明显异常。
- `14 天`：短周期对比，重点看前后两周变化，适合训练目标和生活节律。
- `30 天`：月内趋势，重点看稳定性、波动、样本量和长期状态。
- `周报`：自然周回顾，适合固定周一到周日的总结。
- `月报`：自然月回顾，适合阶段性回看和归档。
- UI 必须显示当前窗口说明，例如“最近 7 天，不按自然周”。
- 不同入口的页面标题、时间范围、副标题和主要内容必须有可感知差异。
- 如果复用同一组件，也必须提供不同解释文案和不同重点模块。

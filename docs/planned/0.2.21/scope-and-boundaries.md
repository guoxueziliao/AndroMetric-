# 0.2.21 范围与边界

## 核心定位

0.2.21 解决「数据量增长后，体检和修复只看一部分表，孤立引用只报告不处理」的问题。

它不是新分析版本，而是数据健康能力的覆盖扩展版本。目标是让体检覆盖全表、孤儿可清理、清理可撤销，降低长期数据腐烂风险。

## 做什么

- 把 `runHealthCheck` 扫描范围扩展到全部事件表。
- 检测跨表孤立引用（事件→伴侣、打卡→训练目标、项目子记录→项目）。
- 把孤立关系从「只报告」升级为「可选、可撤销清理」。
- 统一数据健康入口的分表统计与孤儿列表呈现。
- 复用现有「修复前自动快照」前置。

## 默认边界

- 沿用现有 `DataHealthReport` 结构与 0-100 评分口径，只扩展统计来源。
- 默认不新增 schema / migration；只在现有外键字段内做检测与清理。
- 默认不新增记录类型或业务模型。
- 任何清理需用户确认且可回滚，不默认执行。

## 不做什么

- 不改健康算法、统计算法、复盘逻辑或评分公式。
- 不做医学诊断、风险评分、健康学解释。
- 不静默删除数据。
- 不做自动后台定时清理。
- 不做云端校验、跨设备一致性、账号。
- 不把孤立数据上升为强因果或健康结论。

# 0.2.21 实现交接摘要

## 版本定位

0.2.21 是数据健康与自修复 v1。

数据体检 / 修复已存在，但只覆盖 `logs` + `partners`，孤立引用只报告不修。本版扩展覆盖到全表并提供可撤销的孤儿清理。不改任何健康算法。

## 已确认决策

- 在现有 `DataHealthReport` / `handleRepairData` 基础上扩展，不从零做。
- 体检扩展到全部事件表。
- 孤立关系从「只报告」升级为「可选、可撤销清理」。
- 评分口径不变，只增加被扫描来源。
- 任何清理前必建快照、可回滚、需确认、不静默删数据。
- 默认不新增 schema / migration。

## 推荐实现顺序

1. 真实代码校准：全表外键字段、现有体检 / 修复入口、快照前置。
2. 扩展 `runHealthCheck` / `checkDataHealth` 覆盖全表。
3. 加跨表孤儿检测。
4. 加可选、可撤销清理（复用快照前置）。
5. 接 UI 呈现分表统计与孤儿列表。
6. 回归：issue 数下降、评分算法不变、快照可恢复。

## 停线项

- 需要新增 schema / migration。
- 需要改健康算法或评分公式。
- 需要静默 / 自动后台清理。
- 跨表父子关系无法从现有字段判定。
- 需要把孤立数据上升为健康结论。
- 需要改备份 / 导出 JSON 字段名。

## 验收底线

- 体检覆盖全表。
- 至少检出孤立伴侣引用、孤立打卡、孤立项目子记录。
- 清理前建快照且可恢复。
- 无静默删除，每类清理需确认。
- 评分算法未改。
- 修复 / 清理后 issue 数下降。

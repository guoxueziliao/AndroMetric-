# 0.2.21 验收与交接

## 推荐实现顺序

1. 代码校准：确认全表外键字段名（`domain/types/*`）、现有体检 / 修复入口与快照前置。
2. 体检覆盖扩展：`runHealthCheck` 读全表，`checkDataHealth` 增加分表统计与 issue。
3. 跨表孤儿检测：按检测清单生成 `relation` issue。
4. 可选清理：每类孤儿单独确认，复用「先建快照」前置，可撤销。
5. UI 呈现：数据健康入口显示分表统计与孤儿列表，结果用 Toast + 重新体检反馈。
6. 回归：修复 / 清理后 issue 数下降，评分算法不变，快照可恢复。

## 验收底线

- 体检覆盖全部事件表，不再只统计 logs + partners。
- 至少能检出孤立伴侣引用、孤立打卡、孤立健康项目子记录。
- 任何清理前都创建快照且可恢复。
- 没有静默删除；每类清理需确认。
- structure / completeness / analytics 评分算法未改动。
- 修复 / 清理后重新体检，对应 issue 数下降。
- 未新增 schema / migration（若被迫新增，停线讨论）。

## 停下来重谈

- 检测或清理需要新增 schema / migration。
- 需要改健康算法或评分公式。
- 需要静默 / 自动后台清理。
- 跨表父子关系无法从现有字段判定。
- 需要把孤立数据上升为健康结论或风险判断。
- 需要改备份 / 导出的 JSON 字段名。

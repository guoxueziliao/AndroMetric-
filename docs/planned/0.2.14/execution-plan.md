# 0.2.14 执行计划

## 目标

收束记录、复盘与训练入口。开发完成后，用户应能清楚区分“查看复盘”和“导出报告”，训练目标不会重复创建，也能被管理。

## 进入开发前校准

先读这些入口：

- `app/MainViewRouter.tsx`、`app/viewTypes.ts`：主视图和状态页入口。
- `features/stats/ui/ReviewSection.tsx`：成人行为复盘、周报 / 月报 / 导出报告语义。
- `features/stats/model/adultBehaviorReviewReport.ts`：当前 Markdown 报告生成；如仍存在，应按用户已确认方向移除导出入口。
- `features/stats/ui/TrainingSection.tsx`：训练建议、开始、忽略、目标入口。
- `features/stats/model/trainingGoalService.ts`：目标创建、状态转换、归档 / 恢复。
- `core/storage/StorageService.ts`：`trainingGoals`、`goalCheckins` repository。
- `features/stats/ui/GoalHistorySection.tsx`：历史、筛选、删除 / 归档能力。

## 问题对应

本版本必须逐条处理这些反馈：

- 7 天、14 天、30 天、周报、月报点击后差异不明显。
- 训练建议点多次“开始”会出现多个目标。
- 报告区域被理解为查看，实际却是导出。
- 训练目标缺少删除或归档入口。
- 色情使用入口散落，生活里的“看片”和日期表单里的入口需要收束。

## 实现切片

### 切片 1：复盘窗口语义

- 在复盘 UI 中明确选中窗口：7 天 / 14 天 / 30 天 / 周报 / 月报。
- 每个窗口切换后必须改变时间范围标题、样本数、缺口提示和内容摘要。
- 周报 / 月报是查看模式，不自动导出。
- 若保留导出，只放在“导出”按钮里，不能作为主入口文案。

### 切片 2：训练建议幂等

- 点击“开始”前先检查是否已有同 `linkedInsightId` 或同类 active goal。
- 已存在时不新建，改为跳转 / 展开已有目标。
- `TrainingSection` 的按钮状态要有 `已开始`、`继续`、`已忽略`。
- 忽略只影响建议展示，不删除历史目标。

### 切片 3：训练目标管理动作

- active 目标提供暂停、完成、归档。
- archived 目标提供恢复。
- 删除如果当前数据结构已有 repository 支持，可做二次确认删除；如果删除会破坏签到关系，优先做归档。
- Goal checkin 孤儿检查保留在导入 / integrity 层。

### 切片 4：色情使用入口收束

- “性生活”中应有色情使用入口或入口说明，不能只能从日期表单进入。
- 每日记录里的入口继续可用，但命名要和性生活入口一致。
- “生活”中的“看片”命名改为与成人行为模型一致的“色情使用”或同义短文案。

### 切片 5：报告导出移除或降级

- 用户已确认 Markdown 导出无用，直接砍掉对应入口。
- 如果代码仍保留 `adultBehaviorReviewReport.ts`，先不一定删除模型文件，但 UI 不再暴露 Markdown 导出。
- 避免把“报告”按钮放在查看入口位置。

## 实现顺序

1. 校准当前 stats UI 是否已经被 0.2.4 / 0.2.5 改过。
2. 先改复盘窗口和报告语义。
3. 再改训练建议幂等和目标管理动作。
4. 再改色情使用入口和命名。
5. 最后做回归测试和文案一致性审计。

## 验收步骤

- 7 / 14 / 30 天切换后，标题、范围、样本数和摘要不同。
- 周报 / 月报点击后是查看复盘，不触发下载。
- 同一个训练建议连续点击 5 次，只产生一个 active goal。
- 目标可以暂停、完成、归档；归档后不出现在 active 列表。
- 色情使用能从性生活主入口进入，也能从每日记录关联进入。
- UI 中不再出现主路径 Markdown 导出。

## 测试建议

- 给 `trainingGoalService` 增加防重复选择器或创建前判断测试。
- 给复盘窗口 builder 增加不同 window 的样本范围测试。
- 手动验证移动端：训练按钮连续点击、归档确认、色情使用入口。

## 停线项

- 删除训练目标会导致签到历史无法解释，且无法用归档替代。
- 需要重新设计训练目标 schema。
- 需要恢复 Markdown 导出。
- 需要把色情使用做成内容收藏器、自动识别或成瘾判定。

# 0.2.10 范围审计

> 本文记录 0.2.10 在 0.2.8 - 0.2.12 范围审计中的结论。

## 审计结论

保留为单独实现版本，但产品上是 0.2.8 / 0.2.9 详情里的轻量观察动作，不成为新的训练目标系统。

## 理由

- 0.2.10 的入口来自 0.2.9 解释卡和 0.2.8 个人常态详情。
- 0.2.10 的存储默认复用 0.2.5 goal / check-in，不应新增第二套计划模型。
- 如果把 0.2.10 做成独立“观察计划中心”，会和 0.2.5 Training Center 重复。
- 观察计划的价值是短周期观察，不是训练、治疗、挑战或课程。

## 保留范围

0.2.10 继续保留：

- 用户手动创建。
- 7 / 14 天观察窗口。
- 关联解释卡或个人常态指标。
- 到期事实回看。
- 记录天数、缺失天数和 limitations。
- 不自动创建、不打分、不挑战化。

## 已同步方向

0.2.10 文档已按以下方向收口：

- 对外名称弱化为“短期观察”，不强调独立“计划系统”。
- 入口只从 0.2.8 / 0.2.9 详情进入，不新增独立工作台。
- 存储和历史归档继续归 0.2.5 goal / check-in 或其后续实现。
- 若 0.2.5 未落地，0.2.10 暂停，不单独补 schema。

## 已确认

- 0.2.10 继续作为单独实现版本。
- 0.2.10 产品上作为 0.2.8 / 0.2.9 的轻量观察动作。
- 不并入 0.2.9。
- 不推迟到 0.2.5 完全稳定后再重新讨论，但实现时若 0.2.5 未落地必须暂停。

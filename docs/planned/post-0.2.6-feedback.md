# 0.2.6 后置反馈池

> 本文记录 0.2.4 / 0.2.5 完成后发现、但不并入 0.2.6 数据安全版本的问题。0.2.6 先专注数据安全与长期数据承诺。

## 当前状态

- 状态：0.2.6 之后再定归属。
- 不占用 0.2.6 范围。
- 不直接改变 0.2.7 - 0.2.12 已定主题。
- 0.2.6 完成后再决定是前置插刀、小版本，还是并入已有主线。

## 后置问题

1. 状态 / 历史统计 / 复盘的 7 天、14 天、30 天、周报、月报点击后差异不清。
2. 训练建议的“开始 / 忽略”语义不清，重复点击“开始”会产生多条目标或内容。
3. 训练目标误创建后缺少删除、取消、归档或隐藏入口。
4. 报告入口容易被理解为“查看报告”，实际是导出，需要区分查看和导出。
5. 色情使用缺少稳定入口，不能只从日期表单进入；“生活 / 看片”和成人行为记录需要统一入口语义。
6. 补剂不应写死为固定项，需要支持集中管理、周期、剂量和启停。

## 暂定归属

- 复盘入口与报告查看：偏复盘 / 统计体验债，0.2.6 后单独校准。
- 训练重复开始与删除入口：偏训练目标管理债，优先级高于新训练能力。
- 色情使用入口：归成人行为、复盘、个人常态主线，不做独立色情内容收藏器。
- 补剂周期：偏健康数据模型扩展，可能需要 schema / migration，必须单独讨论。

## 进入后续版本前的判断

- 如果只改入口、文案和已存在数据的管理动作，可以作为 0.2.6 后小插刀。
- 如果需要新增数据模型、迁移或长期统计口径，应单独规划版本。
- 如果会打断 0.2.7 关系上下文主线，应先做范围审计再决定是否前置。

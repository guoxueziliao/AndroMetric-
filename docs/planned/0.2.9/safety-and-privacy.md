# 0.2.9 Safety / Privacy Rails

> 本文负责个人变化解释层的安全、隐私、文案和导出边界。

## 核心风险

解释层容易被误读为：

- 因果结论。
- 医学解释。
- 行为诊断。
- 伴侣归因。
- 色情内容评价。
- 自动行动建议。

0.2.9 必须把解释写成“上下文观察”，而不是“原因判断”。

## 允许文案

- “变化窗口内压力记录较多，建议结合恢复记录一起看。”
- “这段时间睡眠记录偏少，暂不判断趋势。”
- “性负荷变化和恢复反馈可以放在一起观察。”
- “样本有限，仅供观察。”

## 禁止文案

- “睡眠导致硬度下降。”
- “色情使用造成了问题。”
- “某个伴侣让你状态变差。”
- “你应该停止某行为。”
- “这是主要原因。”
- “存在疾病风险。”

## 隐私边界

解释卡片默认本地展示。

不得默认展示：

- notes 全文。
- 色情标题、平台名、URL。
- 伴侣私密备注全文。
- 关系沟通全文。
- 周期或不适细节全文。

JSON backup 仍只负责原始数据恢复；解释卡片不写入备份。

## 导出边界

0.2.9 v1 不新增可读导出。

如果未来需要导出解释卡片，必须回到规划重谈。

## 审计要求

收口前必须检查：

- 解释卡片。
- Dashboard 轻提示。
- 空状态。
- 详情层。
- toast / confirm。
- 导出相关文案。

## 停下来重谈

- 需要输出“原因”。
- 需要医学解释。
- 需要分享或对外报告。
- 需要展示敏感全文。
- 需要外部 AI 分析。

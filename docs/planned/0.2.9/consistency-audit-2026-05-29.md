# 0.2.9 一致性审计 2026-05-29

> 本文记录 0.2.9 从研究草案收口为执行草案前的文档一致性审计。

## 状态

- 版本状态：执行草案 / 待实现。
- 主方向：个人变化解释层 v1。
- 产品归属：单独实现版本，但归属 0.2.8 个人常态 / 长期趋势详情。
- 实现前置：0.2.8 个人常态必须真实落地。
- 默认不新增 schema、store、cache 或导出字段。

## 已一致

- 0.2.9 只解释变化附近的上下文。
- 0.2.9 不做独立解释层工作台。
- 不输出因果、诊断、预测或强行动建议。
- 第一层上下文为睡眠、压力、性负荷、恢复、色情使用、运动。
- 0.2.5 / 0.2.7 条件上下文只在真实落地后读取。
- 解释卡片必须带 window、sampleSize、confidence 和 limitations。
- 用户可见 confidence 最高为 medium。

## 刀序一致性

- 刀 1：代码校准与依赖版本确认。
- 刀 2：上下文窗口构建。
- 刀 3：解释卡片引擎。
- 刀 4：并入 0.2.8 详情页的 UI 与 Dashboard。
- 刀 5：Safety / Privacy 审计。

实现窗口不得跳过刀 1。

## 安全边界

一致不做：

- 强因果结论。
- 医学解释。
- 未来预测。
- 自动生成目标。
- 伴侣归因。
- 色情内容评价。
- 外部 AI。
- 分享图或对外报告。

## 待实现窗口确认

实现时仍需确认：

- 0.2.8 `PersonalNormalResult` 是否真实存在。
- 0.2.3 review facts 可复用范围。
- 0.2.5 目标历史是否已落地。
- 0.2.7 关系 / 周期上下文是否已落地。
- UI 应挂在 Stats 哪个现有区域下。

## 审计结论

0.2.9 文档可以作为执行草案交给实现窗口。若 0.2.8 未落地，暂停 0.2.9；若真实代码与文档不一致，以真实代码为准缩小范围并回改文档。

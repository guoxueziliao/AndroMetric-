# 0.2.8 范围审计

> 本文记录 0.2.8 在 0.2.8 - 0.2.12 范围审计中的结论。

## 确认状态

保留为独立核心版本，不收窄为轻量版。

## 理由

- 0.2.8 是后续 0.2.9 - 0.2.12 的基础来源。
- 用户明确希望 0.2.8 做成大而全的长期趋势系统。
- 0.2.8 应承载个人常态、长期趋势、第一层指标、第二层指标和多窗口比较。
- 后续审计不再把 0.2.8 作为缩小对象，而是检查 0.2.9 - 0.2.12 是否围绕它合理展开。

## 保留范围

0.2.8 继续保留：

- 90 天个人常态 baseline。
- 14 / 30 天 current window。
- 第一层指标：硬度 / 晨勃、恢复 / 疲劳、睡眠 / 压力、性负荷。
- 第二层指标：色情使用、满意度、运动、目标历史和关系上下文。
- 60 / 90 / 180 天窗口比较。
- 长期趋势工作台。
- 三类状态：个人常态内、样本有限的变化、数据不足只展示事实。
- Stats 内二级入口和 Dashboard 低敏轻提示。

## 审计重点

- 0.2.8 可以大，但必须保持非诊断、非人群比较、非预测。
- 第二层指标只在依赖真实落地后读取。
- 大而全不等于新增 schema 或持久化 baseline cache；如实现时需要 cache，仍需回到规划讨论。
- 0.2.9 之后的版本需要避免和 0.2.8 重复造概念。

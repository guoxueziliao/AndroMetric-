# 0.2.8 指标计算口径

> 本文是 0.2.8 指标口径总入口。详细计算拆到第一层 / 第二层指标文档，避免单篇过长。

## 口径原则

- 优先复用 `StatsEngine`、`flattenLogsToEvents` 和 0.2.3 review facts。
- 不为 0.2.8 新增用户输入字段。
- 缺失不能按 0 处理，除非指标本身是次数 / 负荷类。
- 用户可见文案不暴露复杂公式，只显示样本量、窗口和限制说明。
- 所有指标都只和用户自己的历史比较。

## 文档拆分

- [第一层指标口径](./metric-first-layer.md)
  - 硬度 / 晨勃、恢复 / 疲劳、睡眠 / 压力、性负荷。
- [第二层指标口径](./metric-second-layer.md)
  - 色情使用、满意度、运动、目标历史和关系上下文。

## 个人范围

默认个人范围用 baseline window 中的稳健范围：

- 第一选择：P25 - P75，作为“个人常态核心范围”。
- 备选：median +/- median absolute deviation，仅在样本分布不适合分位数时使用。

状态映射和 confidence 上限见 [状态判定规则](./state-decision-rules.md)。实现前由刀 2 按现有工具能力确认。

## 停下来重谈

出现以下情况应回到规划：

- 需要新字段才能计算第一层指标。
- 需要把缺失硬填为 0。
- 需要把多个指标合成总分。
- 需要使用医学阈值或人群标准。

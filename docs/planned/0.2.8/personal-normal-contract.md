# 0.2.8 个人常态产品契约

> 本文定义 0.2.8 的核心产品契约：它首先是个人常态系统 v1，其次才是长期趋势看板。

## 核心问题

0.2.8 回答一个问题：

> 最近的我，和我自己的长期记录相比，有没有值得留意的变化？

它不回答：

- 我是否正常。
- 我是否有医学风险。
- 我比别人好还是差。
- 未来表现会怎样。

## 默认窗口

默认采用：

- baseline window：90 天。
- current window：14 / 30 天。
- extended context：180 天，只在数据足够时显示。

30 / 60 天可以作为辅助窗口，但不作为默认结论来源。

## 指标分层

第一层指标必须优先实现：

- 硬度 / 晨勃。
- 恢复 / 疲劳。
- 睡眠 / 压力。
- 性负荷。

第二层指标可以随后接入：

- 色情使用频率 / 时长。
- 满意度。
- 运动。
- 目标历史。
- 关系上下文，只在 0.2.7 真实落地后显示。

## 输出状态

所有指标只允许三类输出：

- `within_personal_normal`：仍在个人常态内。
- `shift_with_limited_confidence`：偏离个人常态，但样本有限。
- `insufficient_data`：数据不足，只展示事实。

不使用：

- abnormal。
- risk。
- diagnosis。
- pass / fail。
- good / bad。

具体判定流程见 [状态判定规则](./state-decision-rules.md)。

## 文案基调

允许：

- “最近 14 天低于你的 90 天个人常态范围，但样本有限。”
- “当前仍在你的个人常态范围内。”
- “记录不足，先只展示事实。”

不允许：

- “你的硬度异常。”
- “你的性健康风险升高。”
- “你不达标。”
- “你应该达到某个水平。”

# 0.2.8 工作台信息架构

> 本文定义个人常态工作台的入口、首屏、指标卡片和详情层级。它补充 [长期趋势工作台](./long-term-trend-workbench.md)，不替代状态规则。

## 入口原则

当前代码中 Stats 已有 `总览 / 习惯 / 性爱 / 复盘` tab，Dashboard 已有日 / 周 / 月视图。

0.2.8 v1 采用：

- 主入口：Stats / 复盘附近的“个人常态”二级入口。
- 辅入口：Dashboard 轻提示卡片，只跳转到 Stats。
- 可选入口：目标历史中的“长期变化”链接，仅在 0.2.5 真实落地后出现。

不新增顶级导航，不在 Dashboard 展开完整趋势。

## 首屏结构

首屏只回答“有没有值得留意的变化”，不要堆满图表。

建议顺序：

1. 窗口切换：14 天 / 30 天。
2. 个人常态摘要：几项仍在常态、几项样本不足、几项偏离但样本有限。
3. 第一层指标卡片：硬度 / 晨勃、恢复 / 疲劳、睡眠 / 压力、性负荷。
4. 记录缺口：缺哪些记录会影响判断。
5. 限制说明：本页只和自己的历史比较。

摘要不能合成总分，也不能写成健康评分。

## 指标卡片

卡片字段来自 [派生数据契约](./derived-data-contract.md)。

每张卡片固定展示：

- 指标名称。
- 当前窗口值。
- 90 天个人常态范围。
- 三类状态之一。
- sampleSize / missingDays。
- confidence。
- 一句 limitations。

可选展示：

- 迷你趋势线。
- 14 天与 30 天对照。
- “查看详情”入口。

不展示：

- 人群百分位。
- 医学阈值。
- 好坏评价。
- 伴侣归因。

## 详情层

点击指标后进入详情层，详情只解释“为什么这样提示”：

- 计算来源。
- baseline window 与 current window。
- 可用样本和缺失天数。
- baselineRange 的口径。
- 14 天 / 30 天方向是否一致。
- 相关上下文，例如睡眠、压力、恢复，但不做强因果。

详情可以链接到相关记录日期，但不自动修改记录，也不自动创建目标。

## Dashboard 轻提示

Dashboard 只允许一张轻提示卡片：

- “个人常态有更新，可在统计中查看。”
- “最近记录不足，个人常态暂只展示事实。”
- “有几项指标出现波动，建议结合记录缺口一起看。”

Dashboard 不显示敏感指标值、不显示偏离方向、不显示伴侣或色情内容。

如果用户没有足够数据，Dashboard 只提示继续记录，不提示健康评估。

## 空状态

空状态分两类：

- 总数据不足：提示继续记录 30 天。
- 某指标不足：只在该指标卡片中说明缺什么。

空状态不出现“无法诊断”“准确率不足”“健康风险未知”等词。

## 停下来重谈

出现以下情况应回到规划：

- 需要新增顶级导航。
- 需要在 Dashboard 展示完整趋势。
- 需要把多个指标合成一个分数。
- 需要把详情页变成诊断报告。
- 需要自动生成训练目标或行为建议。

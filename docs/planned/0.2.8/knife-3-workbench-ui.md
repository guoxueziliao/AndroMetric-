# 0.2.8 刀 3 个人常态工作台

> 本文定义刀 3 的 UI 落地边界。刀 3 只展示刀 2 的只读派生结果，不重新计算指标。

## 前置条件

- 刀 1 已完成代码校准。
- 刀 2 已输出 `PersonalNormalResult`。
- 第一层指标可用性已确认。
- 第二层指标隐藏规则已确认。

## 入口

默认入口：

- Stats / 复盘附近增加“个人常态”二级入口。

辅助入口：

- Dashboard 轻提示卡片，只跳转到 Stats。

不做：

- 顶级导航。
- Dashboard 完整趋势。
- 独立报告页。

## 首屏

首屏顺序：

1. 14 / 30 天窗口切换。
2. 个人常态摘要。
3. 第一层指标卡片。
4. 记录缺口。
5. 限制说明。

摘要只能展示数量，不合成总分。

## 指标卡片

每张卡片展示：

- 指标名称。
- 当前窗口值。
- 90 天个人常态范围。
- 三类状态之一。
- sampleSize / missingDays。
- confidence。
- 一句 limitations。

卡片可以有迷你趋势线，但不能用红绿好坏制造压力。

## 详情层

详情只解释：

- 数据来源。
- current / baseline window。
- baselineRange 口径。
- 缺失和样本限制。
- 14 / 30 天方向是否一致。

详情可以链接到相关日期，不自动修改记录，不自动创建目标。

## Dashboard

Dashboard 只允许：

- “个人常态有更新，可在统计中查看。”
- “最近记录不足，个人常态暂只展示事实。”
- “有几项指标出现波动，建议结合记录缺口一起看。”

Dashboard 不显示具体敏感数值、偏离方向、伴侣、色情内容或周期细节。

## 验收

- 入口符合 [工作台信息架构](./workbench-information-architecture.md)。
- UI 字段来自 [派生数据契约](./derived-data-contract.md)。
- 状态文案来自 [状态判定规则](./state-decision-rules.md)。
- Dashboard 只做轻提示和跳转。
- 不新增顶级导航。

## 停下来重谈

- 需要完整新页面层级。
- 需要 Dashboard 展示趋势详情。
- 需要总分、等级或健康评分。
- 需要用户在工作台内直接改记录或创建目标。

# 0.2.8 长期趋势工作台

> 本文规划 0.2.8 的个人常态呈现方式。长期趋势是呈现方式，不是产品目的本身。

## 工作台定位

个人常态工作台应让用户看到：

- 我的长期常态是什么。
- 最近窗口和个人常态相比是否有变化。
- 哪些指标缺数据。
- 哪些提示可信度不足。

它不应该制造“达标 / 不达标”的压力。

具体入口、首屏和详情结构见 [工作台信息架构](./workbench-information-architecture.md)。

## 信息层级

建议首屏：

1. 个人基线摘要。
2. 最近 14 / 30 天变化。
3. 记录缺口。
4. 重点指标趋势。
5. 限制说明。

## 卡片候选

第一层卡片：

- 硬度 / 晨勃基线。
- 恢复与疲劳。
- 睡眠与压力。
- 性负荷。

第二层卡片：

- 色情使用上下文。
- 运动与健康习惯。
- 满意度。
- 目标历史。
- 关系沟通上下文，只在 0.2.7 真实落地后显示。

## 偏离提示

状态来源见 [状态判定规则](./state-decision-rules.md)。工作台只展示三类状态，不展示异常、风险等级或 high confidence。

允许：

- “最近 14 天低于你的 90 天个人常态范围，但样本有限。”
- “最近睡眠记录缺失较多，趋势可信度较低。”
- “性负荷较个人常态偏高，建议结合恢复记录一起看。”
- “当前仍在个人常态范围内。”
- “数据不足，先只展示事实。”

禁止：

- “你的硬度异常。”
- “你有健康风险。”
- “你应该减少 / 增加性行为。”
- “你和别人相比偏低。”
- “这是某个伴侣导致的。”

## 入口形态

候选入口：

- Stats / 复盘区域中的个人常态二级入口。
- Dashboard 的轻提示卡片。
- 目标历史中的“长期变化”链接。

不新增顶级导航。

## 空状态

空状态应鼓励继续记录：

- “还没有足够数据建立个人基线。”
- “继续记录 30 天后可以看到更稳定的趋势。”
- “当前只展示事实，不输出趋势判断。”

不使用：

- “数据不足，无法评估你的健康。”
- “缺少记录会影响准确诊断。”

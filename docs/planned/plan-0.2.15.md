# 0.2.15 导航与信息架构整理 v1

> 0.2.15 不做隐私模糊、不做全局搜索 / 快速跳转，也不新增业务模型。它只整理主导航、模块归属、入口命名和跨版本能力的放置规则。

## 当前状态

- 状态：执行草案 / 待实现。
- 前置：0.2.14 记录、复盘与训练入口收束 v1 已收口为执行草案。
- 当前实现线：0.2.6 正在开发。
- 主方向：在 0.2.7 - 0.2.14 继续扩展后，避免入口越来越散、命名越来越乱。
- 当前代码基线：主导航为 `日历 / 状态 / 性生活 / 我的`。
- 默认策略：第一版不增加顶级导航数量，优先整理现有入口内部的信息架构。

## 初步范围

- 明确四个主入口的职责。
- 给 0.2.7 - 0.2.14 的新增能力指定默认归属。
- 收束“状态 / 复盘 / 报告 / 训练 / 趋势 / 经验 / 阶段回顾”等入口层级。
- 收束“性生活 / 关系 / 色情使用 / 女性性健康关怀”等成人健康相关入口关系。
- 规定“我的”只放设置、数据、安全、伴侣/标签等管理类入口，避免变成功能垃圾桶。

## 明确不做

- 不做隐私模式 / 屏幕遮挡 / 隐私模糊。
- 不做全局搜索 / 快速跳转。
- 不新增 schema / migration。
- 不新增业务模型、统计算法、复盘算法或训练能力。
- 不新增顶级导航，除非实现前代码校准证明四入口已经无法承载。
- 不改品牌命名、PWA 名称或应用图标。

## 已定边界

- 第一版保持主导航数量不变。
- 顶级导航只回答“我现在要去哪里”，不承载所有功能说明。
- 同一能力允许有一个主入口和少量上下文快捷入口，但必须落到同一页面或同一数据口径。
- 入口命名必须表达用户意图，避免“报告”这种点击后语义不清的词单独出现。
- 如果某个能力既能从记录进入，也能从复盘进入，必须明确一个主归属，另一个只是快捷入口。
- 0.2.15 是信息架构整理层，不改变 0.2.7 - 0.2.14 的产品能力边界。

## 暂不做

- 不在本窗口改产品代码。
- 不恢复已否决的隐私模糊和全局搜索方向。
- 不把品牌与命名提前为独立版本。

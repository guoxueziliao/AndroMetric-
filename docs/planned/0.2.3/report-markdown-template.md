# 0.2.3 周报 / 月报 Markdown 模板

> 本文负责 0.2.3 刀 56 的报告结构和 Markdown 导出模板。它只定义本地导出的内容边界，不定义导出实现。

## 目标

0.2.3 的周报 / 月报导出用于用户本地复盘，不用于分享。

导出必须：

- 基于 `AdultBehaviorWindowFacts` 和通过 confidence gating 的 insight。
- 明确展示样本量、可信度和限制说明。
- 保持成人健康语境，允许直白词。
- 导出前提醒包含敏感成人健康数据。
- 不默认包含原始备注全文或具体平台名。

## 文件名

建议：

```text
andrometric-adult-review-week-YYYY-MM-DD.md
andrometric-adult-review-month-YYYY-MM.md
```

规则：

- 文件名不默认包含色情、射精、伴侣等露骨词。
- 周报日期使用自然周起始日。
- 月报日期使用 `YYYY-MM`。
- 产品名最终确定后可替换 `andrometric`。

## 导出前提示

导出前必须提示：

```text
该报告包含敏感成人健康数据，可能涉及硬度、性行为、自淫/自慰、色情使用、射精、睡眠和恢复。请只保存在你信任的位置。
```

确认动作：

- 使用明确确认按钮。
- 不提供分享图入口。
- 不默认打开系统分享。

## 内容规则

允许包含：

- 汇总指标。
- 窗口事实。
- 时间范围。
- 事实链路。
- 通过 confidence gating 的弱相关观察。
- 样本量、可信度、限制说明。
- 缺失数据提示。

默认不包含：

- 原始 notes 全文。
- `legacySexRecord.notes` 全文。
- 具体平台名。
- URL。
- 图片 / 视频 / 音频内容本体。
- 伴侣隐私备注。

后续如果允许用户显式选择包含 notes，必须单独确认，不进入 0.2.3 默认导出。

## Insight 展示规则

每条 insight 必须包含：

- 标题或简短摘要。
- `sampleSize`。
- `confidence`。
- `supportingFacts`。
- `limitations`。

不允许：

- 删除限制说明。
- 把 `low` confidence 写成确定结论。
- 输出医学诊断、成瘾判定或强因果。
- 使用羞辱、训诫、道德化语言。

## 缺失数据表达

缺失数据应直接写成记录缺口。

允许：

- “本期睡眠记录较少，睡眠与硬度复盘可信度较低。”
- “本期成人行为事件样本不足，只展示事实，不输出趋势判断。”
- “部分事件存在未补全关联，行为链路可能不完整。”

不允许：

- “你没有认真记录。”
- “数据太少，无法分析你的能力。”
- “缺少记录说明趋势无效。”

## 与 UI 报告的关系

- UI 报告和 Markdown 报告使用同一份事实数据。
- Markdown 可以比 UI 更线性、更适合保存。
- UI 可以使用卡片和图表，Markdown 使用表格和列表。
- 两者都必须展示样本量、可信度和限制说明。

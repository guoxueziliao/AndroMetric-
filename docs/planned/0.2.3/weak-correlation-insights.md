# 0.2.3 弱相关观察指标组合

> 本文负责 0.2.3 刀 54 的第一批弱相关观察。所有观察必须建立在事实聚合和 confidence gating 之后。

## 目标

0.2.3 的弱相关观察只做“记录里看见的共现、窗口比较和趋势提示”，不做原因判定。

允许表达：

- “A 和 B 在同一窗口里一起变多。”
- “A 较高的窗口里，B 的均值更低 / 更高。”
- “当前样本不足，只展示事实。”

禁止表达：

- “A 导致 B。”
- “你有医学问题。”
- “你色情成瘾。”
- “你应该克制。”
- “某种偏好好 / 坏。”

## Insight 输出结构

沿用 [`confidence-rules.md`](./confidence-rules.md) 的 `ReviewInsight`。

0.2.3 每条 insight 必须包含：

- `id`
- `metric`
- `window`
- `sampleSize`
- `confidence`
- `direction`
- `summary`
- `supportingFacts`
- `limitations`

`summary` 可以由 UI 生成，但不能删掉 sampleSize / confidence / limitations。

## 第一批指标组合

### 1. 睡眠与晨间硬度

观察：

- 睡眠记录较少时，只提示缺口。
- 睡眠时长较短窗口的晨间硬度均值。
- 睡眠质量较低窗口的晨间硬度均值。

允许文案方向：

- “睡眠不足后的硬度记录偏低，样本有限，仅供观察。”
- “本期睡眠记录较少，暂不输出睡眠相关判断。”

限制：

- 不说睡眠导致硬度变化。
- 不做医学解释。

### 2. 性活动负荷与疲劳

观察：

- Sex + Masturbation + Ejaculation 次数较高窗口的疲劳均值。
- 边缘控制次数较高窗口的疲劳样本。
- 行为负荷较低窗口的疲劳样本。

允许文案方向：

- “近 14 天射精次数较上个窗口增加，疲劳自评也更高。”
- “本期行为负荷较高，但疲劳样本不足，只展示事实。”

限制：

- 不说“射精过度”。
- 不建议增加或减少性次数。
- 不做挑战式表达。

### 3. Porn use 与次日晨间硬度

观察：

- Porn use 总时长较高窗口的次日晨间硬度均值。
- Porn use 后发生 masturbation / ejaculation 的事件链路数量。
- Porn use 超时或控制感样本与次日硬度样本。

允许文案方向：

- “高色情使用时长窗口的次日硬度均值更低，但样本有限，仅供观察。”
- “本期 Porn use 与 Masturbation 的关联记录较少，链路复盘不完整。”

限制：

- 不说色情导致硬度下降。
- 不输出成瘾判定。
- 不评价内容类型或偏好。

### 4. Masturbation 与满意度 / 恢复

观察：

- Masturbation 次数、射精次数、边缘控制次数与满意度样本。
- Masturbation 后疲劳样本。
- Masturbation 相关硬度样本。

允许文案方向：

- “本期 Masturbation 记录中，满意度样本较少，暂不输出趋势判断。”
- “边缘控制记录增加时，疲劳样本也更高，样本有限，仅供观察。”

限制：

- 不做道德化评价。
- 不把自慰频率写成好 / 坏。

### 5. Sex 与硬度 / 满意度 / 恢复

观察：

- Sex 中硬度样本。
- Sex 后疲劳样本。
- Sex 满意度样本。
- Porn involved 的 Sex 事件数量和硬度 / 满意度样本。

允许文案方向：

- “本期 Sex 记录中的硬度样本不足，只展示事件数量。”
- “有色情内容参与的 Sex 事件样本较少，暂不输出观察。”

限制：

- 不评价伴侣。
- 不做伴侣排名。
- 不把 porn involved 写成好 / 坏。

### 6. 记录质量与可信度

观察：

- 晨间硬度记录天数。
- 睡眠记录天数。
- 成人行为事件缺少 duration / hardness / satisfaction 的数量。
- orphan linked ids 和未补全链路。

允许文案方向：

- “本期记录缺口较多，复盘只展示事实。”
- “补充睡眠和晨间硬度后，后续复盘会更稳定。”

限制：

- 不责备用户。
- 不把记录质量写成能力评分。

## 样本量门槛

0.2.3 初始建议：

- `sampleSize < 3`：`confidence = none`，只展示事实。
- `3 <= sampleSize < 7`：`confidence = low`，只做轻提示。
- `sampleSize >= 7`：可进入 `medium`，但必须看波动和缺失数据。
- 0.2.3 默认不主动输出 `high`。

这些门槛可以在实现时按实际数据形态微调，但不能放宽到样本不足也输出判断。

## Supporting facts

每条 insight 至少包含 2 类事实中的 1 类：

- 窗口事实：当前窗口的数量、均值、样本数。
- 对比事实：当前窗口 vs 上一个同长度窗口。
- 缺失事实：影响可信度的数据缺口。

示例：

- 当前窗口 Porn use 总时长 120 分钟，样本 5。
- 当前窗口次日晨间硬度样本 4，均值 3.2 / 5。
- 上一窗口次日晨间硬度样本 6，均值 3.8 / 5。

## Limitations

常见限制：

- 样本量不足。
- 睡眠记录缺失。
- 晨间硬度记录缺失。
- 成人行为事件未补全时长。
- linked ids 不完整。
- 只显示共现，不代表因果。

## 不做

- 不做医学诊断。
- 不做成瘾判定。
- 不做强因果。
- 不评价成人偏好。
- 不评价伴侣优劣。
- 不做训练建议。
- 不做目标推荐。

训练建议和轻目标进入 0.2.4。

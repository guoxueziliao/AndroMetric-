# 0.2.20 范围与边界

## 核心定位

0.2.20 解决「视觉迁移做了一半」的问题：语义 token 系统已存在并在用，但仍残留旧别名、死文件和不一致的变量命名。

它不是新视觉版本，而是收尾与固化版本。目标是让 token 体系成为唯一事实来源，降低后续视觉维护成本。

## 做什么

- 把 31 处 `brand-*` 引用迁移到语义 token。
- 删除 `tailwind.config.ts` 的 `brand-* / pastel-* / palette-*` 迁移别名。
- 删除死文件 `styles/theme.css` 与旧 `--color-*` 命名。
- 确认 `index.css` 为 token 唯一事实来源，亮 / 暗成对完整。
- 固化「组件只用语义 token」的约定。
- 更正 `CLAUDE.md` 过期的 Tailwind CDN 说明。

## 默认边界

- 只做同义替换，不调色、不改风格。
- 默认不新增 schema / migration。
- 默认不新增主题；多主题死块本版只决定删或留。
- 不新增组件，不重写组件库。

## 不做什么

- 不引入新视觉风格、新配色或新品牌色。
- 不改品牌命名、PWA 名称、应用图标、启动画面。
- 不做无障碍对比度专项。
- 不改健康算法、统计算法、复盘逻辑。
- 不做设计稿对齐 / 像素级还原工程。
- 不做账号、云同步、后端或外部 API。

# 0.2.19 界面与模块使用

## 首页

- 重点关注指标优先进入首页摘要。
- 普通显示指标保持当前信息密度。
- 弱化隐藏指标不应抢占首页首屏。

## 状态页

- 状态卡片按重点关注优先排序。
- 弱化隐藏指标可以折叠到“更多指标”或低优先区域。
- 若指标数据不足，仍要显示“不足以判断”的边界。

## 高级筛选与自定义回看

- 默认筛选维度优先展示重点关注指标。
- 普通显示指标仍可手动选择。
- 弱化隐藏指标不从系统中移除，只降低默认出现频率。

## 洞察与提示

- 重点关注指标优先获得解释、趋势提醒和复盘提示。
- 普通显示指标保持现有提示节奏。
- 弱化隐藏指标只在异常明显、用户主动查看或完整回看时出现。

## 设置入口

- 设置入口应放在长期使用者容易找到的位置。
- 文案要表达“关注重点”，避免让用户误解为医学权重。
- 修改后应立即反映到相关页面，至少在刷新后稳定生效。

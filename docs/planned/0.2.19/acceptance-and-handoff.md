# 0.2.19 验收与交接

## 推荐实现顺序

1. 盘点现有指标 id、设置结构和各模块展示入口。
2. 定义三档分组常量和默认规则。
3. 做个人指标偏好设置界面。
4. 接入首页和状态页排序。
5. 接入高级筛选与自定义回看的默认维度。
6. 接入洞察和提示优先级。
7. 做一致性和回归检查。

## 验收底线

- 用户能把指标设为重点关注、普通显示、弱化隐藏。
- 未配置时，现有页面表现不应明显变化。
- 重点关注会影响首页、状态页和筛选回看默认优先级。
- 弱化隐藏不会删除数据，也不会导致完整记录缺失。
- 页面文案不会把偏好说成医学权重。

## 停线项

- 需要 1-100 精细权重。
- 需要医学评分或风险加权。
- 需要 AI 自动决定用户关注指标。
- 需要新增数据库表或复杂迁移。
- 需要改变核心健康结论算法。
- 需要全局搜索、隐私模糊或 Markdown 导出。

## 开发后同步

- 完成后更新 `docs/roadmap/roadmap.md` 状态。

- 如实现中发现数据结构限制，回写停线原因和下一步建议。

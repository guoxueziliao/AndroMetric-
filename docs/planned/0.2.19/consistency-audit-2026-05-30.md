# 0.2.19 一致性审计 2026-05-30

## 审计结论

0.2.19 可以作为独立实现版本推进。

产品上，它承接 0.2.16 的长期洞察校准和 0.2.17 的自定义回看，让用户把长期使用后的关注重点沉淀为统一偏好。

## 与前置版本关系

- 与 0.2.16：一致。0.2.19 不替代洞察可信度校准，只提供个人关注优先级。
- 与 0.2.17：一致。0.2.19 可影响筛选回看的默认维度，但不做全局搜索。
- 与 0.2.18：一致。实现前应按工作台规则做代码校准和交接记录。

## 与已否决方向关系

- 不恢复隐私模糊 / 屏幕遮挡。
- 不恢复全局搜索 / 快速跳转。
- 不恢复 Markdown 导出。
- 不做 AI 自动判断、医学诊断、成瘾判定或强因果结论。

## 风险检查

- “权重”容易被误解为医学评分，文案必须改写为“关注重点”或“显示优先级”。
- 弱化隐藏容易被误解为删除，界面必须保留完整查看路径。
- 如果设置结构无法承载三档偏好，需要先停线讨论，不直接新增复杂 schema。

## 最终状态

- 0.2.19 已收口为执行草案 / 待实现。
- 第一版范围清楚。
- 已可交给开发会话做实现前代码校准。

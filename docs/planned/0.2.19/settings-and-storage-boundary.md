# 0.2.19 设置与存储边界

## 默认方案

0.2.19 默认作为轻量设置实现。

优先复用现有设置、偏好或本地配置结构，避免为了三档分组新增数据库表。

## 存储内容

- 指标 id。
- 分组：重点关注 / 普通显示 / 弱化隐藏。
- 更新时间可以选做，不作为第一版必要字段。

## 不存什么

- 不存 1-100 数值权重。
- 不存 AI 生成的个人偏好理由。
- 不存医学风险权重。
- 不复制原始记录。

## 迁移边界

- 默认不新增 schema / migration。
- 如果现有设置结构无法承载，需要先停线讨论。
- 如果必须改 `core/storage/db.ts`，必须同步 `core/storage/migration.ts`。
- 任何迁移都不得影响既有历史记录。

## 数据安全

- 弱化隐藏不删除数据。
- 偏好导入导出若已有统一备份能力，可自然包含；不为 0.2.19 单独做导出格式。
- 不接账号、云同步或远程模型。

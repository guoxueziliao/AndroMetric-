# 0.2.18 长期维护与版本执行工作台 v1

> 0.2.18 面向规划和开发执行，不是普通用户功能。它把后续版本、代码校准、开发交接、完成状态和文档状态管理起来，降低 planned 队列越来越长后的维护成本。

## 当前状态

- 状态：执行草案 / 待实现。
- 前置：0.2.17 高级筛选与自定义回看 v1 已收口为执行草案。
- 当前实现线：0.2.7 为下一条候选线。
- 主方向：让开发会话能快速知道当前该做哪个版本、读哪些文档、完成后如何更新状态。
- 用户确认：先作为 docs 工作台，不做应用内 UI；不接 GitHub issue / PR，不做自动发版。

## 初步范围

- 版本队列总览。
- 每个版本的状态、入口文档、实现交接、校准状态和完成状态。
- 开发前校准模板与 0.2.16 对齐。
- 完成后把 planned 移到 completed 摘要的流程。
- 被否决方向和 not doing 列表维护。

## 明确不做

- 不做普通用户功能。
- 不做项目管理 SaaS。
- 不做多人协作、权限、账号或云同步。
- 不做自动生成代码。
- 不做 GitHub issue / PR 集成。
- 默认不新增产品运行时 UI；可以先作为 docs 工作台。

## 已定边界

- 0.2.18 优先是文档和执行流程工作台。
- 如果未来要做应用内维护页，必须重新讨论。
- 它服务开发和规划会话，不服务日常健康记录。
- 不接 GitHub issue / PR，不读取 PR 状态，不自动打 tag / release，不自动推送或发版。
- 0.2.18 排在 0.2.17 之后。

## 暂不做

- 不在本窗口改产品代码。
- 不把 0.2.18 变成外部项目管理系统。

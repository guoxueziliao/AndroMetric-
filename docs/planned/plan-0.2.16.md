# 0.2.16 长期使用体验与洞察校准 v1

> 0.2.16 面向已经长期使用、已经积累数据的用户。它不做新手引导，而是把长期数据质量、洞察可信度、长期用户首页 / 状态页，以及后续开发前的真实代码校准合成一个整理版本。

## 当前状态

- 状态：执行草案 / 待实现。
- 前置：0.2.15 导航与信息架构整理 v1 已收口为执行草案。
- 当前实现线：0.2.6 正在开发。
- 已否决方向：新手引导与首周记录启动 v1，不占用 0.2.16。
- 主方向：让长期用户更容易判断“我的数据哪里可靠、哪里缺口大、哪些洞察值得看、首页应该关注什么”。
- 用户确认：定位、数据质量范围、洞察可信度表达、首页 / 状态页改动深度、开发前代码校准和实现优先级均已确认。

## 初步范围

- 长期数据质量：找出长期缺失字段、记录偏斜、样本不足的维度和无效高频字段。
- 洞察可信度：给复盘、趋势、建议标注“可看 / 谨慎看 / 暂不看”的原因。
- 长期首页 / 状态页：不再以“开始记录”为中心，改为最近变化、待复盘、记录缺口和重要回看。
- 开发前校准：为 0.2.7 及后续实现提供真实代码、真实数据结构、真实入口状态的校准清单。

## 明确不做

- 不做新手引导。
- 不新增 schema / migration，除非实现前校准证明现有数据无法表达基础状态。
- 不新增统计算法大模型。
- 不做医学诊断、风险预测或强因果结论。
- 不做 AI 教练。
- 不做成就、排名、连续打卡或游戏化。
- 不改变 0.2.15 已定主导航结构。

## 已定边界

- 0.2.16 是长期使用后的校准层，不是新功能大版本。
- 优先复用现有 logs、adult behavior、training、state、stats 和 data health 结果。
- 可信度提示必须解释“为什么谨慎”，不能只给红黄绿标签。
- 首页和状态页只突出长期用户当下最该看的内容，不新增完整 Dashboard 重构。
- 开发前校准只产出清单和阻断条件，不替代具体版本的实现刀序。
- 数据质量只读展示，不自动修复、不删除字段、不要求补历史。
- 洞察可信度使用 `可看 / 谨慎看 / 暂不看` 三级，不做单一分数。
- 0.2.16 排在 0.2.15 之后，不插队当前 0.2.6 - 0.2.15 队列。

## 暂不做

- 不在本窗口改产品代码。
- 不把 0.2.16 扩成 0.2.8 - 0.2.12 的替代版本。
- 不恢复新手引导与首周记录启动方向。

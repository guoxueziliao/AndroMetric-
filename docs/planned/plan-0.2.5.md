## 核心决策草案

- 0.2.5 的核心基调：把轻目标变成可回看的长期自我管理系统，但不把产品做成训练游戏或教练平台。
- 0.2.5 解决“多个目标和多个周期之后怎么看”，不解决“如何做完整课程训练”。
- 默认不新增 schema；先用 `TrainingGoal`、`GoalCheckin` 和 0.2.3 / 0.2.4 的复盘结果生成历史视图。
- Training Center 采用轻量工作台形态，优先嵌入复盘 / 统计区域，不新增顶级导航。
- 长期反馈只表达稳定性、恢复、记录质量、关系沟通和目标回顾，不输出医学诊断、成瘾判定或强因果。
- 任何目标历史都不能变成能力分、伴侣排名、性次数挑战、射精次数成就或色情使用时长成就。

## 明确不做

- AI 教练。
- 云端模型或外部 API。
- 完整技巧库。
- 系统训练课程。
- 完整游戏化等级、徽章、连续打卡。
- 性次数 / 射精次数 / 伴侣数量挑战。
- 色情使用时长挑战。
- 伴侣评分 / 排名。
- 分享图。
- 医学诊断、成瘾判定、强因果结论。

## 下一步

0.2.5 当前已形成执行草案。后续进入实现窗口时，应按顺序执行：

1. 刀 1：代码状态与数据校准。
2. 刀 2：目标历史工作台。
3. 刀 3：跨周期进度反馈。
4. 刀 4：入口 UI 与 Dashboard。
5. 刀 5：Safety / Privacy 审计。

如果刀 1 发现 0.2.4 训练目标数据尚未真实落地，应停止 0.2.5 实现，回到 0.2.4 或重新规划。

# 0.2.16 实现交接摘要

## 版本定位

0.2.16 是长期使用体验与洞察校准 v1。

它面向已经长期使用、已经积累数据的用户，不做新手引导。

## 已确认决策

- 四个方向全部进入 0.2.16：长期数据质量、洞察可信度、长期首页 / 状态页、开发前代码校准。
- 默认不新增 schema / migration。
- 数据质量只读展示，不自动修复、不删除字段、不要求补历史。
- 洞察可信度使用 `可看 / 谨慎看 / 暂不看`，必须解释原因。
- 首页 / 状态页轻整理，不重做 Dashboard。
- 0.2.16 排在 0.2.15 后，不插队当前队列。

## 推荐实现顺序

1. 代码校准：确认 stats、state、dashboard、data health、training、review 的真实能力。
2. 数据质量：用现有数据做覆盖率、缺口和低价值字段提示。
3. 可信度：给趋势、复盘、建议和阶段回顾补原因型可信度说明。
4. 首页 / 状态页：调整长期用户重点。
5. 开发前校准模板：固定为后续 planned 版本的进入开发前检查。
6. Safety / Privacy 审计。

## 停线项

- 需要新增数据表或 migration。
- 需要重做 Dashboard。
- 需要新增统计大模型。
- 需要医学风险判断或强因果。
- 需要 AI 教练。
- 需要改变 0.2.15 主导航。
- 需要恢复新手引导方向。

## 验收底线

- 用户能看出哪些数据可靠、哪些只能谨慎看。
- 可信度提示解释样本、窗口和缺口。
- 首页不以新手启动为主。
- 状态页突出长期趋势和可信度。
- 后续版本开发前有代码校准模板。

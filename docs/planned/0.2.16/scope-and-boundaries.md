# 0.2.16 范围与边界

## 核心定位

0.2.16 面向长期用户。

此时产品问题不是“如何开始记录”，而是：

- 长期记录里哪些数据可靠。
- 哪些洞察值得看。
- 哪些结论样本不足。
- 首页和状态页应该优先展示什么。
- 后续版本开发前应该如何避免文档和代码脱节。

## 做什么

- 长期数据质量概览。
- 洞察可信度校准。
- 长期用户首页 / 状态页重点整理。
- 后续版本开发前真实代码校准清单。

## 已确认决策

- 定位：长期使用体验与洞察校准 v1。
- 数据质量：只读概览和提示，不自动修复、不删除字段、不要求补历史。
- 洞察可信度：使用 `可看 / 谨慎看 / 暂不看`，必须解释原因。
- 首页 / 状态页：轻整理，不重做 Dashboard。
- 开发前代码校准：作为 0.2.7 及后续版本开发前的固定门槛。
- 实现优先级：排在 0.2.15 后，不插队当前 planned 队列。

## 默认边界

- 默认不新增 schema / migration。
- 默认不新增顶级导航。
- 默认不新增统计大模型。
- 默认不新增 AI、通知、成就或游戏化。

## 不做什么

- 不做新手引导。
- 不做首周记录启动。
- 不做医学诊断。
- 不做强因果判断。
- 不做风险预测。
- 不替代 0.2.8 - 0.2.12 的长期趋势主线。

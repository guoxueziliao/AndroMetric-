# 0.2.16 长期首页与状态页

## 目标

长期用户打开应用时，不需要再被当成新用户。

首页和状态页应该优先回答：

- 最近有什么变化。
- 哪些记录缺口影响洞察。
- 哪些复盘值得打开。
- 哪些长期趋势正在偏离个人常态。
- 哪些目标、健康项目或观察计划需要处理。

## 首页方向

首页保留记录能力，但主信息应从“开始记录”转向：

- 今日记录入口。
- 最近变化摘要。
- 待复盘提醒。
- 记录缺口提示。
- 进行中的训练 / 健康项目 / 观察计划。

## 首页优先级

1. 今日记录与未完成事项。
2. 近期变化摘要。
3. 待复盘或值得回看的窗口。
4. 影响洞察可信度的记录缺口。
5. 进行中的训练目标、健康项目和观察计划。

## 状态页方向

状态页优先展示：

- 可信度足够的长期趋势。
- 当前最重要的变化。
- 样本不足但值得继续记录的领域。
- 与 0.2.8 - 0.2.12 相关的个人常态、解释、观察、经验和阶段回顾入口。

## 状态页优先级

1. 可信度足够的趋势和变化。
2. 可信度不足但用户应知道的限制。
3. 需要继续记录的关键字段。
4. 与观察计划、经验沉淀、阶段回顾相关的入口。
5. 低优先级历史统计。

## 不做

- 不重做完整 Dashboard。
- 不新增主导航。
- 不把所有 planned 能力提前塞进首页。
- 不制造过度提醒。
- 不用新手空状态替代长期用户摘要。

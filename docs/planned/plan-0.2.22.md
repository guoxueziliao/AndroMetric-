# 0.2.22 设置与偏好整理 v1

> 0.2.22 不改任何设置字段、不动存储结构、不拆组件。它只整理「设置与数据」入口的分区分组、统一分区标题与项样式、明确每组归属，解决设置面板随版本增长后分区杂乱、标题样式不一致的问题。它是信息架构整理层，与 0.2.15（导航整理）、0.2.20（视觉收尾）同系。

## 当前状态

- 状态：执行草案 / 待实现。
- 范围档位：A（纯信息架构整理）。用户已确认不做组件拆分（B）、不做偏好存储收束（C）。
- 前置：0.2.15 已完成；0.2.16 - 0.2.21 已收口为执行草案，0.2.16 为下一候选实现线。
- 主方向：在不改字段、不改存储的前提下，让设置面板分区清晰、命名一致、归属明确。
- 实现前代码校准结论（2026-05-31 实测）：
  - `domain/types/settings.ts` 的 `AppSettings` 仅 8 字段（theme / enableNotifications / notificationTime / hiddenFields / lastExportAt / appLock / backupRetention / backupSchedule），结构克制、未失控；本版**不改字段**。
  - `app/appConfig.ts` 的 `defaultSettings` 与之对应；本版不改默认值。
  - 真正杂乱在 UI：`features/profile/MyView.tsx` 单文件 877 行，设置 Modal（title「设置与数据」）内含分区：外观 / 隐私 / 3 个备份相关分区 / 迁移与备份；另有「标签管理」「虚拟回测实验室」「数据快照」作为独立卡片散落。
  - 分区标题存在两套样式：`text-xs ... text-text-muted uppercase`（设置分区）与 `font-bold text-sm text-text-primary`（卡片标题），不统一。
  - 读写数据流：`AppContent` 持 settings + `onUpdateSettings`，`MyView` / `useProfileMaintenance` 消费；本版不动数据流。

## 初步范围

- 把设置面板分区收束为稳定的几组，明确每项归属。
- 统一分区标题与设置项的视觉样式（复用现有 token，不引入新视觉）。
- 让「标签管理」「数据快照」「迁移与备份」「危险操作」等归入清晰的数据 / 管理分区。
- 消除语义不清或重复的分区标题。
- 不改任何 `AppSettings` 字段、默认值或读写数据流。

## 明确不做

- 不改 `AppSettings` 字段、类型或默认值。
- 不动存储、不新增 schema / migration。
- 不拆分 `MyView.tsx`（B 档，列入暂不做）。
- 不做偏好存储访问层收束（C 档，列入暂不做）。
- 不引入新视觉风格（视觉统一只复用 0.2.20 的语义 token）。
- 不改备份 / 导出的用户可见字段或 JSON 字段名。
- 不新增设置项或新功能。

## 已定边界

- 0.2.22 是设置面板的信息架构整理层，只动分组、归属、标题样式。
- 复用现有 token 与组件，不新增视觉资产。
- 任何「整理」必须是行为等价：设置项功能、读写、默认值不变。
- 若整理过程发现必须拆组件或改存储才能完成，停下来重谈（升级到 B / C 需用户确认）。

## 暂不做

- 不在本规划窗口改产品代码。
- B 档：`MyView.tsx` 组件拆分（JSX-only 拆分需显式同意 + 浏览器验证）。
- C 档：偏好存储访问层收束。
- 不恢复隐私模糊 / 全局搜索。
- 不改品牌命名 / PWA 资产。

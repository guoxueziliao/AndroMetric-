# 0.2.9 范围审计

> 本文记录 0.2.9 在 0.2.8 - 0.2.12 范围审计中的结论。

## 确认状态

保留为单独实现版本，但产品上归属 0.2.8 的解释层。

## 理由

- 0.2.9 的输入强依赖 0.2.8 `PersonalNormalResult`，如果 0.2.8 未落地就无法实现。
- 0.2.9 做的是“变化附近的上下文解释”，本质上是 0.2.8 长期趋势系统的详情能力。
- 如果对用户单独暴露“解释层 / 解释卡系统”，会增加概念负担。
- 0.2.9 不新增 schema，输出也是只读解释卡，适合作为 0.2.8 的派生 UI / 详情模块。

## 保留范围

0.2.9 继续保留：

- 上下文窗口构建。
- 解释卡片。
- 样本量、confidence 和 limitations。
- 第一层上下文：睡眠、压力、性负荷、恢复、色情使用、运动。
- 条件上下文：目标历史、关系 / 周期上下文，只在真实落地后读取。
- 不因果、不诊断、不自动生成目标。

## 已同步方向

0.2.9 文档已按以下方向收口：

- 对外名称不强调“个人变化解释层”，改为“个人常态详情中的上下文解释”。
- UI 入口只放在 0.2.8 个人常态 / 长期趋势详情里，不新增独立工作台。
- Dashboard 只允许提示“有变化可回看”，不单独展示解释卡。
- 0.2.10 如果继续存在，应从 0.2.9 的解释卡进入，但不能让 0.2.9 自动创建观察计划。

## 已确认

- 0.2.9 继续作为单独实现版本。
- 0.2.9 产品上归属 0.2.8。
- 不完全并入 0.2.8 文档集。

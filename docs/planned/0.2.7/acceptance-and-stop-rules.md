# 0.2.7 验收与停下来重谈

> 本文只放版本总验收和停止扩张条件，避免刀序文档过长。

## 版本验收标准

0.2.7 完成时必须满足：

- 关系上下文 v1 模型定稿。
- 女性性健康关怀与周期性生活规划边界定稿。
- 性行为记录可以轻量采集关系上下文。
- 月经 / 生殖健康数据能作为性生活关怀和计划上下文。
- 复盘后可以补充上下文。
- 伴侣资料信息架构更清晰，但不是使用关系上下文的前置条件。
- 新关系上下文优先使用 stable partner id。
- legacy name 记录被兼容而不是强行改写。
- 用户能看到关系沟通类目标的历史事实。
- 所有关系上下文都保持中性，不评分、不排名、不诊断。
- 所有周期上下文都保持不确定性，不诊断、不催促、不替代沟通。
- 伴侣私密备注默认不进入可读导出。

## 停下来重谈

出现以下情况，应停止实现并回到规划：

- 需要大规模重写历史 sex record。
- 需要自动合并伴侣。
- 需要新增关系图谱或关系协议。
- 需要伴侣评分、排名或匹配度。
- 需要分享给伴侣。
- 需要云端或外部 AI。
- 需要用月经 / 排卵窗口自动安排性生活。
- 需要输出女性健康诊断、怀孕结论或排卵确定性。

一句话验收：

> 0.2.7 建立关系上下文与女性性健康关怀系统 v1，让复盘理解情境并支持更好的沟通照顾，而不是评价、控制或诊断伴侣。

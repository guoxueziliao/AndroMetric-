# 0.2.7 候选能力归档

> 本文记录 0.2.7 讨论中暂不进入本版的能力，避免后续反复打开同一问题。

## 完整关系管理系统

不进入 0.2.7。

原因：

- 范围会扩成关系管理产品。
- 会引入关系阶段、协议、冲突、修复等复杂概念。
- 伴侣隐私和评价风险高。

关系上下文系统 v1 只记录情境，不管理关系生命周期。

## 多伴侣关系图谱

不进入 0.2.7。

原因：

- 容易变成伴侣比较和关系网络分析。
- 与本版本“只做上下文”的边界冲突。

## 伴侣评分 / 匹配度

长期不做。

原因：

- 会把成人健康复盘推向伴侣评价。
- 容易制造强因果和关系判断。

## 面向伴侣的分享报告

不进入 0.2.7。

原因：

- 关系上下文和成人健康数据高度敏感。
- 当前产品不鼓励社交传播或对外报告。

## 关系课程或沟通课程

不进入 0.2.7。

原因：

- 内容量过大。
- 容易变成教学平台。
- 需要更严格的内容安全和责任边界。

0.2.7 只允许把“需要后续沟通”记录为上下文或目标，不提供课程内容。

## 性生活排程和强提醒

不进入 0.2.7。

原因：

- 容易把月经 / 排卵窗口变成对女性伴侣的压力。
- 会弱化沟通、同意和伴侣实际感受。
- 与“关怀和共同规划”的基调冲突。

0.2.7 只允许提示舒适度、保护措施、边界和事后照顾，不生成性生活任务。

## 女性健康诊断

不进入 0.2.7。

原因：

- 月经、窗口期、排卵和怀孕状态都需要保留不确定性。
- 产品不能替代医生、验孕或临床判断。
- 诊断化表达会放大误解和关系压力。

0.2.7 只能表达“估算”“记录事实”“建议以伴侣实际感受和专业医疗意见为准”。

## 本地 AI 关系教练

不进入 0.2.7。

长期条件：

- 完全本地运行。
- 不上传隐私数据。
- 输出可解释。
- 可关闭。
- 通过 Safety Rails。

只要不能本地部署，AI 功能不进入计划。

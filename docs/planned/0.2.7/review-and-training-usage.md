# 0.2.7 复盘与训练使用规则

> 本文规定关系上下文进入复盘、建议和目标历史时能说什么、不能说什么。

## 使用原则

关系上下文只作为解释背景。

它可以帮助用户理解：

- 这次记录是否包含沟通和边界信息。
- 女性伴侣当时是否有周期、不适、备孕 / 避孕或恢复上下文。
- 满意度、恢复和关系沟通目标是否有可回看的上下文。
- 哪些信息缺失，导致复盘只能显示事实。

它不能用于：

- 判断某个伴侣好坏。
- 判断某段关系健康与否。
- 把表现变化归因到伴侣。
- 推荐用户选择或远离某个伴侣。
- 用月经、排卵或怀孕估算安排性生活。
- 输出女性健康诊断或怀孕 / 排卵结论。

## 复盘允许输出

允许：

- “本次记录包含关系上下文。”
- “本次记录显示沟通信息较完整。”
- “本次缺少边界 / 偏好上下文，相关复盘只展示事实。”
- “伴侣反馈样本不足，只展示记录事实。”
- “本次记录包含周期 / 性健康关怀上下文，仅作背景提示。”
- “当前目标是暂不怀孕，建议记录保护措施和沟通情况。”
- “本次伴随不适或痛经信息，复盘优先关注舒适度和恢复。”
- “这条记录使用 legacy 伴侣名，尚未关联稳定伴侣 ID。”
- “关系沟通类目标本周期有 check-in。”

## 复盘禁止输出

禁止：

- “某个伴侣让你表现更好。”
- “某个伴侣导致硬度下降。”
- “你和某个伴侣更匹配。”
- “这段关系影响你的性能力。”
- “这个伴侣沟通质量更高。”
- “建议减少 / 增加和某个伴侣的性行为。”
- “窗口期是最佳同房时机。”
- “伴侣今天应该更有性欲。”
- “根据记录判断伴侣怀孕 / 排卵。”

## 训练建议使用规则

关系上下文可以参与轻目标候选：

- 如果多次记录缺少沟通上下文，可以建议“记录质量 / 沟通记录”目标。
- 如果用户标记需要后续沟通，可以建议“关系沟通”目标。
- 如果周期上下文提示不适，只能建议舒适度、恢复和沟通目标。
- 如果目标是暂不怀孕，只能建议保护措施沟通和记录完整性。
- 如果低恢复叠加高关系压力，只能建议恢复、沟通和记录质量，不建议表现提升。

训练建议不得：

- 自动创建目标。
- 将伴侣设为目标对象。
- 输出“改善某个伴侣带来的表现问题”。
- 把关系沟通包装成任务压力。
- 把备孕、避孕或窗口期包装成性生活任务。

## 目标历史使用规则

关系沟通类目标可以展示：

- check-in 次数。
- 是否继续 / 暂停 / 完成。
- 本周期感受。
- 缺少反馈或样本不足。

不能展示：

- 伴侣沟通得分。
- 关系质量分。
- 伴侣比较表。
- 伴侣排名。

## 可信度与样本量

任何关系上下文复盘都必须继承 0.2.3 的样本量和可信度规则：

- 样本不足只展示事实。
- 缺少关系上下文时不输出关系提示。
- 单次记录只能作为事实，不能当作趋势。
- legacy name 未关联 stable id 时，不跨记录聚合。

## 报告与导出

如果后续报告层需要摘要，只能输出低敏摘要：

- “包含关系上下文。”
- “包含周期 / 性健康关怀上下文。”
- “沟通类目标有 check-in。”
- “关系上下文样本不足。”

不得输出原文：

- 伴侣私密备注。
- 沟通记录全文。
- 边界 / 禁忌细节。
- 关系冲突描述。
- 周期不适、出血、验孕或恢复备注全文。

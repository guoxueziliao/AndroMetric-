# 0.2.7 开发刀序

> 本文负责 0.2.7 的候选开发顺序和每刀验收。版本总验收与停止条件见 [`acceptance-and-stop-rules.md`](./acceptance-and-stop-rules.md)。

## 候选刀序

### 刀 1：代码状态与伴侣数据校准

目标：

- 确认 `PartnerProfile` 当前字段。
- 确认 legacy sex record name 字段读写入口。
- 确认 adult behavior `partnerIds` 读写入口。
- 确认 reproductive `partnerId` 读写入口。
- 确认导入导出和 snapshot integrity 当前覆盖。

验收：

- 输出真实代码入口清单。
- 明确是否需要 schema / migration。
- 明确 legacy name 不自动合并。

### 刀 2：关系上下文 v1 模型定稿

目标：

- 定稿关系情境、沟通状态、边界 / 偏好、反馈 / 恢复背景四类字段。
- 定稿周期与女性性健康关怀字段的归属和来源。
- 明确哪些字段进入单次记录，哪些字段只在伴侣详情。
- 明确私密 note 默认不进入可读导出。

验收：

- v1 模型不支持伴侣评分 / 排名。
- 单次记录可以独立填写上下文，不依赖完整伴侣档案。
- 模型字段不制造诊断或关系判断。
- 周期字段只表达关怀、舒适度和沟通需求，不表达性生活要求。

### 刀 3：女性性健康关怀与周期性生活规划

目标：

- 复用现有 `ReproductiveProfile`、`CycleEvent`、`PregnancyEvent` 和 `MenstrualDailySummary`。
- 将经期、预计窗口期、痛经、经量、点滴出血、备孕 / 避孕目标和恢复期作为性生活计划上下文。
- 明确保护措施、舒适度、边界和事后照顾提示。
- 明确所有周期估算都不是医学判断。

验收：

- 不重复造一套女性周期模型。
- 不输出“最佳同房时机”或“应该安排性生活”。
- 不判断伴侣排卵、怀孕或性欲。
- 所有提示都以沟通、同意和伴侣实际感受为前提。

### 刀 4：记录入口与采集体验

目标：

- 在性行为记录中提供轻量关系上下文入口。
- 在性行为记录中显示可跳过的周期 / 性健康关怀提示。
- 在月经 / 生殖健康面板中提供关怀入口，而不是性生活排程任务。
- 支持复盘后补充上下文。
- 伴侣详情保存长期背景。
- 关系沟通目标 check-in 只在真实训练目标落地后接入。

验收：

- 上下文可跳过。
- 快速记录不被复杂表单拖慢。
- 补充上下文回写到对应记录。

### 刀 5：伴侣身份引用契约

目标：

- 新流程优先使用 stable partner id。
- legacy name 兼容读取。
- orphan partner id 进入完整性提示。
- 删除伴侣资料不删除历史记录。

验收：

- 修改显示名不破坏历史引用。
- 同名伴侣不被自动合并。
- 用户能理解哪些记录尚未稳定关联。

### 刀 6：复盘 / 训练使用规则

目标：

- 在成人行为复盘中显示中性关系上下文。
- 在成人行为复盘中显示中性周期 / 性健康关怀上下文。
- 在训练历史中回看关系沟通类目标。
- 允许基于缺失上下文推荐记录质量 / 沟通目标。
- 允许基于不适、保护措施和恢复需求推荐沟通 / 照顾类目标。
- 保留样本不足和可信度限制。

验收：

- 不输出伴侣评分 / 排名。
- 不把表现变化强归因到伴侣。
- 不把周期状态当作表现原因或性生活理由。
- legacy name 记录只显示兼容提示。
- 单次记录不被包装成趋势。

### 刀 7：伴侣详情工作台整理

目标：

- 将伴侣资料整理为基础信息、边界偏好、沟通反馈、健康 / 恢复背景、周期关怀和私密备注。
- 系统生成标签保持中性。
- 用户自填成人内容保留为私密数据。

验收：

- 详情页可扫描。
- 私密备注不会出现在列表卡片和可读导出提示中。
- 不出现系统生成的评价式标签。

### 刀 8：Safety / Privacy 审计

目标：

- 审计记录入口、月经 / 生殖健康面板、伴侣详情、复盘、训练历史、导出、完整性提示和空状态。

验收：

- 禁止文案全部清除。
- CSV / 报告类导出默认不含伴侣私密全文。
- CSV / 报告类导出默认不含周期不适、验孕、出血和恢复备注全文。
- JSON backup 仍完整保留。

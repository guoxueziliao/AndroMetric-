# 0.2.7 记录入口与采集体验

> 本文规划关系上下文系统 v1 如何采集信息。采集必须轻量、可跳过、非评判。

## 采集原则

- 默认可跳过。
- 不要求先维护伴侣资料。
- 不要求女性伴侣必须开启周期追踪。
- 不把关系上下文做成必填。
- 不在快速记录里塞满表单。
- 不用问卷语气审问用户。
- 不把字段命名成伴侣评价。

## 入口一：性行为记录

性行为记录是关系上下文最自然的入口。

建议做一个折叠区：

```text
关系上下文
```

默认只露出 2 - 3 个轻字段：

- 本次沟通：清楚 / 部分 / 未记录。
- 边界确认：已确认 / 部分 / 未记录。
- 事后反馈：有反馈 / 无反馈 / 未记录。
- 如有周期上下文，只显示轻提示，不要求填写。

展开后可以补：

- 关系情境。
- 周期 / 舒适度上下文。
- 本次是否需要后续沟通。
- 私密备注。
- 边界 / 偏好备注。

## 入口二：复盘后补充

用户在复盘时可能意识到上下文缺失。

允许在复盘中提示：

- “这次记录没有关系上下文。”
- “可以补充沟通或边界信息，方便之后回看。”

补充动作应回写到对应记录，而不是只存在复盘 UI。

## 入口三：伴侣详情

伴侣详情保存长期背景：

- 长期偏好。
- 长期边界。
- 沟通方式。
- 需要注意的恢复 / 情绪背景。
- 生殖健康目标和周期追踪开关。
- 私密备注。

伴侣详情不能替代单次记录。本次发生了什么，仍应以本次记录为准。

## 入口四：关系沟通目标 Check-in

如果 0.2.4 / 0.2.5 的训练目标真实落地，关系沟通类目标的 check-in 可以补充：

- 是否完成沟通。
- 是否有反馈。
- 是否需要调整目标。
- 本周期感受。

这些只用于目标历史和复盘提示，不生成伴侣评价。

## 入口五：月经 / 生殖健康面板

当前已经有月经和生殖健康内容。0.2.7 可以在此基础上增加关怀入口：

- 今日是否处于经期 / 预计窗口期。
- 是否有痛经、经量、点滴出血或不适记录。
- 当前目标是共同备孕、暂不怀孕、已怀孕或恢复期。
- 是否需要提醒性生活前沟通保护措施、舒适度和边界。

这些入口只做提示，不生成性生活排程任务。

## 快速记录边界

快速记录只允许：

- 不显示关系上下文。
- 或显示一个“稍后补充关系上下文”的入口。

快速记录不应要求用户当场填写详细关系信息。

## 空状态与提示

允许：

- “可以稍后补充关系上下文。”
- “今天有周期上下文，性生活前优先沟通舒适度。”
- “没有关系上下文时，复盘只显示事实数据。”
- “这部分是本地私密记录。”

不允许：

- “补全关系信息才能得到准确分析。”
- “记录伴侣反馈以评估伴侣质量。”
- “填写越多越能判断关系是否健康。”
- “窗口期适合安排性生活。”

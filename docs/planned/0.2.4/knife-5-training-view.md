# 0.2.4 刀 5 性表现训练视图

> 本文只规划刀 5：在成人行为复盘后承载训练建议、当前目标和表现稳定性视图。它不新增独立 Training 页面。

## 状态

- 所属版本：0.2.4。
- 当前阶段：待刀 4 完成后执行。
- 前置：[`knife-4-goal-creation-checkin.md`](./knife-4-goal-creation-checkin.md)。
- 实现边界：复盘后训练区、Dashboard 轻提示、表现指标展示边界。

## 目标

刀 5 完成后，用户可以在成人行为复盘中看到下一步建议、当前轻目标、到期 check-in 和表现稳定性摘要。

必须满足：

- 成人行为复盘作为主入口。
- Dashboard 只做轻提示。
- 不新增独立 Training 页面。
- 硬度稳定和恢复优先。
- 做爱时长不做最高纪录。
- 射精控制不做挑战榜。

## 信息架构

复盘后训练区建议包含：

1. 本周关注。
2. 规则建议。
3. 候选轻目标。
4. 当前 active 目标。
5. 到期 check-in。
6. 表现稳定性摘要。

移动端优先级：

1. 到期 check-in。
2. 高负荷 / 低恢复提示。
3. 当前 active 目标。
4. 1 - 3 条建议。
5. 表现稳定性摘要。

## 表现维度

展示优先级：

1. 硬度稳定。
2. 事后恢复。
3. 射精控制观察。
4. 做爱时长稳定性。
5. 满意度。

展示规则：

- 每个指标显示样本量或缺失提示。
- 表现指标必须和恢复、满意度、样本量一起展示。
- 低恢复时，恢复提示优先于表现提升。
- 样本不足时显示继续记录，不输出判断。

## Dashboard 轻提示

Dashboard 允许展示：

- 当前 active 目标数量。
- 最近一个 active 目标。
- 即将到期的 check-in。
- 低恢复 / 高负荷提示。
- 一条本周关注。

Dashboard 禁止展示：

- 完整目标列表。
- 完整建议列表。
- 训练课程。
- 任务面板。
- 连续打卡。
- 性次数、射精次数或色情使用时长挑战。

## 禁止指标形态

不展示：

- 最高做爱纪录。
- 本月性次数挑战。
- 射精次数成就。
- 色情使用时长累计成就。
- 最佳 / 最差伴侣。
- 伴侣评分或排名。

不表达：

- 越久越好。
- 更多性行为更好。
- 更多射精更好。
- 某个伴侣让你表现更好 / 更差。
- 低恢复就是失败。

## 空状态

无建议时：

- 展示继续记录。
- 展示缺失数据。
- 不制造焦虑。

无目标时：

- 可展示候选轻目标。
- 不强迫开始。
- 不展示任务空列表。

无 check-in 时：

- 展示暂无到期回顾。
- 不催促。

## 测试边界

必须覆盖：

- 移动端首屏不被目标和建议挤爆。
- 低恢复时恢复提示优先。
- 样本不足时不展示表现判断。
- Dashboard 不出现完整训练中心。
- 不出现最高纪录、排行榜、挑战或伴侣评分。
- 候选目标不自动创建。

## 非目标

- 不新增独立 Training 页面。
- 不做完整目标历史。
- 不做训练课程。
- 不做技巧库。
- 不做游戏化等级。

## 交接给刀 6

刀 6 可以在训练区和建议中接入关系上下文，但仍不能做完整关系管理系统。

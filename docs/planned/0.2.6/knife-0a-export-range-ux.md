# 0.2.6 插刀 0A 导出范围 UX

> 本文只规划导出范围表达：默认全部导出，日期区间只是可选筛选。

## 现状

- 导出弹窗直接展示开始日期和结束日期。
- 用户会以为必须选择日期。
- 需求是全部导出。

## 目标

- 默认导出全部数据。
- 日期筛选必须是用户主动开启。
- UI 明确展示当前范围是全部数据还是日期筛选。
- JSON / 加密 JSON 继续作为完整迁移格式。

## 建议状态

可在 `ExportOptions` 增加概念性状态：

- `rangeMode: 'all' | 'date'`

规则：

- `all`：忽略 `startDate` / `endDate`。
- `date`：按 `startDate` / `endDate` 筛选。
- 从 `date` 切回 `all` 时，可以保留日期输入值但不生效；也可以清空，按实现便利决定。

## UI 建议

导出弹窗的日期区块改成：

1. 范围选择：全部 / 按日期。
2. 默认选全部。
3. 只有按日期时显示或启用开始 / 结束日期。
4. 区块文案提示“JSON 是完整备份，CSV 是可读导出，Markdown 已取消”。

## 格式验收

JSON：

- 默认导出完整数据。
- 不被日期区间误导。
- 加密 JSON 同规则。

CSV：

- 默认导出全部可读 CSV。
- 按日期时只筛选有日期语义的数据。

Markdown：

- 导出入口移除或隐藏。
- 不参与日期范围筛选。

## 文案边界

允许：

- “当前范围：全部数据。”
- “按日期筛选只影响本次可读导出 / 选择性导出。”
- “JSON 是完整备份。”

禁止：

- “请选择开始和结束日期。”
- “不填日期无法导出。”
- “Markdown 可替代完整备份。”

## 验收

- 打开导出弹窗默认是全部。
- 用户不填日期也能导出。
- 切到按日期后日期才生效。
- 切回全部后日期不影响导出。
- JSON / CSV 覆盖默认全部导出和日期筛选测试。
- Markdown 导出入口不可见或不可用。

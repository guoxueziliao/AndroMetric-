# 0.2.6 候选能力归档

> 本文记录 0.2.6 讨论中暂不进入本版的能力，避免后续反复打开同一问题。

## 云同步

不进入 0.2.6。

原因：

- 与本地优先和无账号基线冲突。
- 会引入密钥、账号、冲突、服务器和合规问题。
- 当前问题可以先用本地 backup / preview / integrity 解决。

## 账号系统

不进入 0.2.6。

原因：

- 当前产品定位不需要账号。
- 账号会把私密成人健康数据推向服务端治理。

## CRDT / OT

不进入 0.2.6。

原因：

- 当前仍以单设备和手动导入为主。
- 多设备实时协作不是核心场景。
- 工程成本远高于当前收益。

## 自动内容识别

不进入 0.2.6。

原因：

- 不做色情内容本体。
- 不分析用户文件。
- 不引入外部模型。

## 医生报告 / 对外报告

不进入 0.2.6。

原因：

- 0.2.6 是数据安全版本，不是医学报告版本。
- 对外报告会带来诊断、责任和隐私风险。

## 分享图

不进入 0.2.6。

原因：

- 成人健康数据不应被鼓励社交传播。
- CSV 也必须强调本地保存，而不是分享；Markdown 导出已取消。

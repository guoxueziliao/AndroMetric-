# 0.2.6 恢复演练与信心反馈

> 本文负责备份写后读、恢复演练、只读预检和用户可理解的信心提示。

## 目标

用户不应该只在真正覆盖数据时才知道备份是否可用。

0.2.6 可以提供：

- 备份写后读确认。
- 只读恢复预检。
- 导入前完整性报告。
- 备份信心提示。

## 只读恢复预检

当前矩阵见 [`knife-4-current-readonly-preflight-matrix-2026-05-28.md`](./knife-4-current-readonly-preflight-matrix-2026-05-28.md)。

只读恢复预检指：

- 读取备份文件。
- 解密或解析。
- migration 到当前 dataVersion。
- 生成 import preview。
- 运行 integrity check。
- 不写入 IndexedDB。

预检应输出：

- 可导入 / 不可导入。
- dataVersion 状态。
- 主要数据数量。
- warnings。
- blockers。

## 备份写后读

备份创建后可以检查：

- JSON 可重新解析。
- dataVersion 存在。
- 关键数组存在。
- 关键数量和写入前一致。
- 加密备份可解密到同一结构。

写后读不应该：

- 改写用户数据。
- 自动上传。
- 自动删除旧备份。

## 信心提示

允许展示：

- “最近一次备份已通过结构检查。”
- “此备份可读取，但存在 2 条关联 warning。”
- “此文件来自更新版本，当前应用不能导入。”

避免展示：

- “绝对安全。”
- “永远不会丢失。”
- “已自动修复全部问题。”

## 和现有功能的关系

0.1.1 / 0.1.3 已经做过：

- 导入预览。
- 安全快照。
- 自动备份。
- quota 监控。
- 保留策略。

0.2.6 不重复做设置项，而是把复杂数据放进预检和信心反馈。

## 验收

- 用户可以在不写入当前数据库的情况下预检备份。
- 预检能显示 dataVersion 和数据数量。
- 预检能显示成人行为事件 warning；训练数据 warning 只在真实 store / 类型已落地后展示。
- 高版本备份不能写入。
- 信心提示不承诺绝对安全。

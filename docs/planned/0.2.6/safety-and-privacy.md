# 0.2.6 Safety / Privacy Rails

> 本文负责 0.2.6 数据安全能力的隐私、安全、分享和外部分析边界。

## 核心原则

数据安全能力不能变成数据外流能力。

0.2.6 必须继续坚持：

- 本地优先。
- 用户主动导出。
- 无账号。
- 无云同步。
- 无外部分析。
- 不默认暴露敏感 notes。

## 导出隐私

导出时必须提醒：

- JSON backup 是完整敏感数据。
- 成人行为事件可能包含性健康记录。
- training goals 和 check-ins 可能包含私密自我管理信息。
- CSV 仍可能包含敏感摘要。
- Markdown 导出已取消，不应作为敏感数据外流入口保留。

默认不导出：

- 色情内容本体。
- URL。
- 缩略图。
- 伴侣私密备注全文。
- notes 全文。

如未来允许包含 notes 全文，必须由用户显式选择。

## 分享边界

明确不做：

- 分享图。
- 社交传播。
- 云端链接。
- 一键发给他人。
- 医生报告自动生成。
- 外部 AI 分析包。

允许：

- 本地下载 JSON backup。
- 本地下载 CSV。
- 用户自己决定如何保存文件。

## 文案边界

允许：

- “此文件包含敏感成人健康数据，请妥善保存。”
- “CSV 不是完整备份。”
- “导入前将先预览数据结构和风险。”

禁止：

- “上传以获得更完整分析。”
- “分享给伴侣看看你的表现。”
- “用 AI 自动判断问题。”
- “此备份绝对安全。”

## 验收

- 所有导出入口都有敏感数据提示。
- CSV 不默认包含 notes 全文。
- Markdown 导出入口已移除或隐藏。
- 不出现云同步、账号、分享或外部 AI 引导。
- 高风险导入 / repair 操作需要确认。
- 任何 repair 不静默删除敏感历史。

# 0.2.19 个人指标自定义与权重 v1

> 0.2.19 让长期用户决定哪些指标更重要、哪些指标只需普通展示、哪些指标可以弱化或隐藏。它是长期趋势、筛选回看和洞察校准之后的个人关注重点层，不是新的评分系统。

## 当前状态

- 状态：执行草案 / 待实现。
- 前置：0.2.16 长期使用体验与洞察校准、0.2.17 高级筛选与自定义回看、0.2.18 长期维护与版本执行工作台均已收口为执行草案。
- 主方向：把个人关注指标沉淀为统一偏好，影响首页、状态页、筛选回看和洞察排序。
- 用户确认：第一版只做轻量分组：重点关注 / 普通显示 / 弱化隐藏；不做 1-100 分精细权重。

## 初步范围

- 提供个人指标偏好设置。
- 指标分为重点关注、普通显示、弱化隐藏三档。
- 首页和状态页优先展示重点关注指标。
- 高级筛选与自定义回看优先提供重点关注指标。
- 洞察和提示文案优先解释重点关注指标，弱化低关注指标。
- 保持所有原始记录完整，不删除、不改写历史数据。

## 明确不做

- 不做 1-100 数值权重。
- 不做医学权重或健康风险评分。
- 不做 AI 自动判断哪些指标重要。
- 不改变核心健康结论算法。
- 不把弱化隐藏理解为删除数据。
- 不做账号、云同步或跨设备偏好同步。

## 已定边界

- 0.2.19 是个人显示和排序偏好，不是新分析模型。
- 第一版只做三档轻量分组。
- 默认不新增 schema / migration；若现有设置结构无法承载，再停下来讨论。
- 偏好只影响展示、排序、默认筛选维度和提示优先级。
- 任何健康解释仍必须保留数据不足、非医学诊断和低确定性的边界。

## 暂不做

- 不在本窗口改产品代码。
- 不扩展为全局搜索。
- 不恢复隐私模糊。
- 不恢复 Markdown 导出。

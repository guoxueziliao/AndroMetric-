# 0.2.12 刀 1 代码校准与来源确认

> 本文定义 0.2.12 实现前的第一刀。目标是确认阶段回顾能读取哪些真实来源。

## 必查来源

- 0.2.8 personal normal 实际结果。
- 0.2.9 explanation cards 实际结构。
- 0.2.10 observation plans 实际结构。
- 0.2.11 experience cards 实际结构。
- 0.2.3 review facts / review engine。
- Stats / Dashboard / History 入口。

## 输出清单

刀 1 必须输出：

- 可读取来源列表。
- 每个来源的字段名和窗口口径。
- 每个来源是否已真实落地。
- 未落地来源的隐藏策略。
- 是否需要新增 schema，默认应为否。

## 校准规则

- 以真实代码为准，不以规划文档为准。
- 未落地来源不做 mock 章节。
- 字段名和类型不猜测。
- 只读派生，不落库。
- 若必须新增 store，回到规划重谈。

## 验收

- 有一份实现窗口可用的来源矩阵。
- 0.2.8 - 0.2.11 任一缺失时，回顾仍能降级。
- 没有新增 IndexedDB store。
- 没有新增 backup / export 字段。

## 停线条件

- 0.2.8 个人常态未落地且无法构建基础月度事实。
- 阶段回顾需要持久化。
- 需要导出阶段回顾。
- 需要 AI 总结或自动结论。

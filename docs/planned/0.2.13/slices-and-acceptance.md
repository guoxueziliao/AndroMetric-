# 0.2.13 开发刀序与验收

> 本文只记录 0.2.13 的最终实现顺序和验收底线，详细边界见相邻专题文档。

## 刀序

1. 代码校准
   - 确认 `DailyLog.supplements`、每日记录健康区、生殖 / 备孕面板和 0.2.6 数据安全扩展点。
2. 数据模型与 migration
   - 新增健康项目、计划和执行记录；旧 `supplements` 继续作为 legacy daily fact。
3. 项目管理
   - 创建、编辑、暂停、恢复、结束、归档。
4. 每日记录接入
   - 按计划生成今日待记录项；支持完成、跳过和备注。
5. 数据安全接入
   - JSON backup、import preview、snapshot integrity 覆盖新数据。
6. Safety / Privacy 审计
   - 清除医疗建议、剂量安全判断、服药提醒和自动推荐风险。

## 验收底线

- 用户可以创建补剂项目和健康习惯项目。
- 用户可以设置每日、隔日、每周若干天、连续 N 天、暂停 / 恢复和手动结束。
- 补剂支持可选剂量、单位、服用时间标签和备注。
- 每日记录健康区显示今日待记录项，并提供管理入口。
- Dashboard 只显示今日轻提示。
- 项目详情展示执行历史、近 7 / 30 天完成次数和跳过次数。
- 旧 `supplements` 记录继续可读，且不被自动迁移成长期计划。
- import preview 区分 legacy supplements 和新健康项目数据。
- JSON backup 覆盖新项目、计划和执行记录。
- 不出现医疗建议、剂量安全判断、服药提醒、成就、评分、排名或补剂效果分析。

## 停下来重谈

- 需要把用药做成计划。
- 需要提醒通知。
- 需要自动推荐补剂或周期。
- 需要剂量安全判断。
- 需要根据旧补剂记录自动推断长期计划。
- 需要账号、云同步或外部 API。

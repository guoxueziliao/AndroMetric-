
# Hardness Diary · Core Governance Documents

本目录包含本项目中 **不可轻易破坏的核心治理文档**。

这些文档不描述功能需求（PRD），
而定义系统在面对 **数据、标签、语言** 时的长期态度与边界。

它们的存在目的是：
- 防止短期设计冲动破坏长期数据可信度
- 为未来版本设立清晰的不可回退底线
- 确保个人数据在多年后仍然可理解、可回溯

---

## 📜 文档索引

### 1. 数据与交互宪章
**data-and-interaction-charter.md**

定义本项目在“数据层面”的最高原则：
- 什么是事实，什么是解释
- 系统与用户的责任边界
- 对不完美、矛盾、老化数据的官方态度

👉 所有涉及 **数据结构、自动修复、系统干预、历史数据处理** 的修改，
必须先阅读此文档。

---

### 2. Tag 系统规则与反熵规范
**tag-system-rules.md**

定义 Tag 系统的结构性约束：
- 六维 Tag 结构及其唯一职责
- 标签创建、清洗、合并、校验规则
- 防止 Tag 系统长期熵增的机制

👉 所有涉及 **标签维度、命名、校验逻辑、体检机制** 的修改，
必须先阅读此文档。

---

### 3. UI 文案不可破坏原则
**ui-copy-rules.md**

约束所有“面向数据的 UI 文案”：
- 防止隐喻、中二、情绪化语言污染数据结构
- 保证长期可读性与低理解成本

👉 所有涉及 **字段标题、标签名、数据展示文案** 的修改，
必须先阅读此文档。

---

## 🛑 变更声明

这些文档不是 PRD，
也不随功能版本频繁调整。

任何修改应满足以下条件：
- 已进行真实、长期使用
- 问题来自使用摩擦，而非设计表达欲
- 修改能明确提升长期可理解性

否则，应保持不变。

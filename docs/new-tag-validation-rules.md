
# v0.0.6 · 新建标签即时校验规则 & 提示文案（Final）

## 0. 本规范的定位

本规范只解决一件事：
**在“标签进入系统的那一刻”，最大限度阻止未来的数据熵增。**

特点：
*   即时校验
*   只拦高风险
*   只提示，不替用户做决定

---

## 1. 标签新建的基本前提（前置条件）

### 1.1 新建入口限制
*   ❌ **记录页面**：禁止新建标签
*   ✅ **标签管理页面**：唯一新建入口

原因：任何新标签都可能影响历史统计，必须在“管理语境”下创建。

---

## 2. 强制字段校验（Blocking，必须通过）

**这些校验不通过则无法创建标签。**

### 2.1 非空校验
*   规则：去除首尾空格后，长度必须 ≥ 1
*   提示：`标签名称不能为空`

### 2.2 长度限制
*   规则：最大长度 20 个字符
*   提示：`标签名称过长，建议简化（≤20字）`

### 2.3 重复校验
*   规则：忽略大小写、首尾空格
*   提示：`已存在同名标签，请直接使用或在标签管理中合并`

### 2.4 非法字符校验
*   禁止字符：`/ \ | → => & + = @ # *`
*   原因：强烈暗示复合标签或结构化语义
*   提示：`标签名称不应包含连接符或结构符号，请拆分为多个标签`

### 2.5 时间 / 数值模式拦截
*   禁止模式：
    *   包含时间：`09:00`、`23点`
    *   包含数值单位：`350ml`、`3次`、`5分钟`
    *   包含纯数字或明显数值含义：`2025`、`18岁`
*   提示：`标签不应包含时间或数值，请使用字段记录`

---

## 3. 语义风险校验（Non-blocking，仅提示）

**这些校验允许创建，但必须提示风险。**

### 3.1 复合语义检测（最重要）
*   触发条件：同时命中两个 XP 维度关键词，或包含「身份+行为」等结构。
*   示例：`人妻出轨`、`制服诱惑`、`强制高潮`、`黑丝足交`
*   提示：`该标签可能同时包含多个语义维度，建议拆分为多个标签以便长期统计`

### 3.2 维度不清晰检测（XP 专用）
*   触发条件：无法明确归入某一 XP 维度
*   示例：`痴女`、`中年女性`、`自拍`
*   提示：`该标签维度不够明确，后续可能影响统计准确性`

### 3.3 非标签语义提示
*   触发条件：更像状态 / 行为结果 / 情绪
*   示例：`很爽`、`一般`、`满足`、`后悔`
*   提示：`该内容更像状态或结果，通常不建议作为标签`

---

## 4. 标签类型专属校验规则

### 4.1 症状标签（symptom）
*   规则：必须符合“医学可描述性”，禁止出现原因、情绪、行为。
*   高风险示例：`压力大`、`心情不好`、`熬夜`
*   提示：`症状标签应描述身体不适本身，而非原因或感受`

### 4.2 事件标签（event）
*   规则：应描述“发生过的事”，不应描述身体状态或性刺激。
*   高风险示例：`头痛`、`巨乳`、`口交`
*   提示：`事件标签应描述当天发生的事情，而非状态或偏好`

### 4.3 XP 标签（xp）
*   规则：必须能明确归入六个 XP 维度之一。
*   高风险示例：`人妻出轨`、`师生调教`
*   提示：`XP 标签应只描述一个刺激维度，建议拆分以保持清晰`

---

## 5. 提示交互规范

| 等级 | 是否阻断 | 使用场景 |
| :--- | :--- | :--- |
| **错误 (Error)** | ✅ 阻断 | 非法 / 高风险必错 |
| **警告 (Warning)** | ❌ 不阻断 | 语义风险 |
| **信息 (Info)** | ❌ 不阻断 | 建议性提醒 |

**原则**：不出现“你不该”，不出现“系统已自动修改”，始终把决定权交给用户。


# v0.0.6 · XP 标签统计口径 & 去噪规则（Final）

## 0. 目标与边界

**目标**
*   XP 统计必须回答“偏好是什么”，而不是“那天发生了什么”。
*   统计结果必须可复现。
*   抗噪：少量脏标签不会毁掉整体趋势。

**边界**
*   只针对 XP 标签（6 维度）。
*   不做系统自动修复，只做统计层面的去噪与提示。

---

## 1. XP 统计的最小单位 (Unit of Analysis)

### 1.1 Record-level（记录级，默认推荐）
*   **规则**：一条自慰/性爱记录中，某标签出现过就记 1 次（不管点了几次）。
*   **优点**：抗重复点击噪声。
*   **适用**：所有 XP 偏好排行、维度分布图表。

### 1.2 Day-level（日期级）
*   **规则**：同一天内多个记录合并去重。
*   **适用**：长期趋势图、热力图、与睡眠/压力的相关性分析。

---

## 2. 去重规则 (Dedup Rules)

### 2.1 同一记录内去重（强制）
XP 标签数组内允许重复输入，但统计必须 `unique()` 后再算。

---

## 3. 维度统计口径 (Dimension Metrics)

### 3.1 维度占比 (Distribution)
*   **口径**：Record-level
*   某维度只要在记录里出现至少 1 个标签，则该记录对该维度贡献 1。

### 3.2 维度丰富度 (Diversity)
*   **口径**：Unique-tag count
*   不看次数，只看“你这个维度有多少不同标签”。

---

## 4. XP 标签去噪规则 (核心)

### 4.1 复合标签的统计处理
*   **定义**：同时属于多个维度的标签（如“人妻出轨”）。
*   **规则**：**不得进入“维度统计”主口径**。
*   **处理**：允许在频次榜出现，但禁止参与雷达图计算。

### 4.2 低频标签去噪
*   **规则**：长期极低频（≤1次）的标签在 Top 榜单中折叠。

### 4.3 “非 XP 语义”处理
*   **规则**：判定为“状态/结果词”（如“很爽”）的标签，从主统计排除。

---

## 5. UI 约束
任何 XP 统计图表必须在 UI 上显示“口径说明”。
> 示例：“统计口径：按记录去重 (Record-level) • 已自动过滤噪声”

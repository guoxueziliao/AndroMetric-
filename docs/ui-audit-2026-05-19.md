## 3. 跨切面主题(按问题数排序)

### 主题 A:**死代码与不可达功能**(9 项)
SidebarNav、ThemeSettings、HardnessChart、useTheme 主题色、trendComparison setter、SimulationLab(应隐藏)、AnalysisView 跳转按钮、ContentItem "去选择"、PWA 更新对未安装用户不显示

→ 0.0.7 应先做"清扫":明确删除 / 接入 / 隐藏。少 9 处困惑就少 9 次"这是干嘛的"。

### 主题 B:**触屏不可用**(11 处)
hover-only(5 处) + 触控目标过小(6 处)。在 mobile-first 应用里这是最致命的硬伤。

→ 0.0.7 必修。规则:**任何 hover-only 控件改成 long-press 或 visible button;任何 < 44px 的目标扩到 44px**。

### 主题 C:**导航与 IA 混乱**(8 项)
AnalysisView 三层嵌套、BottomNav vs SidebarNav 同视图三个名字、"分析" tab 二次嵌入 StatsView、"我的" tab 装了 7 类功能、Dashboard 三个 tab "日/周/月" 语义重叠、Reproductive 入口隐蔽、TagHealthCheck 名字与 MyView 体检冲突、Backup 双入口

→ 0.0.7 必修。IA 不清,所有具体修复都白费。

### 主题 D:**输入时间/默认值碎片**(6+ 处)
6 套时间输入并存;Alcohol 默认 20:00、Beverage 无控件、Sleep 无智能默认、Masturbation auto-fill 20+ 字段、wokeWithErection 默认 true(无法区分"没碰过")

→ 0.0.7 强烈建议修。修一遍后所有录入摩擦显著下降。

### 主题 E:**长表单 + 7 个 modal 缺合并机会**
LogForm 7-10 屏;SexRecordModal 816 行 130 chip;MasturbationRecordModal 554 行 10 区块;Alcohol+Beverage 可合并;Sleep+Nap 大量重叠;HealthSection 死代码

→ 0.0.7 可做"窗口期"的几个合并:Alcohol+Beverage → DrinkModal;Sleep+Nap 共享 section;删 HealthSection。**不做激进 state-hook 拆分**(memory 里你说过这个要慎重)。

### 主题 F:**空态/反馈缺失**
Dashboard、StatsView、CalendarHeatmap、LogHistory 都没有真正的空态;LogForm 保存无 loading/success;BackupSettings 反馈不消失;Toast 不堆叠;`window.confirm/alert` 在 PWA 多处使用

→ 0.0.7 必修触屏关键路径(LogForm 保存、ongoing 完成、Delete);其余可以 0.0.8。

### 主题 G:**数据真实性问题**
SexRecordModal 注入 "通宵硬仗"、Masturbation location 自清、`/吞精` 复合值保存丢失、touchedPaths 14 处手拼、auto-fill 20+ 字段被当真实数据

→ 0.0.7 必修(数据可信度的前置)。如果数据被偷偷改了,所有洞察都是脏的。


# ContentItem 数据体检规范

（ContentItem Health Rules · Final）

**适用版本**：v0.0.6
**状态**：Final / 稳定层

## 1. 文档目的

本规范用于定义：
- 什么是一个**“健康的 ContentItem”**
- 哪些问题需要被检测
- 检测后如何通过 Notice 系统 呈现
- 系统 **不应该** 做什么（非常重要）

---

## 2. ContentItem 的角色定位

### 2.1 定义
ContentItem 是一次自慰 / 性活动中“一个具体素材来源”的抽象单元。
它描述的是：
- 看了 / 用了 / 回忆了 什么内容
- 内容来自 哪里
- 内容大概是 什么特征

### 2.2 核心原则（不可破坏）
- 一个 ContentItem = 一个素材来源
- ContentItem **不自动合并**
- ContentItem **不推断用户意图**
- ContentItem 可以 **信息不完整**
- ContentItem 永远是 **用户事实**，不是系统判断

---

## 3. ContentItem 基础字段分组（逻辑视角）

### 3.1 身份字段（Identity）
| 字段 | 说明 |
| :--- | :--- |
| `id` | 唯一标识（必须） |
| `createdAt` | 创建时间 |
| `source` | 创建来源（manual / migrated） |

### 3.2 类型字段（Type）
| 字段 | 说明 |
| :--- | :--- |
| `type` | 内容形式（video / image / text / memory / fantasy） |

### 3.3 来源字段（Source）
| 字段 | 说明 |
| :--- | :--- |
| `platform` | 平台枚举或自定义 |
| `platformName` | 自定义平台名（当 platform=other） |

### 3.4 描述字段（Description）
| 字段 | 说明 |
| :--- | :--- |
| `title` | 标题 / 备注 |
| `actors` | 主演 / 对象（数组） |

### 3.5 标签字段（Preference）
| 字段 | 说明 |
| :--- | :--- |
| `xpTags` | 性癖标签数组（仅 XP） |

---

## 4. 体检规则总览

**所有规则 只做检测，不做修复**

### 5. 强一致性规则（Error 级）

#### C-E1｜缺少 ContentItem.id
- **说明**：无法定位、编辑或迁移
- **严重级别**：`error`
- **系统行为**：
    - 允许展示
    - 禁止编辑（可选）
    - 允许自动修复：✅（生成新 id）
- **Notice**：`素材缺少唯一标识`

#### C-E2｜ContentItem 不是对象或结构损坏
- **说明**：数据结构异常
- **严重级别**：`error`
- **系统行为**：
    - 只读展示
    - 禁止写入
    - 允许自动修复：❌

---

### 6. 重要完整性规则（Warn 级）

#### C-W1｜未选择内容类型（type）
- **说明**：无法判断素材形态
- **严重级别**：`warn`
- **影响**：分类统计失效、回顾页难以识别
- **Notice**：`未选择素材类型`

#### C-W2｜type ≠ memory / fantasy 且 platform 为空
- **说明**：素材无来源
- **严重级别**：`warn`
- **影响**：平台统计失效
- **Notice**：`未选择来源平台`

#### C-W3｜platform = other 但 platformName 为空
- **说明**：来源模糊
- **严重级别**：`warn`
- **影响**：来源聚合失败
- **Notice**：`平台名称为空`

#### C-W4｜同一 ContentItem 绑定多个平台（历史遗留）
- **说明**：违背“一素材一来源”
- **严重级别**：`warn`
- **影响**：后续迁移困难
- **Notice**：`一个素材包含多个平台`

---

### 7. 低风险优化规则（Info 级）

#### C-I1｜title 与 actors 同时为空
- **说明**：素材描述不足
- **影响**：回顾困难
- **允许忽略**：✅
- **Notice**：`未填写素材备注`

#### C-I2｜xpTags 为空
- **说明**：未标注性癖
- **影响**：不参与偏好统计
- **允许忽略**：✅
- **Notice**：`未添加性癖标签`

#### C-I3｜xpTags 含非 XP 语义（体检识别）
- **说明**：标签体系污染风险
- **影响**：XP 统计噪声
- **系统行为**：只提示，不自动修正
- **Notice**：`标签可能混入非性癖内容`

---

## 8. 规则执行顺序
`Error`（结构） → `Warn`（完整性） → `Info`（优化）

## 9. 系统禁止行为（写死）
❌ **不允许系统**：
- 自动补 `type`
- 自动猜 `platform`
- 自动生成 `title`
- 自动迁移为多个 ContentItem（除非用户确认）

## 10. 与 UI Notice System 的关系
- **Health Rule**：检测事实
- **NoticeItem**：承载结果
- **UI**：展示 + 行动

体检层永远不关心 UI 样式；UI 永远不判断数据正确性。

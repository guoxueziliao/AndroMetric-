
# v0.0.6 · 标签体检页面检测项规范（Final）

## 0. 标签体检页面的定位（先定清）

标签体检不是“修复工具”，而是一个 **长期风险仪表盘**。

它只做三件事：
1.  **发现问题**
2.  **描述问题**
3.  **引导用户自己处理**

❌ **绝不自动修改任何标签或历史数据**

---

## 1. 体检页面的核心输出结构

### 1.1 总览区（Overview）
必须展示：
*   当前标签总数（按类型拆分）
*   有问题的标签数量
*   风险等级分布
*   最近一次体检时间

### 1.2 问题列表区（Issues）
每一条问题必须包含：
*   **标签名称**：哪个标签
*   **标签类型**：XP / 事件 / 症状
*   **问题类型**：复合 / 冲突 / 冗余等
*   **风险等级**：高 / 中 / 低
*   **说明**：人类可理解的解释
*   **建议操作**：去拆分 / 去合并 / 去删除

---

## 2. 高风险检测项（High Severity）
**高风险 = 强烈建议尽快人工处理**。这些问题会直接破坏统计口径。

### H1. 复合语义标签（XP）
*   **检测条件**：单一 XP 标签同时命中 ≥2 个维度关键词，或明显包含「身份+行为」结构。
*   **示例**：`人妻出轨`, `制服诱惑`, `强制高潮`, `黑丝足交`
*   **风险说明**：该标签同时包含多个刺激维度，无法用于准确统计。
*   **建议**：拆分为多个 XP 标签

### H2. XP 维度冲突
*   **检测条件**：标签内容明显可归属两个以上 XP 维度。
*   **示例**：`痴女` (角色/行为?), `中年女性` (身体/角色?), `Cosplay` (形式/装扮?)
*   **风险说明**：标签维度不稳定，长期会导致统计结果失真。

### H3. 标签类型错放
*   **检测条件**：
    *   症状标签中出现事件/情绪词（如 `压力大`）
    *   事件标签中出现身体症状或 XP 词（如 `头痛`）
    *   XP 标签中出现状态/结果词（如 `很爽`）
*   **风险说明**：标签类型与语义不匹配，会破坏系统层级。

---

## 3. 中风险检测项（Medium Severity）
**中风险 = 可长期存在，但会逐渐增加维护成本**。

### M1. 同义 / 非标准标签
*   **检测条件**：存在已知的标准化替代词。
*   **示例**：`sp` (应为 `打屁股`), `做爱` (应为 `性爱`)
*   **风险说明**：同义标签会分散统计结果。
*   **建议**：合并到一个标准标签

### M2. 极低使用频率标签
*   **检测条件**：总使用次数 ≤ 1 且 创建时间较早
*   **风险说明**：可能是误建或一次性标签。
*   **建议**：考虑删除或合并

### M3. XP 标签维度分布失衡（暂缓）
*   *注：此项依赖完善的维度分类库，初期版本可暂不实现。*

---

## 4. 低风险检测项（Low Severity）
**低风险 = 仅做提醒，不影响系统正确性**。

### L1. 命名不规范
*   **检测条件**：包含多余空格、特殊符号或纯数字。
*   **示例**：` ntr`, `JK `, `123`
*   **建议**：统一命名风格

### L2. 标签长期未使用
*   **检测条件**：最近 90 天未被使用
*   **风险说明**：可能已经不再需要。

---

## 5. 风险等级的定义（统一口径）

| 等级 | 含义 | 行动建议 |
| :--- | :--- | :--- |
| **高** | 会破坏统计口径 | 尽快人工处理 |
| **中** | 会增加维护成本 | 有空时处理 |
| **低** | 仅影响整洁度 | 可忽略 |

---

## 6. 体检页面的交互边界（必须写死）

*   **系统可以做的**：高亮问题、分类展示、跳转到「标签管理」并定位标签
*   **系统不可以做的**：自动拆分、自动合并、自动迁移类型、自动改名

---

## 7. 一句话总结
体检不是为了“马上修好”，而是为了“你永远知道系统哪里开始变脏”。

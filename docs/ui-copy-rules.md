# UI 文案不可破坏原则  
Data-Facing Copy Rules

---

## 核心目标

所有面向数据的 UI 文案，
必须优先保证：

- 直观
- 可理解
- 长期稳定

---

## 不可破坏原则（摘要）

1. 文案必须直接指向记录的事实  
2. 标题只负责指路，不承载情绪  
3. 用户在登记事实，而不是讲故事  
4. 一条文案只回答一个问题  
5. 避免只有当前版本才懂的词  
6. 文案不教育用户，只避免误导  
7. 所有数值必须明确单位  
8. 不进行任何价值判断  
9. 结构性文案不得反复更名  
10. 未来优先于当下表达欲  

---

## 允许与禁止

### 禁止
- 隐喻
- 玩梗
- 中二表达
- 英文缩写作为标题

### 允许
- 中性描述
- 事实指向
- 必要说明

---

## 最终声明

凡违反以上原则的 UI 文案修改，
即使短期看起来更“有趣”，
也应被视为长期风险。

在不确定是否应修改时，
请选择不修改。
